Wave 6 Lane 1 は、FEATURES.md の **Company API** セクションに対応する。EDINET に登録された 11,223 件の提出者（上場・非上場・個人・外国法人を含む全提出者）を名前・EDINET コード・証券コード・業種で検索し、`Company` オブジェクトとして返す機能を提供する。

### 1.4 証券コードの 4 桁/5 桁問題

EDINET API の `sec_code` は末尾チェックディジット付きの **5 桁** で返る（例: `"72030"`）。
一般的には **4 桁**（`"7203"`）で検索されることが多い。

正規化ルール:
- `"7203"` → `"72030"` （4 桁の場合は末尾に `"0"` を付与して 5 桁にする）
- `"72030"` → `"72030"` （5 桁はそのまま）

`Company.ticker` プロパティは既に `sec_code[:4]` で 4 桁を返す実装がある。

## 2. ゴール

1. `Company.search(query, *, limit=20) -> list[Company]` — 企業名の部分一致検索
2. `Company.from_edinet_code(code) -> Company | None` — EDINET コードから Company 構築
3. `Company.from_sec_code(sec_code) -> Company | None` — 証券コード検索
4. `Company.by_industry(industry, *, limit=100) -> list[Company]` — 業種名で検索
5. `Company.all_listed(*, limit=0) -> list[Company]` — 上場企業一覧

完了条件:

```python
from edinet.models.company import Company

# 名前検索
results = Company.search("トヨタ")
assert len(results) >= 1
assert any(c.name_ja and "トヨタ自動車" in c.name_ja for c in results)

# 英語名検索（case-insensitive）
results = Company.search("toyota")
assert len(results) >= 1

# 読み仮名検索
results = Company.search("トヨタジドウシャ")
assert len(results) >= 1

# EDINET コード検索
company = Company.from_edinet_code("E02144")
assert company is not None
assert company.name_ja == "トヨタ自動車株式会社"

# 証券コード検索（4 桁）
company = Company.from_sec_code("7203")
assert company is not None
assert company.edinet_code == "E02144"

# 証券コード検索（5 桁）
company = Company.from_sec_code("72030")
assert company is not None

# 業種検索
results = Company.by_industry("輸送用機器")
assert len(results) >= 1

# 上場企業一覧
listed = Company.all_listed()
assert len(listed) > 100  # 上場企業は数千社

# limit
results = Company.search("トヨタ", limit=5)
assert len(results) <= 5
```

## 3. スコープ / 非スコープ

### 3.1 スコープ

| # | 内容 | 対応する API |
|---|------|------------|
| S1 | 日本語企業名の部分一致検索（`submitter_name`） | `Company.search()` |
| S2 | 英語企業名の部分一致検索（`submitter_name_en`、case-insensitive） | `Company.search()` |
| S3 | 読み仮名の部分一致検索（`submitter_name_yomi`） | `Company.search()` |
| S4 | EDINET コードからの Company 構築 | `Company.from_edinet_code()` |
| S5 | 証券コードからの Company 構築（4 桁/5 桁対応） | `Company.from_sec_code()` |
| S6 | 業種名の完全一致検索 | `Company.by_industry()` |
| S7 | 上場企業の一覧取得 | `Company.all_listed()` |
| S8 | `tools/generate_edinet_codes.py` への検索ヘルパー関数テンプレート追加 | `search_edinet_codes()` 等 |

### 3.2 非スコープ

| # | 内容 | 理由 |
|---|------|------|
| N1 | ファジーマッチング（編集距離、N-gram） | 外部ライブラリ依存を避ける。v0.2.0 以降で検討 |
| N2 | 読み仮名の正規化（カタカナ↔ひらがな変換） | 同上 |
| N3 | 正規表現ベースの検索 | 複雑度が高い。部分一致で十分 |
| N4 | Company フィールドの拡張（`industry`, `address` 等） | D3: God Object 防止。`EdinetCodeEntry` 経由で取得可 |
| N5 | `EdinetCodeEntry` の API ダウンロード更新 | v0.2.0 以降。現状は静的レジストリ |
| N6 | 外部ライブラリ依存（rapidfuzz, jellyfish 等） | ゼロ依存ポリシー |
| N7 | 全角英数字の正規化（`"ＴＯＹＯＴＡ"` → `"TOYOTA"` 等） | v0.1.0 では半角のみ対応。レジストリの英語名は半角で格納されている |
| N8 | 業種名一覧取得 API（`Company.industries()` 等） | 便利だが v0.1.0 のスコープ外。`all_edinet_codes()` から集計可能 |

### 4.0 `tools/generate_edinet_codes.py` の修正（自動生成対応）

**背景**: `src/edinet/models/edinet_code.py` は `tools/generate_edinet_codes.py` による自動生成ファイルであり、直接編集すると次回の再生成で上書きされて消失する。検索ヘルパー関数は generator の `render_edinet_module()` にテンプレートとして追加する。

**修正箇所**: `render_edinet_module()` 関数の末尾（現在の `all_edinet_codes()` と `_reset_warning_state()` の間）に、以下の関数テンプレートを追加する。

```python
lines.append("")

lines.append("    sec_code = sec_code.strip()")
lines.append("    if len(sec_code) == 4:")
lines.append('        return sec_code + "0"')
lines.append("    return sec_code")

lines.append("")
lines.append("")
lines.append("def _match_rank(entry: EdinetCodeEntry, query_lower: str) -> int:")
lines.append('    """エントリと検索クエリのマッチ度を返す。')
lines.append("")
lines.append("    Returns:")
lines.append("        0: submitter_name が query で始まる（最優先）")
lines.append("        1: submitter_name_en が query で始まる（case-insensitive）")
lines.append("        2: submitter_name_yomi が query で始まる")
lines.append("        3: いずれかのフィールドに query を含む（部分一致）")
lines.append("        -1: マッチしない")

lines.append("    name_lower = entry.submitter_name.lower()")

lines.append("        return 0")
lines.append("    name_en_lower = entry.submitter_name_en.lower() if entry.submitter_name_en else None")
lines.append("    if name_en_lower and name_en_lower.startswith(query_lower):")
lines.append("        return 1")
lines.append("    name_yomi_lower = entry.submitter_name_yomi.lower() if entry.submitter_name_yomi else None")
lines.append("    if name_yomi_lower and name_yomi_lower.startswith(query_lower):")

lines.append("    if query_lower in name_lower:")
lines.append("        return 3")
lines.append("    if name_en_lower and query_lower in name_en_lower:")
lines.append("        return 3")
lines.append("    if name_yomi_lower and query_lower in name_yomi_lower:")
lines.append("        return 3")
lines.append("    return -1")

lines.append("    結果は関連度順にソートされる:")
lines.append("    - 名前が query で始まるもの（prefix match）が先頭")
lines.append("    - 名前に query を含むもの（contains match）が後続")
lines.append("")
lines.append("    Args:")
lines.append("        query: 検索キーワード。空文字の場合は空リストを返す。")
lines.append("        limit: 最大返却件数。0で無制限。")
lines.append("")
lines.append("    Returns:")
lines.append("        マッチしたEdinetCodeEntryのリスト（関連度順）。")
```

### 4.1 生成される検索ヘルパー関数の仕様

以下は `edinet_code.py` に生成される関数の仕様。実装は §4.0 の generator テンプレートで定義済み。

#### `search_edinet_codes(query, *, limit=20) -> list[EdinetCodeEntry]`

企業名で EdinetCodeEntry を部分一致検索する。`submitter_name` / `submitter_name_en` / `submitter_name_yomi` のいずれかに `query` が含まれるエントリを返す。大文字・小文字を区別しない（`str.lower()` で比較）。

- 空文字列 → 空リスト
- `limit=0` → 無制限
- 結果順序: **関連度順**（`_match_rank()` によるソート）。同一ランク内は `_EDINET_CODE_TABLE` の挿入順（≒EDINET コード昇順）。
  - rank 0: `submitter_name` が query で始まる（prefix match、最優先）
  - rank 1: `submitter_name_en` が query で始まる（case-insensitive）
  - rank 2: `submitter_name_yomi` が query で始まる
  - rank 3: いずれかのフィールドに query を含む（contains match）

**注意**: ソートのため全件スキャンが必要。limit を指定しても全マッチを収集してからソート → 切り詰めとなる。11,223 件の線形スキャン + ソートは < 20ms で完了するため問題にならない。

#### `get_edinet_code_by_sec_code(sec_code) -> EdinetCodeEntry | None`

証券コードから EdinetCodeEntry を返す。4 桁の場合は `_normalize_sec_code()` で末尾に `"0"` を付与して 5 桁にしてから検索する。

#### `get_edinet_codes_by_industry(industry, *, limit=100) -> list[EdinetCodeEntry]`

業種名の**完全一致**で EdinetCodeEntry を検索する。

#### `get_listed_edinet_codes(*, limit=0) -> list[EdinetCodeEntry]`

`listing_status == "上場"` のエントリを返す。

**性能の考慮**: `all_edinet_codes()` は 11,223 件のリストを返す。線形探索だが、Python のループで 11,223 件は < 10ms で完了するため、インデックス構築は不要。将来的にパフォーマンスが問題になった場合は `_SEC_CODE_INDEX: dict[str, str]` を generator で生成時に追加すればよい。

```python
@classmethod
def search(cls, query: str, *, limit: int = 20) -> list[Company]:
    """企業名でCompanyを部分一致検索する。

    ``submitter_name`` / ``submitter_name_en`` / ``submitter_name_yomi`` の
    いずれかに ``query`` が含まれるエントリをCompanyに変換して返す。
    大文字・小文字を区別しない。

    Args:
        query: 検索キーワード。空文字の場合は空リストを返す。
        limit: 最大返却件数。0で無制限。デフォルト20。

    Returns:
        マッチしたCompanyのリスト。

    利用例:
        >>> results = Company.search("トヨタ")
        >>> results[0].name_ja
        'トヨタ自動車株式会社'
    """
    from edinet.models.edinet_code import search_edinet_codes

    entries = search_edinet_codes(query, limit=limit)
    return [_entry_to_company(e) for e in entries]
```

### 4.3 EdinetCodeEntry → Company 変換ヘルパー

`company.py` のモジュールレベルに非公開ヘルパーを追加:

```python
def _entry_to_company(entry: EdinetCodeEntry) -> Company:
    """EdinetCodeEntryからCompanyを構築する。

    Args:
        entry: 変換元のEdinetCodeEntry。

    Returns:
        Companyオブジェクト。

    Note:
        ``name_ja`` は ``EdinetCodeEntry.submitter_name``（EDINET コード一覧 CSV 由来）
        を使用する。``Company.from_filing()`` は ``Filing.filer_name``（API レスポンス由来）
        を使用するため、同一企業でも情報源の違いにより値が異なる場合がある。
    """
    return Company(
        edinet_code=entry.edinet_code,
        name_ja=entry.submitter_name,
        sec_code=entry.sec_code,
    )
```

**import 方針**: `_entry_to_company` の型注釈で `EdinetCodeEntry` を参照するため、`company.py` の既存 `TYPE_CHECKING` ブロックに `EdinetCodeEntry` の import を追加する:

```python
if TYPE_CHECKING:
    from edinet.models.doc_types import DocType
    from edinet.models.edinet_code import EdinetCodeEntry  # 追加
    from edinet.models.filing import Filing
```

各クラスメソッド内の遅延 import（`from edinet.models.edinet_code import search_edinet_codes` 等）は循環 import 防止のためそのまま維持する。`TYPE_CHECKING` import は型チェッカー用であり実行時には評価されないため、循環 import を引き起こさない。

**設計判断**: `_entry_to_company` は `company.py` 内の非公開関数とする。`EdinetCodeEntry` の全フィールドを Company に転写するのではなく、`Company` の既存 3 フィールド（`edinet_code`, `name_ja`, `sec_code`）のみを使う。これは D3（LineItem/Company にフィールドを追加しない）の方針に従う。

ユーザーが `industry` や `address` 等の詳細情報を必要とする場合は、`EdinetCodeEntry` を直接使う:

```python
from edinet.models.edinet_code import get_edinet_code

entry = get_edinet_code("E02144")
print(entry.industry)   # "輸送用機器"
print(entry.address)    # "愛知県豊田市..."
```

## 5. 実装の注意点

### 5.1 大文字・小文字の無視

英語名の検索は case-insensitive にする。`query.lower()` と `submitter_name_en.lower()` を比較。

日本語名・読み仮名は基本的に大文字・小文字の区別がないが、全角/半角の違いがある。v0.1.0 では正規化せず、ユーザーが入力した文字列をそのまま `in` 演算子で部分一致検索する。全角英数字（`"ＴＯＹＯＴＡ"` 等）での検索は非スコープ（N7）。

### 5.2 空クエリの処理

`Company.search("")` は空リスト `[]` を返す。全件返すのは意図しない API 呼び出しを防ぐため。

### 5.3 limit=0 の意味

`limit=0` は「制限なし」を意味する。`limit > 0` の場合のみ件数制限を適用する。

### 5.4 既存メソッドとの衝突回避

`Company.from_filing()` は既に存在する。`from_edinet_code()` / `from_sec_code()` は新しいクラスメソッドとして追加するが、`from_filing()` のシグネチャ・動作は一切変更しない。

### 5.5 自動生成ファイルの安全性

`edinet_code.py` の手動編集は禁止。以下の手順を厳守する:

1. `tools/generate_edinet_codes.py` の `render_edinet_module()` を修正
2. `uv run python tools/generate_edinet_codes.py` で再生成
3. 再生成後に既存テスト（`tests/test_models/test_edinet_code.py`）が PASS することを確認
