## 3. Namespace バージョン不一致バグの発見・修正

### 3.1 問題

トヨタ自動車の有報（S100VWVY）をフルパイプラインに通したところ、
**FALLBACK 率 91.1%** が発生。

原因: タクソノミパッケージ `ALL_20251101` は `2025-11-01` 版の namespace URI でラベルを保持。
一方、実データは `2024-11-01` 版の namespace URI を使用。
`_ns_to_prefix` が完全一致のみだったため、ほぼ全概念が FALLBACK に落ちていた。

### 3.2 修正

`taxonomy.py` に以下を追加（約15行）:

- `_NS_VERSION_RE = re.compile(r"/\d{4}-\d{2}-\d{2}/")` — namespace URI の日付部分を正規化
- `_normalized_ns_to_prefix` 辞書 — 日付除去した正規化キーで再検索
- `resolve_clark()` のフォールバック — 完全一致 → 正規化一致の2段階

### 3.3 結果

トヨタ自動車: FALLBACK 91.1% → **0.0%**

QA 参照: H-3（タクソノミバージョニング）, Q-2（バージョン不一致のリカバリ）

### 4.4 FALLBACK の原因分析

FALLBACK が残った6書類タイプ（240, 250, 290, 300, 350, 360）を調査。

スクリプト: `tools/day14_fallback_debug.py`

**原因: EDINET タクソノミの deprecated（廃止済み）概念**

FALLBACK 概念の特徴:
- 全て末尾が `NA` サフィックスで終わる（例: `NameOfLegalRepresentativeCoverPageNA`）
- 現行 `2025-11-01` 版では `deprecated/` ディレクトリに移動済み
- 対応する現行版概念（サフィックスなし）は `label/` にラベルが存在
- 実データの namespace URI はバージョンが古い（`2020-11-01`, `2013-08-31`, `2014-07-31`）

deprecated ラベルファイルは `deprecatedLabel` ロール（廃止理由テキスト）のみで、
表示用ラベル（`role/label`）を含まない。

| 書類タイプ | 実データの NS バージョン | タクソノミの NS バージョン | 差分 |
|---|---|---|---|
| 公開買付届出書 (240) | `2020-11-01` | `2025-11-01` | 5年 |
| 意見表明報告書 (290) | `2013-08-31` | `2025-11-01` | 12年 |
| 大量保有報告書 (350) | `2014-07-31` | `2025-11-01` | 11年 |

**影響**: v0.1.0 スコープの書類タイプ（120, 140, 160）には影響なし。
v0.2.0 で公開買付・大量保有を対象にする際に `taxonomy/versioning`（FEATURES.md）で対応する。
