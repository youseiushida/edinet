## E-2. 連結/個別の区別

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。

| サブ質問 | 回答 |
|----------|------|
| E-2.1 連結/個別の dimension | Dimension = `jppfs_cor:ConsolidatedOrNonConsolidatedAxis`、個別 Member = `jppfs_cor:NonConsolidatedMember`。Context の `scenario` 内に `xbrldi:explicitMember` で設定される。 |
| E-2.2 dimension なしの Fact | **連結**として扱う。XBRL Dimensions 仕様のデフォルトメンバーは `ConsolidatedMember` であり、連結の場合は scenario にメンバーを設定しない（デフォルトが適用される）。dimension が一切ない Context の Fact は連結データである。 |
| E-2.3 連結財務諸表がない企業 | 個別のみの Fact は `_NonConsolidatedMember` 付き Context を使用する。DEI の `WhetherConsolidatedFinancialStatementsArePreparedDEI` = `false` で連結なし企業を判別可能。連結なし企業でも、個別 Fact には必ず `NonConsolidatedMember` ディメンションが付与される。 |

### 連結/個別の Context 構造

#### 連結（デフォルト: scenario なし）

```xml
<xbrli:context id="CurrentYearInstant">
  <xbrli:entity>
    <xbrli:identifier scheme="http://disclosure.edinet-fsa.go.jp">X99001-000</xbrli:identifier>
  </xbrli:entity>
  <xbrli:period>
    <xbrli:instant>2026-03-31</xbrli:instant>
  </xbrli:period>
  <!-- scenario なし = 連結（デフォルト） -->
</xbrli:context>
```

Context ID に `_NonConsolidatedMember` が含まれない = 連結。

#### 個別（scenario に NonConsolidatedMember を設定）

```xml
<xbrli:context id="CurrentYearInstant_NonConsolidatedMember">
  <xbrli:entity>
    <xbrli:identifier scheme="http://disclosure.edinet-fsa.go.jp">X99001-000</xbrli:identifier>
  </xbrli:entity>
  <xbrli:period>
    <xbrli:instant>2026-03-31</xbrli:instant>
  </xbrli:period>
  <xbrli:scenario>
    <xbrldi:explicitMember dimension="jppfs_cor:ConsolidatedOrNonConsolidatedAxis">jppfs_cor:NonConsolidatedMember</xbrldi:explicitMember>
  </xbrli:scenario>
</xbrli:context>
```

Context ID に `_NonConsolidatedMember` が含まれる = 個別。

### Dimension の詳細

| 属性 | 値 |
|------|-----|
| Axis（軸） | `jppfs_cor:ConsolidatedOrNonConsolidatedAxis` |
| デフォルト Member | `jppfs_cor:ConsolidatedMember`（連結） |
| 個別 Member | `jppfs_cor:NonConsolidatedMember` |
| 設定場所 | Context の `scenario` 内（`segment` は使用しない） |
| 設定方法 | `xbrldi:explicitMember` 要素 |

XBRL Dimensions 仕様では、デフォルトメンバーの場合は scenario にメンバーを設定しない。EDINET ではデフォルトメンバーが `ConsolidatedMember`（連結）に設定されているため:
- **連結**: scenario が存在しない（またはこの軸のメンバーが含まれない）
- **個別**: scenario に `NonConsolidatedMember` が明示的に設定される

### Context ID の命名パターン

A-2.a.md の命名規則に従い、連結/個別は Context ID のサフィックスで区別される。

| 区分 | Context ID 例（instant） | Context ID 例（duration） |
|------|------------------------|-------------------------|
| 連結 | `CurrentYearInstant` | `CurrentYearDuration` |
| 個別 | `CurrentYearInstant_NonConsolidatedMember` | `CurrentYearDuration_NonConsolidatedMember` |
| 連結前期 | `Prior1YearInstant` | `Prior1YearDuration` |
| 個別前期 | `Prior1YearInstant_NonConsolidatedMember` | `Prior1YearDuration_NonConsolidatedMember` |

### 連結財務諸表がない企業

連結決算を行わない企業（非上場企業や子会社を持たない企業等）の場合:

1. DEI の `WhetherConsolidatedFinancialStatementsArePreparedDEI` が `false`
2. 財務データの Fact は全て `_NonConsolidatedMember` 付き Context を使用
3. dimension なし（連結用）の Context には財務 Fact が存在しない

DEI 判定例:
```xml
<jpdei_cor:WhetherConsolidatedFinancialStatementsArePreparedDEI
  contextRef="FilingDateInstant">false</jpdei_cor:WhetherConsolidatedFinancialStatementsArePreparedDEI>
```

### 実装上の注意点

1. **判定方法**: Context の `scenario` に `ConsolidatedOrNonConsolidatedAxis` の `NonConsolidatedMember` が含まれるかで判定する。含まれなければ連結。
2. **Context ID でも判定可能**: `_NonConsolidatedMember` サフィックスの有無で簡易判定可能（EDINET の命名規則が保証）。ただし Context の XML 構造から正式に判定するほうが堅牢。
3. **他の Dimension との共存**: 個別 Context にセグメント軸等の他の Dimension が同時に存在する場合がある。例: `CurrentYearDuration_NonConsolidatedMember_OperatingSegmentMember`。

### 推論（Reasoning）

1. **E-2.1 Dimension の特定**: [F1]+[F2]+[F3] より、連結/個別は `jppfs_cor:ConsolidatedOrNonConsolidatedAxis` 軸で区別される。[F7] より、個別の場合は scenario に `NonConsolidatedMember` を `xbrldi:explicitMember` で設定する。[F4] のサンプルインスタンスでも同構造が確認できる。

2. **E-2.2 デフォルトメンバーの解釈**: [F2]+[F3] より、EDINET ではデフォルトメンバーの場合は scenario に設定しない仕様。[F2] で連結が `ConsolidatedMember`（デフォルト）であり scenario に設定不要と明記されている。したがって dimension が一切ない Context の Fact は、全ての Dimension がデフォルト値（連結軸ではConsolidatedMember = 連結）であることを意味する。

3. **E-2.3 連結なし企業**: [F6] より、`WhetherConsolidatedFinancialStatementsArePreparedDEI` = `false` で連結なし企業を判別できる。連結なし企業の場合、連結用 Context（scenario なし）には財務 Fact が存在せず、全ての財務 Fact は `_NonConsolidatedMember` 付き Context に紐づく。連結の有無に関わらず、個別の Fact には常に `NonConsolidatedMember` ディメンションが付与されるという一貫したルールが適用される。
