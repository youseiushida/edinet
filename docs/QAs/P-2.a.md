### 推論（Reasoning）

1. [F1]+[F2]+[F3] より、IXDS（Inline XBRL Document Set）構造において、Context と Unit の定義は **表紙ファイル (header)** の `ix:resources` 内に集約されている。仕様書 Instance Guideline L.2097 の記述と一致。
2. [F4] より、本文ファイルには `ix:header` が存在せず、Context/Unit の定義もない。IXDS では表紙ファイルの ix:resources が全ファイル共通の定義源。
3. [F5] より、schemaRef は相対パスで提出者別タクソノミ `.xsd` を参照。DTS 解決は `.xsd` → `import` の URL から EDINET タクソノミを辿る構造。
4. [F6] より、名前空間プレフィックスは提出者間で一貫している（`jpcrp_cor`, `jppfs_cor`, `jpdei_cor` 等は標準タクソノミ、提出者別は `jpcrp030000-asr_E02144-000` のような固有プレフィックス）。
