## E-3. 期間の取り扱い

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。

| サブ質問 | 回答 |
|----------|------|
| E-3.1 複数期間データ | **全て別の Context で区別される**。当期 = `CurrentYear*`、前期 = `Prior1Year*`、前々期 = `Prior2Year*`。有価証券届出書では `Prior5Year*` まで存在しうる。各 Context は異なる period（日付範囲）を持つ。 |
| E-3.2 四半期の期間表現 | 2025 年版タクソノミで**四半期の ELR は廃止済**。中間期は `Interim` 系 Context を使用し、累計期間のみ（単独四半期なし）。有価証券届出書で四半期情報を開示する場合も、タグ付けは不要とされている。 |
| E-3.3 BS の期間 | **期末時点の instant のみ**。BS には duration Context は使われない。期首時点の Context は作成せず、`Prior1YearInstant`（前期末）が期首と一致する。 |
| E-3.4 CF の期間 | **duration（累計期間）**。PL と同じ `CurrentYearDuration` / `Prior1YearDuration` を使用。 |
| E-3.5 Filing period と Context period の関係 | **当期の Context period は原則一致**する（`CurrentYearDuration` の startDate/endDate が Filing の period_start/period_end と一致）。ただし比較期間の Context（`Prior1Year*` 等）は Filing period の範囲外であり、これは正常。 |
| E-3.6 変則的な会計期間 | **実際の日付をそのまま記入**する。例: 15 ヶ月の移行期間は `startDate=2024-01-01`, `endDate=2025-03-31`。Context ID の命名パターン（`CurrentYearDuration` 等）は変わらない。 |
| E-3.7 連結/個別の期末ずれ | 連結会計年度と個別事業年度の期末が異なる場合、**別の period を持つ別の Context が定義される**。ただし実務上は連結と個別で同一期末であることがほとんど。 |
| E-3.8 非 3 月決算企業 | **同一の命名パターン**（`CurrentYear*`, `Prior1Year*` 等）を使用する。period の日付値のみが異なる。12 月決算なら instant=`2025-12-31`、6 月決算なら instant=`2026-06-30` 等。構造的な違いはない。 |

### Context ID と期間の対応表

#### 有価証券報告書（通期）の典型例（3 月決算）

| Context ID | 種別 | period | 用途 |
|-----------|------|--------|------|
| `FilingDateInstant` | instant | 2026-06-12 | DEI・表紙 |
| `CurrentYearInstant` | instant | 2026-03-31 | 当期末 BS |
| `CurrentYearDuration` | duration | 2025-04-01 〜 2026-03-31 | 当期 PL/CF/SS |
| `Prior1YearInstant` | instant | 2025-03-31 | 前期末 BS（= 当期首） |
| `Prior1YearDuration` | duration | 2024-04-01 〜 2025-03-31 | 前期 PL/CF |
| `Prior2YearInstant` | instant | 2024-03-31 | 前々期末 |

各 Context に `_NonConsolidatedMember` サフィックス付きの個別版も存在する。

#### 半期報告書の典型例

| Context ID | 種別 | period | 用途 |
|-----------|------|--------|------|
| `FilingDateInstant` | instant | 2026-11-19 | DEI・表紙 |
| `InterimInstant` | instant | 2026-09-30 | 当中間期末 BS |
| `InterimDuration` | duration | 2026-04-01 〜 2026-09-30 | 当中間期 PL/CF |
| `Prior1InterimInstant` | instant | 2025-09-30 | 前中間期末 BS |
| `Prior1InterimDuration` | duration | 2025-04-01 〜 2025-09-30 | 前中間期 PL/CF |
| `Prior2YearInstant` | instant | 2025-03-31 | 前期末（比較用） |

#### 有価証券届出書の典型例（5 年分の経営指標）

| Context ID | 種別 | period | 備考 |
|-----------|------|--------|------|
| `Prior5YearDuration` | duration | 2021-04-01 〜 2022-03-31 | 5 期前 |
| `Prior4YearDuration` | duration | 2022-04-01 〜 2023-03-31 | 4 期前 |
| `Prior3YearDuration` | duration | 2023-04-01 〜 2024-03-31 | 3 期前 |
| `Prior2YearDuration` | duration | 2024-04-01 〜 2025-03-31 | 前々期 |
| `Prior1YearDuration` | duration | 2025-04-01 〜 2026-03-31 | 前期 |
| `InterimDuration` | duration | 2026-04-01 〜 2026-09-30 | 次の中間期 |

注: 届出書に次の中間期を含む場合、最新の通期は `CurrentYear` ではなく `Prior1Year` となる（ガイドライン §7-5）。

### 詳細解説

#### E-3.1 複数期間の Context

ガイドライン §5-4-4 の図表 5-4-4 に定義される相対期間:

| Context ID プレフィックス | 意味 |
|-------------------------|------|
| `CurrentYear` | 当年度 |
| `Interim` | 中間期 |
| `Prior1Year` | 前年度 |
| `Prior1Interim` | 前中間期 |
| `Prior2Year` | 前々年度 |
| `Prior{n}Year` | n 年度前 |
| `Prior{n}Interim` | n 年度前中間期 |
| `FilingDate` | 提出日 |
| `RecentDate` | 最近日 |
| `FutureDate` | 予定日 |
| `RecordDate` | 議決権行使の基準日 |

各プレフィックスに `Instant` または `Duration` を付加して Context ID が構成される。

#### E-3.2 四半期の廃止

2025 年版タクソノミで四半期の ELR（`310030 四半期連結貸借対照表`、`321031 四半期連結損益計算書 累計期間` 等）は廃止済みである（I-1.a.md で確認済み）。ただし四半期を表す Context ID（`QuarterlyXxx` 等）は ELR の問題であり Context 命名自体とは独立。

中間期は `Interim` プレフィックスの Context を使用する。半期報告書の PL は累計期間（期首から中間期末まで）の duration のみ。単独の四半期期間（例: 第 2 四半期のみの 3 ヶ月間）の Context は標準では定義されない。

有価証券届出書の「経営指標等の推移」セクションで四半期情報を記載する場合でも、XBRL タグ付けは不要とされている（EDINET タクソノミ概要説明より）。

#### E-3.3 BS は期末 instant のみ

BS の concept は全て `periodType="instant"` で定義されている。期首時点の Context は作成せず、前期末（`Prior1YearInstant`）が当期首と一致する（ガイドライン §5-4-4-1「期首日開始時点のコンテキスト」）。

例: 当期首の総資産が必要な場合は `Prior1YearInstant` の Context で `TotalAssets` を参照する。

#### E-3.4 CF は duration

CF の concept は `periodType="duration"` で定義されている。PL と同じ `CurrentYearDuration`（累計期間）を使用する。中間期の CF も `InterimDuration`（期首から中間期末までの累計）。

#### E-3.5 Filing period との関係

EDINET API の `Filing.period_start` / `period_end` は報告対象期間を示す。当期の Context period と原則一致する:
- `period_start` ≒ `CurrentYearDuration` の `startDate`
- `period_end` ≒ `CurrentYearInstant` の `instant` = `CurrentYearDuration` の `endDate`

ただし:
- **比較期間の Context**（`Prior1Year*` 等）は Filing period の範囲外。これは正常（前期比較データ）
- **訂正報告書**: API の period は原訂正対象書類の期間と同一。XBRL 内の Context period も訂正前と同一（期間は変わらない）
- **有価証券届出書**: Filing period は直近の通期だが、XBRL には次の中間期（Filing period 外）や 5 期分の過去データが含まれうる

#### E-3.6 変則期間

決算期変更に伴う変則期間（例: 15 ヶ月、6 ヶ月短縮決算）の場合、Context の period に実際の日付をそのまま記入する（ガイドライン §5-4-4-2）。

例: 3 月決算→ 12 月決算への変更で 9 ヶ月間の移行期間:
```xml
<xbrli:context id="CurrentYearDuration">
  <xbrli:period>
    <xbrli:startDate>2026-04-01</xbrli:startDate>
    <xbrli:endDate>2026-12-31</xbrli:endDate>  <!-- 9ヶ月間 -->
  </xbrli:period>
</xbrli:context>
```

Context ID は `CurrentYearDuration` のまま変わらない。期間の長短は日付の差分で判断する。

#### E-3.7 連結/個別の期末ずれ

連結会計年度と個別事業年度の期末が異なる場合（稀）:
- 連結用 Context: 連結会計年度の期末を使用
- 個別用 Context: 個別事業年度の期末を使用

例:
```xml
<!-- 連結: 3月決算 -->
<xbrli:context id="CurrentYearInstant">
  <xbrli:period><xbrli:instant>2026-03-31</xbrli:instant></xbrli:period>
</xbrli:context>

<!-- 個別: 12月決算（仮） -->
<xbrli:context id="CurrentYearInstant_NonConsolidatedMember">
  <xbrli:period><xbrli:instant>2025-12-31</xbrli:instant></xbrli:period>
  <xbrli:scenario>
    <xbrldi:explicitMember dimension="jppfs_cor:ConsolidatedOrNonConsolidatedAxis">jppfs_cor:NonConsolidatedMember</xbrldi:explicitMember>
  </xbrli:scenario>
</xbrli:context>
```

実務上は連結と個別で同一期末であることがほとんど。DEI の `CurrentFiscalYearStartDateDEI` は「連結会計年度と事業年度が異なる場合は連結会計年度の開始日」と定義されている（F-1.a.md）。

#### E-3.8 非 3 月決算企業

12 月決算（例: ソニーグループ）や 6 月決算企業でも、Context ID の命名パターンは同一:
- `CurrentYearInstant`: `2025-12-31`（12 月決算の場合）
- `CurrentYearDuration`: `2025-01-01` 〜 `2025-12-31`
- `Prior1YearInstant`: `2024-12-31`

命名規則は相対的（`CurrentYear` = 当期、`Prior1Year` = 前期）であり、絶対的な月日に依存しない。日付値のみが企業ごとに異なる。

### 推論（Reasoning）

1. **E-3.1 複数期間**: [F1]+[F4] より、EDINET のコンテキスト ID は相対的な期間名（`CurrentYear`, `Prior1Year` 等）で命名される。各 Context は固有の period 要素を持ち、異なる期間のデータを区別する。有価証券届出書では [F8] より `Prior5Year` まで存在しうる。

2. **E-3.2 四半期廃止**: [F10] より、2025 年版タクソノミで四半期 ELR は廃止済み。[F11] より、届出書の四半期情報はタグ付け不要。中間期は `Interim` 系 Context を使用し、[F5]+[F9] のサンプルで累計期間（startDate=期首, endDate=中間期末）の duration のみが定義されていることを確認。

3. **E-3.3 BS の instant**: [F2] より、期首日時点のコンテキストは作成せず前期末を使う仕様。BS は期末 instant のみで、期首が必要な場合は `Prior1YearInstant` を参照する。

4. **E-3.4 CF の duration**: [F4] のサンプルで CF の Fact が `CurrentYearDuration` を参照していることを確認。CF は PL と同一の duration Context を使用する。

5. **E-3.5 Filing period**: [F4] の `CurrentYearDuration` の period は会計期間と一致する。EDINET API の `period_start/period_end` は報告対象期間であるため、当期 Context と原則一致する。比較期間（`Prior1Year` 等）や届出書の過去データ（`Prior5Year` 等）は Filing period の範囲外だが正常。

6. **E-3.6 変則期間**: [F3] より、決算期変更時は実際の日付をそのまま Context の period に設定する。Context ID の `CurrentYearDuration` は変わらず、日付値のみが変則期間を反映する。

7. **E-3.7 連結/個別ずれ**: [F12] より、DEI で「連結会計年度と事業年度が異なる場合」への言及がある。この場合、連結用と個別用の Context は異なる period を持つ。ただし実務上はほぼ全企業で連結・個別が同一期末。

8. **E-3.8 非 3 月決算**: [F1] の命名規則は相対的（`CurrentYear` = 当期）であり、絶対的な月日に依存しない。[F4] の XML 例でも日付値は例示であり、企業ごとの決算期に応じた値が入る。構造的な違いはない。
