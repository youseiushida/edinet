### ZIP の標準構造

```
XBRL/
  PublicDoc/
    {命名規則}.xbrl           — XBRL インスタンス（EDINET 自動生成）
    {命名規則}.xsd            — 提出者別タクソノミスキーマ
    {命名規則}_lab.xml        — 提出者別ラベル（日本語）
    {命名規則}_lab-en.xml     — 提出者別ラベル（英語）
    {命名規則}_pre.xml        — 提出者別プレゼンテーション
    {命名規則}_cal.xml        — 提出者別計算
    {命名規則}_def.xml        — 提出者別定義
    0000000_header_*.htm      — iXBRL 表紙（ix:header, Context, Unit 定義）
    0101010_honbun_*.htm      — iXBRL 本文（複数ファイル）
    ...
    0201010_honbun_*.htm      — iXBRL 提出確認書
    manifest_PublicDoc.xml    — マニフェスト
    fuzoku/                   — 画像ファイル（.gif, .jpg, .png）
  AuditDoc/
    jpaud-aai-cc-*.xbrl       — 連結監査報告書 XBRL
    jpaud-aar-cn-*.xbrl       — 個別監査報告書 XBRL（有報の場合）
    jpaud-*.xsd               — 監査報告書タクソノミ
    jpaud-*_pre.xml           — 監査報告書プレゼンテーション
    jpaud-*_def.xml           — 監査報告書定義
    jpaud-*_ixbrl.htm         — 監査報告書 iXBRL
    manifest_AuditDoc.xml     — マニフェスト
```

### ファイル命名規則の詳細 (B-1.1)

PublicDoc のファイル名パターン:
```
jpcrp030000-asr-001_E02144-000_2025-03-31_01_2025-06-18.xbrl
│          │   │   │       │   │          │  │
│          │   │   │       │   │          │  └── 提出日
│          │   │   │       │   │          └── 提出回数（01=当初、02=1回目訂正）
│          │   │   │       │   └── 期末日（決算期末日）
│          │   │   │       └── 枝番（通常 000）
│          │   │   └── EDINETコード
│          │   └── 連番（通常 001）
│          └── 報告書略号: asr=有報, ssr=半期, q1r-q4r=四半期
└── 府令略号+様式番号: jpcrp030000=企業内容等開示府令 第三号様式
```

AuditDoc のファイル名パターン:
```
jpaud-aai-cc-001_E02144-000_2025-03-31_01_2025-06-18.xbrl
│     │   │
│     │   └── cc=連結(consolidated), cn=個別(non-consolidated)
│     └── aai=独立監査人の監査報告書(independent auditor)
│         aar=独立監査人の四半期レビュー報告書等
└── jpaud: 監査報告書タクソノミ
```

### 推論（Reasoning）

#### B-1.1 ファイル命名規則
1. [F1] より、PublicDoc のファイル名は 7 パートで構成される。正規表現 `{府令略号}{様式番号}-{報告書略号}-{連番}_{EDINETコード}-{枝番}_{期末日}_{提出回数}_{提出日}` で完全に分解可能。
2. AuditDoc のファイル名は `jpaud` プレフィックスで始まり、構造は PublicDoc と同様だが府令・様式が異なる。

#### B-1.2 複数 .xbrl
3. [F1]+[F2]+[F4]+[F5] より、有報 (doc_type=120/130) では PublicDoc 1 + AuditDoc 2 = 合計 3 件の .xbrl が標準。
4. [F3] より、四半期報告書では AuditDoc 1 件のみ（四半期レビュー報告書は連結/個別で分離しない）。

#### B-1.4 AuditDoc
5. [F1] より、AuditDoc は jpaud タクソノミを使用。リンクベースは _pre, _def のみ（_lab, _cal なし）。
6. AuditDoc は監査報告書のテキスト情報のみを含み、財務諸表の数値データは PublicDoc に集約される。v0.1.0 では AuditDoc のパースは不要。

#### B-1.5 リンクベース
7. [F1]+[F2] より、PublicDoc には _lab, _lab-en, _pre, _cal, _def の5種が必ず存在する。_ref は全ターゲットで不在。

#### B-1.7 IFRS vs J-GAAP
8. [F6] より、ZIP のファイル構造は IFRS/J-GAAP で同一。差異は XSD の import 内容のみ。IFRS 企業は `jpigp_cor` / `jpigp_rt` を追加で import する。

#### B-1.9 エンコーディング
9. [F1]+[F2] より、ZIP 内の全パスは ASCII のみ。日本語ファイル名は存在しない。Shift_JIS エンコーディングへの対応は不要。

#### B-1.11 複数エンティティ
10. [F8] より、全ターゲットで EDINET コードは 1 つのみ。`find_primary_xbrl_path` のロジックに複数エンティティ対応は不要。

#### B-1.12 file_type=1 と 5 の共存
11. [F7] より、同一 doc_id で file_type=1（XBRL ZIP）と file_type=5（CSV ZIP）の両方がダウンロード可能。排他ではない。

#### B-1.13 META-INF
12. [F1]+[F11] より、提出者 ZIP にも ALL_20251101 にも META-INF は存在しない。XBRL Report Package 仕様には非準拠。DTS 解決は H-1b.a.md のプレフィックス置換で実装する。
