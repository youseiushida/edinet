## F-1. DEI 要素の一覧

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。

| サブ質問 | 回答 |
|----------|------|
| F-1.1 `FilerNameInJapaneseDEI` | 存在する。型は `xbrli:stringItemType`。日本語の提出者名を格納する。`periodType="instant"`、コンテキストは `FilingDateInstant`。 |
| F-1.2 `FilerNameInEnglishDEI` | 存在する。型は `xbrli:stringItemType`。英語の提出者名を格納する。半角で入力する必要がある。`periodType="instant"`。 |
| F-1.3 `SecurityCodeDEI` | 存在する。型は `xbrli:stringItemType`。証券コード協議会が付与するアラビア数字（例: `11110`）。該当する場合に記載、該当しない場合は nil。 |
| F-1.4 `EDINETCodeDEI` | 存在する。型は `xbrli:stringItemType`。EDINETコード（例: `X99001`）を格納する。 |
| F-1.5 `AccountingStandardsDEI` | 存在する。型は `xbrli:stringItemType`（enumerationItemType ではない）。値は `"Japan GAAP"` / `"US GAAP"` / `"IFRS"` / `"JMIS"` の4種。有報・届出書・半期報告書以外、または経理の状況が記載されない場合は nil。 |
| F-1.6 `WhetherConsolidatedFinancialStatementsArePreparedDEI` | 存在する。型は `xbrli:booleanItemType`。連結決算ありは `true`、なしは `false`。有報等以外・経理の状況なしの場合は nil。 |
| F-1.7 `CurrentFiscalYearEndDateDEI` | 存在する。型は `xbrli:dateItemType`。当事業年度終了日（決算日）。XBRL対象の連結/個別財務諸表がある場合に記載。 |
| F-1.8 `TypeOfCurrentPeriodDEI` | 存在する。型は `xbrli:stringItemType`。値は `"FY"`（年度）/ `"HY"`（中間期）。XBRL対象の財務諸表がある場合に記載。 |
| F-1.9 `IndustryCode...DEI` (業種コード) | 存在する（連結用・個別用の2要素）。型は `xbrli:stringItemType`。ラベルは「別記事業（連結）」「別記事業（個別）」。これは財務諸表等規則の「別記事業」に該当する場合の業種コードであり、EDINET API の「業種カテゴリ」(C-3) とは別概念。 |
| F-1.10 `DocumentIDDEI` (doc_id) | 該当する DEI concept は **存在しない**。`jpdei_cor` スキーマに `DocumentID` を名前に含む要素は定義されていない。訂正対象書類の管理番号を格納する `IdentificationOfDocumentSubjectToAmendmentDEI` は存在するが、自身の doc_id を格納する要素はない。EDINET API の `docID` に対応する値は XBRL インスタンス内の DEI には格納されない。 |
| F-1.11 その他の重要な DEI 要素 | 下記の全要素一覧を参照。 |

### 推論（Reasoning）

1. **F-1.1 ~ F-1.4**: [F1] の XSD 定義から、FilerNameInJapaneseDEI, FilerNameInEnglishDEI, SecurityCodeDEI, EDINETCodeDEI は全て `xbrli:stringItemType` で定義されている。[F5] の設定例からインスタンス内で `ix:nonnumeric` 要素として `FilingDateInstant` コンテキストで記載されることが確認できる。[F15] より英語表記の項目は半角入力が必須。

2. **F-1.5 (AccountingStandardsDEI)**: [F1] より型は `xbrli:stringItemType` である。`enumerationItemType` ではなく、XSD レベルでは自由な文字列型だが、[F9] のドキュメンテーションラベルおよび添付5（[F18]）により取りうる値が `"Japan GAAP"` / `"US GAAP"` / `"IFRS"` / `"JMIS"` / nil に限定されている。`"JMIS"`（修正国際基準）は documentation ラベルには記載がなく添付5でのみ明記されている。パーサー実装では事実上の列挙型として扱い、これら以外の値が出現した場合はバリデーションエラーとすべき。

3. **F-1.6 (WhetherConsolidatedFinancialStatementsArePreparedDEI)**: [F1] より型は `xbrli:booleanItemType`。[F10] より `true`（連結あり）/ `false`（連結なし）/ nil（対象外書類）の3値。

4. **F-1.7 (CurrentFiscalYearEndDateDEI)**: [F1] より型は `xbrli:dateItemType`。XBRL 標準では `YYYY-MM-DD` 形式の文字列で表現される。パーサーでは `datetime.date` への変換が必要。

5. **F-1.8 (TypeOfCurrentPeriodDEI)**: [F1] より型は `xbrli:stringItemType`。[F11] のドキュメンテーションラベルに `"FY"` (年度) と `"HY"` (中間期) の2値のみが明記されている。添付5（[F19]）でも「通期(FY)、中間期(HY)」の別を記載するとされており、四半期を示す値は存在しない。

6. **F-1.9 (IndustryCode...DEI)**: [F1] より連結用（`IndustryCodeWhenConsolidatedFinancialStatementsArePreparedInAccordanceWithIndustrySpecificRegulationsDEI`）と個別用（`IndustryCodeWhenFinancialStatementsArePreparedInAccordanceWithIndustrySpecificRegulationsDEI`）の2要素が存在する。いずれも `xbrli:stringItemType`。[F2] より日本語ラベルは「別記事業（連結）」「別記事業（個別）」。これは財務諸表等規則における「別記事業」（銀行、保険、電力等）に該当する場合の業種コードであり、EDINET API の提出書類一覧で返される業種カテゴリ（C-3 で議論される分類体系）とは異なる概念である。

7. **F-1.10 (DocumentIDDEI)**: [F1] の XSD を網羅的に確認した結果、`DocumentID` を名前に含む要素は存在しない。XSD 内の全34要素を調べたが、doc_id（EDINET API が返す書類管理番号 `docID`）を格納するための DEI concept は定義されていない。[F12] の `IdentificationOfDocumentSubjectToAmendmentDEI` は「訂正対象の当初提出書類の書類管理番号」であり、提出書類自身の doc_id ではない。従って、XBRL インスタンスから EDINET API の `docID` を直接取得する手段は DEI にはなく、両者の紐付けはファイル名の命名規約やメタデータ等から別途行う必要がある。

8. **F-1.11 (その他重要な要素)**: [F1] より、質問で明示的に列挙されていない重要な DEI 要素として以下が存在する:
   - `FundCodeDEI` / `FundNameInJapaneseDEI` / `FundNameInEnglishDEI`: ファンド（投資信託等）固有の情報
   - `CabinetOfficeOrdinanceDEI`: 根拠となる府令名
   - `DocumentTypeDEI`: 様式名（例: 「第三号様式」）
   - `NumberOfSubmissionDEI`: 提出回数（`nonNegativeIntegerItemType`）
   - `AmendmentFlagDEI`: 訂正の有無（`booleanItemType`）
   - `IdentificationOfDocumentSubjectToAmendmentDEI`: 訂正対象書類の書類管理番号
   - `ReportAmendmentFlagDEI` / `XBRLAmendmentFlagDEI`: 訂正の種別フラグ
   - 各種日付系要素（`CurrentFiscalYearStartDateDEI`, `CurrentPeriodEndDateDEI`, `PreviousFiscalYearStartDateDEI`, `ComparativePeriodEndDateDEI`, `PreviousFiscalYearEndDateDEI`, `NextFiscalYearStartDateDEI`, `EndDateOfQuarterlyOrSemiAnnualPeriodOfNextFiscalYearDEI`）

9. **追加の実装上の注意点**:
   - [F16] より DEI タクソノミは定義リンクのみで定義され、表示リンクには含まれない。
   - [F17] より開示書類等提出者が DEI タクソノミに独自の要素を追加することはない。つまり上記34要素が DEI の全要素である。
   - 全ての非Abstract要素は `nillable="true"` であり、`xsi:nil="true"` でnil値を取りうる。
   - 全ての非Abstract要素は `periodType="instant"` であり、コンテキストは `FilingDateInstant`（提出日時点）を使用する。
