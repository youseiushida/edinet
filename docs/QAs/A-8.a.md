## A-8. Fact 要素の id 属性と xml:lang 属性

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| A-8.1 id 属性 | 一部の Fact 要素に `id` 属性が付与される。フットノートリンク (`link:footnoteLink`) との紐付けに使われる。id は `IdFact{数字}` 形式（例: `IdFact1758417265`）。フットノートの `link:loc` が `xlink:href="#IdFact..."` で参照する |
| A-8.2 xml:lang 属性 | Fact 要素自体には `xml:lang` は付与されない。`xml:lang` はフットノート（`link:footnote` 要素）にのみ設定される（値: `ja`）。従来型 .xbrl では同一 concept・同一 context の日英テキスト Fact の共存はない |
| A-8.3 xml:lang の値の形式 | `ja`（ISO 639-1 の2文字コード）。`ja-JP` や `jpn` は使われない。英語の場合は `en` が使われる（iXBRL の仕様書記載に基づく推定。サンプル .xbrl では `en` の出現なし） |

### 情報源（Fact）

- [F1] サンプルファイル: J-GAAP サンプル `.xbrl` L.4335-4336 — Fact 要素に id 属性付与の実例:
  ```xml
  <jppfs_cor:NotesAndAccountsReceivableTradeAndContractAssets contextRef="Prior1YearInstant" decimals="-6" id="IdFact1758417265" unitRef="JPY">77058000000</jppfs_cor:NotesAndAccountsReceivableTradeAndContractAssets>
  <jppfs_cor:NotesAndAccountsReceivableTradeAndContractAssets contextRef="CurrentYearInstant" decimals="-6" id="IdFact1575656414" unitRef="JPY">76965000000</jppfs_cor:NotesAndAccountsReceivableTradeAndContractAssets>
  ```
- [F2] サンプルファイル: 同上 L.29020-29023 — フットノートリンクが id を参照:
  ```xml
  <link:footnoteLink xlink:role="http://disclosure.edinet-fsa.go.jp/role/jppfs/rol_ConsolidatedBalanceSheet" xlink:type="extended">
    <link:footnote xlink:label="footnote" xlink:role="http://disclosure.edinet-fsa.go.jp/role/jppfs/role/NotesNumber" xlink:type="resource" xml:lang="ja">※１</link:footnote>
    <link:loc xlink:href="#IdFact1758417265" xlink:label="fact" xlink:type="locator"/>
    <link:footnoteArc xlink:arcrole="http://www.xbrl.org/2003/arcrole/fact-footnote" xlink:from="fact" xlink:to="footnote" xlink:type="arc"/>
  </link:footnoteLink>
  ```
- [F3] サンプルファイル: 同上 — `xml:lang` が `link:footnote` 要素にのみ出現し、値は全て `ja`。Fact 要素（`jppfs_cor:*`, `jpcrp_cor:*`, `jpdei_cor:*` 等）には `xml:lang` 属性なし
- [F4] サンプルファイル: 同上 L.3655-3685 — 数値 Fact 要素の属性に `id` がないものが多数（フットノート参照がない Fact には id なし）:
  ```xml
  <jpcrp_cor:NetSalesSummaryOfBusinessResults contextRef="Prior4YearDuration" decimals="-6" unitRef="JPY">231282000000</jpcrp_cor:NetSalesSummaryOfBusinessResults>
  ```
- [F5] サンプルファイル: 同上 L.5130-5155 — テキスト Fact（DEI 要素）にも `xml:lang` なし:
  ```xml
  <jpdei_cor:FilerNameInJapaneseDEI contextRef="FilingDateInstant">Ａ株式会社</jpdei_cor:FilerNameInJapaneseDEI>
  <jpdei_cor:FilerNameInEnglishDEI contextRef="FilingDateInstant">A Corporation</jpdei_cor:FilerNameInEnglishDEI>
  ```
- [F6] パス: `docs/仕様書/XBRL Instance Guideline 報告書インスタンス作成ガイドライン.md` L.1675-1704 — 図表 5-6-6 ix:nonFraction 要素の属性一覧。`id` 属性は記載なし（ix:nonFraction の属性は name, contextRef, unitRef, decimals, scale, format, sign, nil, footnoteRefs）
- [F7] パス: 同上 L.1800-1814 — 図表 5-6-13 ix:nonNumeric 要素の属性一覧。`id` 属性は記載なし（ix:nonNumeric の属性は name, contextRef, escape, format, nil）。`xml:lang` も記載なし

### 推論（Reasoning）

1. [F1]+[F4] より、Fact 要素の `id` 属性はフットノートリンクが参照する Fact にのみ付与される。フットノートの必要がない Fact（経営指標の推移等）には `id` がない。id の形式は `IdFact{数字}` で、数字部分はランダムなハッシュ値のように見える。

2. [F2] より、`link:footnoteLink` → `link:loc` の `xlink:href="#IdFact..."` でFactを参照し、`link:footnoteArc` で Fact と footnote を結ぶ。この構造は XBRL 2.1 の標準的なフットノートリンクの仕組み。

3. [F3] より、`xml:lang` は Fact 要素自体ではなく `link:footnote` 要素に設定される。値は `ja` のみが確認された。

4. [F5] より、日本語名 (`FilerNameInJapaneseDEI`) と英語名 (`FilerNameInEnglishDEI`) は **別の concept** として定義されている（`xml:lang` による言語区別ではない）。したがって同一 concept・同一 context で日英テキスト Fact が共存することは、少なくとも DEI の設計上は想定されていない。

5. [F6]+[F7] より、iXBRL の仕様書でも `id` 属性は ix:nonFraction / ix:nonNumeric の属性として明示されていない。id は EDINET が自動変換時に付与するものであり、提出者が設定するものではない。

6. **パーサーへの影響**: id 属性はオプショナルとして扱う。存在すればフットノートリンクの解決に使用可能。`xml:lang` は Fact 要素のパースでは不要（footnoteLink のパース時にのみ必要）。

### 確信度

- **id 属性**: 高（サンプル .xbrl で実例確認済み。フットノートとの紐付けを検証済み）
- **xml:lang 不使用（Fact 要素）**: 高（サンプル .xbrl の全行を検索し、`xml:lang` が Fact 要素に付いていないことを確認。`link:footnote` にのみ出現）
- **xml:lang の値形式**: 高（`ja` のみ確認。`ja-JP` / `jpn` は未出現）
