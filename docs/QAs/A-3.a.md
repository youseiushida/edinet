## A-3. Fact 要素の構造

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| A-3.1 数値 Fact の属性一覧 | `contextRef`(必須), `unitRef`(必須), `decimals`(必須)。`precision` は不使用。`decimals` と `precision` は XBRL 仕様上排他的だが、EDINET は `decimals` のみ使用 |
| A-3.2 decimals の取りうる値 | `-6`(百万円), `-3`(千円), `-2`, `0`(円), `1`, `2`(X.XX), `3`(X.XXX), `4`(X.XXXX) がサンプルで確認。仕様書では `5`(X.XXX%) も記載。`INF` は未確認だが仕様上は可能 |
| A-3.3 xsi:nil の扱い | nil の場合、要素の内容は空（自己終了タグ `<... xsi:nil="true"/>`）。`unitRef` は残存するが値なし。`decimals` は省略される |
| A-3.4 テキスト Fact | `contextRef`(必須) のみ。`unitRef` は省略。`xml:lang` は付与されない（A-8 参照）。escape 系の属性は従来型 .xbrl には存在しない |
| A-3.5 非数値データ型 | `dateItemType`: 存在する（DEI の日付要素。テキストとして `YYYY-MM-DD` 形式で格納）。`booleanItemType`: 存在する（DEI の `true`/`false` 文字列）。`enumerationItemType`: EDINET では不使用 |
| A-3.6 空文字 Fact | 存在しない（仕様上、値がない場合は `xsi:nil="true"` を使用） |
| A-3.7 符号の扱い | 値自体がマイナス数値になる（例: `-19481000000`）。`balance` 属性はタクソノミの要素定義にあるが Fact 要素の属性ではない |
| A-3.8 数値テキストのフォーマット | 従来型 .xbrl の Fact 値にカンマ・空白は含まれない。純粋な数値文字列（例: `231282000000`, `-19481000000`）。strip やカンマ除去は不要 |
| A-3.9 Fact の出現順序 | 出現順序に仕様上の意味はない。プレゼンテーション順は表示リンクベースで定義される |
| A-3.10 precision vs decimals | EDINET では `precision` は不使用。`decimals` のみ。サンプル全体で `precision` 属性は0件 |
| A-3.11 XBRL値と表示単位 | **円単位で格納**。表示が百万円でも Fact 値は円単位（例: 売上高 231,282百万円 → Fact 値 `231282000000`）。`decimals="-6"` は百万円未満が丸められていることを示す。iXBRL の `scale` 属性で表示値→XBRL 値の変換が行われるが、従来型 .xbrl では変換済みの円単位値が格納される |
| A-3.12 Extensible Enumerations 2.0 | EDINET では不使用。`enum2` 関連の要素はサンプル・タクソノミに見つからない。`AccountingStandardsDEI` は `stringItemType` であり列挙型ではない |

### decimals 分布（J-GAAP サンプル .xbrl の実測）

| decimals 値 | 出現数 | 用途 |
|------------|--------|------|
| `-6` | 984 | 百万円単位（最多、連結財務諸表等） |
| `0` | 225 | 円単位、株数 |
| `2` | 56 | X.XX（小数点2桁、比率等） |
| `3` | 38 | X.XXX（小数点3桁） |
| `4` | 30 | X.XXXX（小数点4桁） |
| `-3` | 17 | 千円単位 |
| `1` | 8 | X.X（小数点1桁） |
| `-2` | 8 | - |
| **合計** | **1,366** | - |

### xsi:nil の実例

```xml
<!-- unitRef は残存するが値なし。decimals は省略 -->
<jpcrp_cor:InterimDividendPaidPerShareSummaryOfBusinessResults contextRef="Prior4YearDuration_NonConsolidatedMember" unitRef="JPYPerShares" xsi:nil="true"/>
<!-- テキスト型の nil -->
<jpdei_cor:FundCodeDEI contextRef="FilingDateInstant" xsi:nil="true"/>
```

### 推論（Reasoning）

1. [F1]+[F9] より、`decimals` は仕様書記載の値（`-6`, `-3`, `0`, `2`, `3`, `4`, `5`）に加え、`1`, `-2` もサンプルで出現する。パーサーでは `decimals` を整数として扱い、特定の値に限定しないべき。`INF` の可能性もあるため文字列としてパースし、`INF` でなければ int 変換する。

2. [F2]+[F10] より、**XBRL の Fact 値は常に円単位**。「百万円表示で売上高 231,282」→ Fact 値は `231282000000`（円）。`decimals="-6"` は「百万円未満が丸められている」ことを示すメタデータ。iXBRL では `scale="6"` で表示値→円単位変換が行われるが、自動生成 `.xbrl` では変換済みの値が格納される。これを誤ると全数値が6桁ずれるため、パーサーの最重要ポイント。

3. [F3]+[F11] より、負の値は Fact のテキスト内容自体がマイナス数値（例: `-19481000000`）。`balance` 属性（debit/credit）はタクソノミの要素定義であり、Fact 要素の属性ではない。パーサーでは値の符号をそのまま解釈すればよい。

4. [F4]+[F12] より、nil Fact は `xsi:nil="true"` + 自己終了タグ。テキスト内容は存在しない。`unitRef` は残存するが `decimals` は省略される。パーサーでは nil Fact の値を `None` として扱う。

5. [F7]+[F13] より、テキスト Fact には `unitRef` がない。`contextRef` のみ必須。TextBlock 型の場合、値に HTML エスケープされたマークアップが含まれる（`&lt;br/&gt;` 等）。

6. [F14]+[F15] より、DEI の日付要素やブーリアン要素はテキストとして格納される。型情報はタクソノミ XSD の `type` 属性から取得する必要がある。しかし v0.1.0 のパーサーでは、値の型変換はタクソノミ参照時に行えばよく、パース段階ではテキストとして保持すれば十分。

7. [F16] より、Extensible Enumerations 2.0 は EDINET で不使用。`AccountingStandardsDEI` は `stringItemType` であり、列挙制約はタクソノミレベルではなくドキュメント（添付5）で規定される。パーサーでの列挙値バリデーションは不要。

8. [F5]+[F6] より、consistent duplicate（同一値の重複タグ付け）は仕様上認められるが、inconsistent duplicate（異なる値）は禁止される。詳細は A-7 参照。

9. **数値フォーマット**: 従来型 `.xbrl` の Fact 値にカンマ・空白は含まれない。iXBRL では `format="ixt:numdotdecimal"` でカンマ区切りされるが、自動生成 `.xbrl` では変換済みの純粋数値。パーサーでの strip/カンマ除去は不要（ただし防御的に実装しても害はない）。

10. **出現順序**: Fact の出現順序に仕様上の意味はない。表示順はプレゼンテーションリンクベースで定義される。パーサーでは出現順を保持する必要はないが、デバッグの便宜のために保持してもよい。
