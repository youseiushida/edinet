## I-1. Role URI と財務諸表の対応

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。

| サブ質問 | 回答 |
|----------|------|
| I-1.1 どの role URI がどの財務諸表に対応するか（完全リスト） | jppfs_rt に 1005 件の roleType が定義。主要財務諸表本表は BS(16), PL(24), CI(10), SS(16), CF(32) の計 98 件（各々 std/non-std のペア）。業種別に異なる URI はなく、業種別は科目一覧ロール（ListOfAccounts）のみ。詳細は推論セクション参照。 |
| I-1.2 非財務セクションの role URI（jpcrp_cor） | jpcrp_rt に 449 件の roleType が定義。有価証券報告書(10), 有価証券届出書(18), 半期報告書(102), 発行登録(10), その他(309) に分類。表紙・経営指標・経理の状況・注記事項等のセクション別ロールを含む。詳細は推論セクション参照。 |
| I-1.3 roleType の定義場所と発見方法 | ロールタイプスキーマ（`*_rt_*.xsd`）内の `<xsd:annotation><xsd:appinfo>` に `<link:roleType>` として定義。提出者も独自の roleType を定義する。 |
| I-1.4 definition 属性と自動マッピング | definition は日本語。パターン `({ELR番号}) {日本語名}` で、「貸借対照表」「損益計算書」「キャッシュ・フロー計算書」等の文字列を含む。これを使って role URI から財務諸表種別への自動マッピングが構築可能。 |
| I-1.5 roleType 要素の完全な属性構造 | `<link:roleType roleURI="..." id="...">` の内部に `<link:definition>` と `<link:usedOn>` を持つ。usedOn の値は7種: `link:presentationLink`, `link:calculationLink`, `link:definitionLink`, `link:footnoteLink`, `link:label`, `link:footnote`, `link:referenceLink`。 |

### 推論（Reasoning）

#### 2. 非財務セクションの role URI（I-1.2）

[F12] より、jpcrp_rt には 449 件の roleType が定義されている。主要な非財務セクションロールの例（`rol_std_*` = EDINET タクソノミ用、対応する提出者用 `rol_*` も存在）:

| role ID | ELR番号 | definition |
|---------|---------|------------|
| `rol_std_CoverPage` | 020000 | 表紙 |
| `rol_std_BusinessResultsOfGroup` | 210000 | 連結経営指標等 |
| `rol_std_BusinessResultsOfReportingCompany` | 211000 | 提出会社の経営指標等 |
| `rol_std_MajorShareholders-01` | 222000a | 大株主の状況 |
| `rol_std_MajorShareholders-02` | 222000b | 大株主の状況 (議決権) |
| `rol_std_FinancialInformation` | 250000 | 経理の状況 |

さらに jpcrp には報告書様式ツリーのロールも含まれる（01____ 番台）:
- 有価証券報告書: 第三号様式(010300), 第四号様式(010400) 等
- 有価証券届出書: 第二号様式(010200), 第七号様式(010700) 等
- 半期報告書: 第四号の三様式(010430), 第五号様式(010500) 等

注記事項関連ロール（4____ 番台）は jpcrp 側の「その他」カテゴリに多数含まれる。セグメント情報等の連番付き分割ロール（`-01` ～ `-09`）も存在する。

**パーサーにとっての実用的な意義**: 財務4表のみを対象とする v0.1.0 では、`3____` 番台の ELR 番号を持つロールのみを処理対象とし、それ以外（`01____`, `02____`, `2____`, `4____` 番台）をスキップすることで、非財務セクションを効率的に除外できる。

#### 3. roleType の定義場所と発見方法（I-1.3）

[F1] + [F11] より、roleType 要素はロールタイプスキーマ（`*_rt_*.xsd`）内の `<xsd:annotation><xsd:appinfo>` 配下に `<link:roleType>` として定義される。具体的なファイルは:
- `jppfs_rt_2025-11-01.xsd`: 財務諸表本表関連の全ロール（1005件）
- `jpcrp_rt_2025-11-01.xsd`: 開示府令関連の全ロール（449件）

#### 4. definition 属性と自動マッピング（I-1.4）

[F1] + [F9] + [F11] より、definition 属性は日本語テキストで、パターンは `({ELR番号}) {日本語名}` である。例: `310010 連結貸借対照表`, `342040 キャッシュ・フロー計算書　間接法`。

自動マッピングは以下のいずれかの方法で構築可能:

**方法 A: definition の日本語キーワードによるマッチング**

[F9] の検証結果より、以下のキーワードマッチが有効:
- `貸借対照表` → BS（39件マッチ、科目一覧含む）
- `損益計算書` → PL（10件マッチ）
- `損益（及び包括利益）計算書` → PL（連結のみ、definition に含まれる）
- `包括利益計算書` → CI（10件マッチ）
- `株主資本等変動計算書` or `純資産変動計算書` or `社員資本等変動計算書` or `投資主資本等変動計算書` → SS
- `キャッシュ・フロー計算書` → CF（54件マッチ、科目一覧含む）

注意: 連結 PL のロール `rol_std_ConsolidatedStatementOfIncome` の definition は「連結損益（及び包括利益）計算書」であり、「包括利益計算書」を含むため CI と誤判定しうる。roleID の `StatementOfIncome` を優先して判定すべきである。

**方法 B: ELR 番号の数値範囲による判定**

[F3] の番号体系を利用し、`3____` 番台の 2～4 桁目で本表種類を判定:
- `100`: BS
- `210`: PL
- `220`: CI
- `300`: SS
- `410`: CF（直接法）
- `420`: CF（間接法）

方法 B のほうが曖昧さが少なく堅牢。ただし ELR 番号は definition テキストの先頭にあるため、正規表現で抽出する必要がある（例: `^\d{6}`）。

**方法 C: roleID のパターンマッチング**

roleID（= role URI の末尾パス）自体に英語の財務諸表名が含まれるため、`BalanceSheet`, `StatementOfIncome`, `StatementOfCashFlows` 等のキーワードでマッチングすることも可能。最も直接的かつ言語非依存な方法。

#### 5. roleType 要素の完全な属性構造（I-1.5）

[F10] + [F11] より、roleType 要素の XML 構造は以下の通り:

```xml
<link:roleType roleURI="{完全URI}" id="{roleID}">
  <link:definition>{日本語テキスト}</link:definition>
  <link:usedOn>{リンクベースタイプ}</link:usedOn>  <!-- 複数可 -->
</link:roleType>
```

属性:
- `roleURI`: 完全な URI 文字列（例: `http://disclosure.edinet-fsa.go.jp/role/jppfs/rol_std_ConsolidatedBalanceSheet`）
- `id`: XML ID（URI の末尾パス部分と同一。例: `rol_std_ConsolidatedBalanceSheet`）

子要素:
- `<link:definition>`: 1つ。日本語のテキスト。パターン `{ELR番号} {日本語名}`（ラベルロール等の一部は ELR 番号なし）
- `<link:usedOn>`: 1つ以上。そのロールが使用可能なリンクベースタイプを指定

usedOn の値一覧（[F10] より、jppfs_rt で確認された全7種）:

| usedOn 値 | 用途 | 設定される roleType の種類 |
|-----------|------|--------------------------|
| `link:presentationLink` | 表示リンク | 財務諸表本表・科目一覧・注記事項等 |
| `link:calculationLink` | 計算リンク | 同上 |
| `link:definitionLink` | 定義リンク | 同上 |
| `link:footnoteLink` | 注記リンク | 同上 |
| `link:label` | 名称リンク | ラベルロール（連結/個別/業種別等） |
| `link:footnote` | 脚注 | 注記番号ロール（`NotesNumber` 等、3件のみ） |
| `link:referenceLink` | 参照リンク | 業種参照ロール（`rol_pe`、1件のみ） |

典型的な財務諸表本表の roleType は `presentationLink`, `calculationLink`, `definitionLink`, `footnoteLink` の 4 つの usedOn を持つ。ラベルロールは `link:label` のみを持つ。
