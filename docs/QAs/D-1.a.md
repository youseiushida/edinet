## D-1. IFRS 適用企業の XBRL

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。

| サブ質問 | 回答 |
|----------|------|
| D-1.1 タクソノミの入手方法 | (c) EDINET が独自に IFRS 拡張タクソノミ（`jpigp_cor`）を提供している。IASB の `ifrs-full` タクソノミは使用されない。`ALL_20251101/taxonomy/jpigp/` 配下にスキーマ・リンクベースが格納されている（例: `jpigp_cor_2025-11-01.xsd`、要素数1803）。提出者の ZIP では提出者別タクソノミが `jpigp_cor` をインポートする形で参照する（例: Sony の XSD は `jpigp_cor_2024-11-01.xsd` をインポート）。 |
| D-1.2 名前空間 | `ifrs-full` ではない。EDINET 独自のプレフィックス `jpigp_cor` が使用される。名前空間 URI は `http://disclosure.edinet-fsa.go.jp/taxonomy/jpigp/{タクソノミ日付}/jpigp_cor`。IASB の `ifrs-full` 名前空間は EDINET では使用されない。 |
| D-1.3 勘定科目の違い | 下記の「主要勘定科目の J-GAAP / IFRS 対応表」を参照。IFRS では `jppfs_cor:NetSales` の代わりに `jpigp_cor:RevenueIFRS`（売上収益）が使用される。要素名には末尾に `IFRS` サフィックスが付く命名規則がある。 |
| D-1.4 経常利益の扱い | IFRS に「経常利益」（OrdinaryIncome）に対応する concept は存在しない。`jpigp_cor` の全1803要素に "Ordinary" を含む要素は皆無。IFRS の P/L 構造では、営業利益（`OperatingProfitLossIFRS`）の下に金融収益（`FinanceIncomeIFRS`）・金融費用（`FinanceCostsIFRS`）等を加減して税引前利益（`ProfitLossBeforeTaxIFRS`）に至る。J-GAAP の「経常利益」に最も近い位置付けは「税引前利益」である。 |
| D-1.5 Context 構造の違い | IFRS 企業でも Context の構造は J-GAAP と同じ。同じ identifier スキーム、同じ Context ID 命名規約（`FilingDateInstant`, `CurrentYearDuration`, `Prior1YearDuration` 等）を使用する。ディメンション軸も `jppfs_cor:ConsolidatedOrNonConsolidatedAxis` 等、J-GAAP と共通の `jppfs_cor` の要素を利用する。 |
| D-1.6 ラベル | IFRS の日本語ラベルは `ALL_20251101/taxonomy/jpigp/{日付}/label/jpigp_{日付}_lab.xml` から取得する。英語ラベルは `jpigp_{日付}_lab-en.xml`。ラベルロールは J-GAAP と同じく開示府令タクソノミ・財務諸表本表タクソノミで用意されるロールを用いる。 |

### 主要勘定科目の J-GAAP / IFRS 対応表

Calculation Linkbase（`jpigp_ac_2025-11-01_cal_pl.xml`）に基づく P/L の主要科目対応:

| J-GAAP (`jppfs_cor`) | J-GAAP ラベル | IFRS (`jpigp_cor`) | IFRS ラベル | 備考 |
|---|---|---|---|---|
| `NetSales` | 売上高 | `RevenueIFRS` | 売上収益 | |
| `CostOfSales` | 売上原価 | `CostOfSalesIFRS` | 売上原価 | |
| `GrossProfit` | 売上総利益 | `GrossProfitIFRS` | 売上総利益 | |
| `OperatingIncome` | 営業利益 | `OperatingProfitLossIFRS` | 営業利益（△損失） | IFRS は損失も含む |
| `OrdinaryIncome` | 経常利益 | （なし） | IFRS に該当なし | D-1.4 参照 |
| （なし） | | `FinanceIncomeIFRS` | 金融収益 | IFRS 固有 |
| （なし） | | `FinanceCostsIFRS` | 金融費用 | IFRS 固有 |
| （なし） | | `ProfitLossBeforeTaxIFRS` | 税引前当期利益 | J-GAAP の経常利益に近い位置 |
| `ProfitLoss` | 当期純利益 | `ProfitLossIFRS` | 当期純利益（△損失） | |

B/S の主要科目:

| J-GAAP (`jppfs_cor`) | J-GAAP ラベル | IFRS (`jpigp_cor`) | IFRS ラベル | 備考 |
|---|---|---|---|---|
| `TotalAssets` | 総資産 | （Presentation に存在） | 資産合計 | Calculation linkbase では直接の対応要素なし |
| `NetAssets` | 純資産 | `EquityIFRS` | 資本 | IFRS では「純資産」ではなく「資本」 |

IFRS P/L の Calculation 階層構造:
```
ProfitLossIFRS（当期純利益）
  └─ ProfitLossBeforeTaxIFRS（税引前当期利益）
       └─ OperatingProfitLossIFRS（営業利益）
            └─ GrossProfitIFRS（売上総利益）
                 ├─ RevenueIFRS（売上収益）  [+]
                 └─ CostOfSalesIFRS（売上原価）  [-]
       ├─ FinanceIncomeIFRS（金融収益）  [+]
       └─ FinanceCostsIFRS（金融費用）  [-]
```

### 推論（Reasoning）

1. **D-1.1 タクソノミの入手方法について**: [F1] より、実際の IFRS 適用企業（Sony）の提出者別タクソノミが `jpigp_cor` スキーマをインポートしていることが確認できる。[F2] より、このスキーマは EDINET タクソノミパッケージ（`ALL_20251101`）にローカルで含まれている。[F3][F4] の仕様書でも「国際会計基準タクソノミ」の使用が明記されている。IASB の `ifrs-full` タクソノミではなく、EDINET が独自に構築した `jpigp` タクソノミファミリーが使用される。したがって選択肢 (c) が正解。

2. **D-1.2 名前空間について**: [F5] の実データと [F6] の仕様書の両方が `jpigp_cor` プレフィックスと `http://disclosure.edinet-fsa.go.jp/taxonomy/jpigp/...` 名前空間 URI を確認している。[F7] より IASB の `ifrs-full` 名前空間は EDINET では使用されない。パーサー実装では `jppfs_cor` の代わりに `jpigp_cor` の名前空間を検出することで IFRS 要素を識別できる。

3. **D-1.3 勘定科目の違いについて**: [F8] より `jpigp_cor` には1119個の concrete 要素があり、J-GAAP の `jppfs_cor` とは独立した要素体系を持つ。[F9] の Calculation Linkbase 解析から P/L 階層の完全な対応関係が判明した。[F13] のラベル情報から日本語名称の対応も確認できた。要素名の命名規則として、J-GAAP の概念名に `IFRS` サフィックスを付加するパターンが見られる（`NetSales` → `RevenueIFRS`、`GrossProfit` → `GrossProfitIFRS` 等）。ただし概念自体が異なるため名称が変わるケースもある（`NetSales`（売上高）→ `RevenueIFRS`（売上収益））。

4. **D-1.4 経常利益の扱いについて**: [F10] より `jpigp_cor` の全要素名に "Ordinary" を含むものが皆無であることが確認された。これは IFRS に「経常利益」の概念が存在しないという会計基準上の事実と整合する。[F9] の P/L 計算構造から、J-GAAP の「経常利益」の位置（営業利益 + 営業外収益 - 営業外費用）に対応するのは IFRS の「税引前利益」（`ProfitLossBeforeTaxIFRS`）であり、その内訳として金融収益（`FinanceIncomeIFRS`）・金融費用（`FinanceCostsIFRS`）が営業利益に加減される構造となっている。パーサー実装では、IFRS 企業に対して「経常利益」フィールドを求めるクエリが来た場合、`ProfitLossBeforeTaxIFRS` で代替するか、該当なしを返す設計判断が必要。

5. **D-1.5 Context 構造の違いについて**: [F11] の実データ（Sony）から、IFRS 企業でも Context の構造・命名規約は J-GAAP と完全に同一であることが確認された。[F12] の仕様書でも、ディメンション軸（連結・単体軸、セグメント軸）は `jppfs_cor`（財務諸表本表タクソノミ）の要素を利用すると明記されている。つまり、Context パーサーは J-GAAP と IFRS で共通のロジックを使用できる。

6. **D-1.6 ラベルについて**: [F13] よりラベルファイルから日本語ラベルの取得が確認された。[F14] の仕様書では、IFRS でも J-GAAP と同じラベルロール（`http://www.xbrl.org/2003/role/label` 等）が使用されると明記されている。パーサー実装では `jpigp` ディレクトリ配下の `label/` フォルダからラベルリンクベースを読み込めばよい。
