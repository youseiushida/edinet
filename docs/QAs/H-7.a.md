## H-7. Table Linkbase（構造化レンダリング）

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| H-7.1 EDINET で Table Linkbase は使われているか | 使われていない |
| H-7.2 使われている場合のプレゼンテーションリンクベースとの関係 | N/A（Table Linkbase 不使用のため） |
| H-7.3 SS（株主資本等変動計算書）の2次元テーブル組み立てに Table Linkbase が必要か | 不要。XBRL Dimensions 1.0 で2次元構造を表現しており、定義リンクベースが列構造を提供する |

### 情報源（Fact）

- [F1] パス: `docs/仕様書/EDINETフレームワーク設計書.md` L.186-201 — 採用する XBRL の技術仕様 7 項目に「Table Linkbase」は含まれていない。採用仕様は XBRL 2.1, Dimensions 1.0, Generic Labels 1.0, Inline XBRL 1.0, FRTA, FRIS, GFM のみ
- [F2] 検索結果: `docs/仕様書/` 配下の全ファイルに対し「Table Linkbase」「テーブルリンクベース」「table linkbase」を検索 — **該当なし**（0 件）
- [F3] パス: `docs/QAs/E-1c.a.md` — SS は XBRL Dimensions 1.0 により2次元構造を表現。ComponentsOfEquityAxis の各メンバーが列、LineItems の concept が行。定義リンクベース（`_def.xml`）が dimension-domain/domain-member の関係を定義し、Table Linkbase は使用されていないことを確認済み

### 推論（Reasoning）

1. [F1] + [F2] より、EDINET の仕様書には Table Linkbase（XBRL Table 1.0 仕様）への言及が一切なく、採用仕様にも含まれていない。よって EDINET では Table Linkbase は使われていない。
2. [F3] より、SS のような2次元テーブルは XBRL Dimensions 1.0（Explicit Dimension + domain-member 関係）で表現されている。プレゼンテーションリンクベースが行構造のツリーを提供し、定義リンクベースが列構造（dimension メンバー一覧）を提供する。
3. パーサー実装への影響: Table Linkbase のパーサーは不要。SS の2次元テーブル組み立てには `_def.xml` から dimension メンバーを取得し、`_pre.xml` から行構造を取得する方式で十分。

### 確信度

- 高（仕様書の採用仕様一覧に不在 + 仕様書全文の否定検索 + E-1c.a.md の実データ検証）
