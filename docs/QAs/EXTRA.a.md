### 推論（Reasoning）

#### 質問1: バリデーション規則一覧

1. [F1] より、EDINET のバリデーション規則は『バリデーションガイドライン』と『バリデーションメッセージ一覧』という2つの独立した文書に集約されている。これらは EDINET 概要書の文書一覧で提出者向け（○）、閲覧者向け（×）と分類されており、FSA が別途公開している文書である。
2. [F2] + [F3] より、当リポジトリの仕様書に散在するバリデーション関連の記述から以下の10カテゴリを特定できる:
   - (1) ファイル形式（指定以外のフォーマットはエラー）
   - (2) フォルダ構成（空フォルダ、必須階層の欠如）
   - (3) ファイル名（使用可能文字、長さ制限）
   - (4) 文字エンコーディング（UTF-8 必須、BOM の問題）
   - (5) HTML 構造（`<head>`/`<body>` 必須、DOCTYPE 重複禁止、セキュリティチェック）
   - (6) PDF 形式（圧縮方式制限、バージョン）
   - (7) リンク参照（壊れた参照、ファイル存在チェック）
   - (8) コンテンツ検証（目次の階層・番号整合性）
   - (9) XBRL 検証（計算リンク整合性、スキーマバリデーション）
   - (10) ファイルシステム（隠しファイル・システムファイルの検出）
3. ただし、これらは仕様書内に点在する記述の集約であり、網羅的な規則一覧は上記2文書を別途入手する必要がある。

#### 質問2: schemaRef 以外の DTS エントリポイント

4. [F4] + [F5] より、報告書インスタンスが参照するのは `schemaRef`（スキーマファイル）と `roleRef`（拡張リンクロール、フットノートリンクのみ）の2つのみ。`linkbaseRef` はインスタンス文書の参照先として定義されていない。
5. [F6] + [F7] より、`link:linkbaseRef` 要素は提出者別タクソノミのスキーマファイル（.xsd）内の `xsd:appinfo` セクションに配置され、インスタンス文書（.htm）には含まれない。サンプルデータでもこのパターンが確認された。
6. [F4] + [F5] + [F6] + [F7] → **結論: EDINET においてインスタンス文書から `link:linkbaseRef` が直接使われるケースはない。** DTS エントリポイントは常に `link:schemaRef` → `.xsd` → `link:linkbaseRef` の経路であり、パーサーは `schemaRef` のみをインスタンスから取得すれば DTS を完全に解決できる。

#### 質問3: xlink:title 属性

7. [F8] + [F9] より、サンプルインスタンス（2026年版）および EDINET タクソノミ（ALL_20251101）の主要モジュール（jppfs, jpcrp）のリンクベースファイルにおいて `xlink:title` は一切使用されていない。
8. [F10] より、タクソノミ差分比較ツールの設定に「title属性を比較する」が有効化されているが、これはツール側の汎用比較設定であり、実データに title 属性が存在することの証拠にはならない。XBRL 2.1 仕様上 `xlink:title` はオプショナル属性であり、EDINET は使用していないと判断できる。
9. [F8] + [F9] → **結論: パーサーで `xlink:title` 属性を処理する必要はない。** EDINET のリンクベースでは `xlink:type`, `xlink:href`, `xlink:label`, `xlink:arcrole`, `xlink:from`, `xlink:to`, `xlink:role` が使用属性の全量であり、`xlink:title` は使用されていない。

### 確信度

- **質問1**: 中 — 仕様書内の散在する記述から10カテゴリを特定したが、網羅的な規則一覧は未入手の別文書（『バリデーションガイドライン』『バリデーションメッセージ一覧』）に依存する
- **質問2**: 高 — 仕様書の明示的な記述（図表 5-3-1）とサンプルデータの両方で一致。`linkbaseRef` がインスタンスから直接使われないことは仕様レベルで確認済み
- **質問3**: 高 — サンプルデータ及び EDINET タクソノミの主要モジュールの全量検索で `xlink:title` の使用が0件。パーサーでの処理は不要と断定できる
- 不確実な点: 質問1について、『バリデーションガイドライン』と『バリデーションメッセージ一覧』は FSA が提出者向けに公開している文書だが、当リポジトリには含まれていない。FSA のウェブサイト（https://www.fsa.go.jp/search/20251111.html 等）で入手できる可能性がある
