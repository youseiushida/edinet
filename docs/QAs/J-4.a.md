## J-4. 連結子会社・関連会社の情報

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| J-4.1 子会社情報は XBRL で構造化されているか | **部分的に構造化されている**。API レベルでは `subsidiaryEdinetCode` で最大10社の EDINET コードが取得可能。タクソノミレベルでは jpcrp_cor に467件の関連 concept が存在するが、concrete 311件のうち200件（64.3%）が `textBlockItemType` であり、真に構造化された数値型は18件のみ。子会社固有の Dimension Axis（例: `SubsidiaryAxis`）は標準タクソノミに存在しない。 |

### 推論（Reasoning）

1. **XBRL における子会社情報の構造化レベル**: 子会社情報は以下の3層で部分的に構造化されている。
   - **API 層**（`subsidiaryEdinetCode`）: [F6] より、EDINET API の書類一覧レスポンスに含まれる `subsidiaryEdinetCode` フィールドで最大10社の子会社 EDINET コードがカンマ区切り文字列として取得可能。これは XBRL タクソノミではなく API メタデータとしての構造化。
   - **タクソノミ層（集約数値）**: [F4] より、`NumberOfConsolidatedSubsidiaries`（連結子会社数）、`NumberOfAssociatesAccountedForUsingEquityMethod`（持分法適用関連会社数）等の整数型 concept、および `EquityInEarningsLossesOfAffiliates`（持分法投資損益）等の金額型 concept が存在し、これらは XBRL Fact として構造化データで取得可能。ただし「連結子会社数: 50」のような集約値であり、個別子会社の情報ではない。
   - **タクソノミ層（テキストブロック）**: [F3] より、concrete 311件のうち200件（64.3%）が `textBlockItemType` であり、連結の範囲、子会社名、関連会社の詳細情報等は HTML テキストブロックとして格納される。個別の企業名・持株比率・所在地等の構造化された項目としてはタグ付けされていない。

2. **Dimension Axis の不在**: [F2] より、jpcrp_cor には子会社を個別に識別するための Dimension Axis（例: `SubsidiaryAxis`）が存在しない。jpcrp_cor の全 **9件** の Axis は、大株主、セグメント、株式分類、発行済株式区分、連番、役員分類、役員区分、GHG 排出量開示基準に関するものであり、子会社・関連会社の個別識別には使えない。【検証時修正: 10件→9件。スクリプトの `"Axis" in name` 検索は `OtherOperatingSegmentsAxisMember` を誤カウントしていた】[F5] の Member 要素も `ConsolidatedAccountingGroupMember`（連結グループ）と `OtherInvesteesExcludedFromConsolidatedAccountingGroupMember`（連結外投資先）の2つのみで、個別子会社のメンバーではない。これは、EDINET タクソノミでは子会社個別の財務データ（売上高、利益等）を XBRL で個別にタグ付けする設計になっていないことを意味する。

3. **実用上の推奨**: 子会社情報の取得には以下のアプローチが適切:
   - **子会社一覧の取得**: API の `subsidiaryEdinetCode` を使用（最大10社のカンマ区切り文字列をパース）。10社を超える子会社を持つ企業では不完全。
   - **子会社数の取得**: `NumberOfConsolidatedSubsidiaries` 等の XBRL Fact から構造化データとして取得可能。
   - **子会社の詳細財務情報**: 各子会社自身が EDINET に提出している Filing を、子会社の EDINET コードで別途検索・取得する。親会社の Filing からは子会社個別の財務データは構造化データとして取得できない。
   - **連結注記からの企業名抽出**: `NotesChangesInScopeOfConsolidationOrEquityMethodTextBlock`、`NumberOfConsolidatedSubsidiariesAndNamesOfMajorConsolidatedSubsidiariesTextBlock` 等の textBlock から企業名を抽出するには HTML テキスト処理（パース・正規表現等）が必要。
   - **人的資本指標（連結子会社別）**: 2023年以降の有報で新設された女性管理職比率・育休取得率・男女賃金格差は percentItemType として構造化されているが、[F2] の Axis 不在により、どの子会社の数値かは `SequentialNumbersAxis`（連番）で区別される可能性がある（要実データ検証）。
