### 推論（Reasoning）

1. [F1] より、IFRS 企業（ソニー）の ZIP ディレクトリ構成は J-GAAP 企業（P-1 のトヨタ/村田）と完全に同一。`XBRL/PublicDoc/` 配下に `.xbrl` ファイルが存在するため、`find_primary_xbrl_path` は IFRS 企業でも修正なしで動作する。v0.1.0 の互換性問題はない。

2. [F2] より、IFRS 企業は `ifrs-full`（IFRS Foundation 提供）ではなく `jpigp_cor`（EDINET 提供）名前空間を使用する。これは EDINET が IFRS の概念を独自に再定義した日本版 IFRS タクソノミである。

3. [F2] + [F3] より、Context 構造（`FilingDateInstant`, `CurrentYearDuration` 等）は J-GAAP と同一。ディメンションも `jppfs_cor` を共用している。したがって、Context 解析ロジックは J-GAAP / IFRS で共通化可能。

4. [F4] + [F5] より、ソニー（E01777）もトヨタ（E02144）も同じ `jpigp_cor_2024-11-01.xsd` を import しており、IFRS 企業に共通のパターンであることが確認できる。DTS 解決時に `jpigp` import が見つかった場合は `ALL_20251101/taxonomy/jpigp/` 配下のローカルファイルに解決すればよい。

5. [F3] + [F4] より、D-1.1 の IFRS タクソノミ入手方法は **(c) EDINET が独自に IFRS 拡張タクソノミを提供している** が正解。提出者 XSD の import 先が `jpigp_cor_2024-11-01.xsd` であり、これは ALL_20251101 タクソノミパッケージ内の `taxonomy/jpigp/` に含まれる。IFRS Foundation サイトからの別途ダウンロードは不要。

6. [F3] より、`AccountingStandardsDEI = "IFRS"` で会計基準の判別が可能（D-3 の結論と整合）。この値により、パーサーが IFRS 固有の処理パス（`jpigp_cor` 名前空間の勘定科目解決等）に分岐できる。
