## I-2. Presentation ツリーの構造

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| I-2.1 科目の階層関係（from/to で親子か） | はい。`presentationArc` の `xlink:from`/`xlink:to` が親子関係を表す。arcrole は `http://www.xbrl.org/2003/arcrole/parent-child` のみ。ルート要素 `{諸表名}Heading` → `Table`/`LineItems` → `Abstract` → 具体的科目、という多段階のツリーを構成する。 |
| I-2.2 order 属性の存在とデフォルト | order 属性は**必ず設定する**（設定規約書 L.1797 に明記）。0以上の半角数値（小数可）。同一親の兄弟要素間で一意。デフォルト順への fallback は不要（値が必ず存在する）。実ファイルでは `1.0`, `2.0` 等の整数値が使用されていた。 |
| I-2.3 ツリー深さとインデント | ツリーの深さがインデントレベルに対応する。ただし dimension 関連の中間ノード（`Table`, `Axis`, `Member`, `LineItems`）はインデント計算からスキップすべき。実ファイルでは最大深さ 5（ルート含む 6 階層）で、科目の実質インデントは 0〜3 程度。 |
| I-2.4 合計行の識別 | `preferredLabel="http://www.xbrl.org/2003/role/totalLabel"` で識別可能。連結 BS では 12 か所で `totalLabel` が使用されていた（`CurrentAssets`, `PropertyPlantAndEquipment`, `IntangibleAssets`, `InvestmentsAndOtherAssets`, `NoncurrentAssets`, `Assets`, `CurrentLiabilities`, `NoncurrentLiabilities`, `Liabilities`, `ShareholdersEquity`, `NetAssets`, `LiabilitiesAndNetAssets`）。また連結用ラベル `http://disclosure.edinet-fsa.go.jp/jppfs/Consolidated/role/label` が `preferredLabel` に設定されるケースもある。 |

### 推論（Reasoning）

1. **I-2.1 科目の階層関係**: [F1] により、表示リンクのアークロールは `parent-child` の1種類のみであり、`presentationArc` の `from` が親、`to` が子を表す。[F4] の実ファイル解析で、`ConsolidatedBalanceSheetHeading` をルートとする完全なツリーが構築できることを確認した。ツリーの構造は `Heading` > `Table`（ディメンション軸）+ `LineItems`（科目本体）> `Abstract`（見出し）> 具体的科目、という多段階になっている。

2. **I-2.2 order 属性**: [F1] が「必ず設定する」と明記し、[F2] が「親の報告項目が同一である場合、order 属性は一意」と補足する。[F4] の実ファイルでは全 61 アークに order が設定されており、order なしのアークは 0 件であった。値は `1.0`, `2.0`, ... のように整数の 10 進数表記が使用されていた。したがってパーサーは order 属性のデフォルト値処理を実装する必要はない。

3. **I-2.3 ツリー深さとインデント**: [F3] により、表示リンクの項目順序は原則として表示上の並び順と整合するが、意味的な親子関係が優先される場合がある。[F4] の深さ分析では最大深さ 5 だが、これにはルートの `Heading`（深さ 0）、dimension 中間ノード（`Table`/`LineItems` が深さ 1、`Axis`/`Member` が深さ 2）が含まれる。実際の財務諸表のインデントに対応するのは `LineItems` 以下の部分であり、`AssetsAbstract`（深さ 2 → 実質インデント 0）、`CurrentAssetsAbstract`（深さ 3 → 実質インデント 1）、`CashAndDeposits`（深さ 4 → 実質インデント 2）のように、dimension 中間ノードをスキップしてインデントを計算する必要がある。

4. **I-2.4 合計行の識別**: [F1] + [F4] より、合計行には `preferredLabel` 属性に `http://www.xbrl.org/2003/role/totalLabel` が設定される。[F4] の実ファイルでは連結 BS で 12 か所に `totalLabel` が使用されており、`CurrentAssets`, `PropertyPlantAndEquipment`, `IntangibleAssets`, `InvestmentsAndOtherAssets`, `NoncurrentAssets`, `Assets`, `CurrentLiabilities`, `NoncurrentLiabilities`, `Liabilities`, `ShareholdersEquity`, `NetAssets`, `LiabilitiesAndNetAssets` がそれぞれ合計行として識別される。さらに連結用ラベル（`http://disclosure.edinet-fsa.go.jp/jppfs/Consolidated/role/label`）も `preferredLabel` に設定され、この場合は連結用の名称（例:「連結資産合計」等）に切り替わる。
