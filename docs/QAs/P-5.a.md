### CSV ヘッダー（9列）

```
"要素ID"	"項目名"	"コンテキストID"	"相対年度"	"連結・個別"	"期間・時点"	"ユニットID"	"単位"	"値"
```

| # | 列名 | XBRL 対応 | 説明 |
|---|------|-----------|------|
| 1 | 要素ID | Concept QName | 名前空間プレフィックス付き（例: `jppfs_cor:Assets`） |
| 2 | 項目名 | Label（日本語） | タクソノミ定義のラベル |
| 3 | コンテキストID | Context ID | ディメンション情報を含む文字列 |
| 4 | 相対年度 | Period の相対表現 | 「前期末」「当期末」「前期」「当期」等 |
| 5 | 連結・個別 | Dimension member | 「個別」「連結」「その他」 |
| 6 | 期間・時点 | Period type | 「時点」（instant）/「期間」（duration） |
| 7 | ユニットID | Unit ID | `JPY`, `pure`, `－` 等 |
| 8 | 単位 | Unit label | 「円」「」「－」等 |
| 9 | 値 | Fact value | 数値またはテキスト |

### 推論（Reasoning）

1. [F2] + [F3] より、CSV と呼ばれているが実態は **TSV (Tab-Separated Values)** であり、UTF-16 LE エンコーディング。Python で読む際は `encoding="utf-16"` を指定する必要がある。
2. [F4] より、CSV ファイル名は XBRL ファイル名と対応しており、AuditDoc と PublicDoc の両方について CSV が生成される。AuditDoc の CSV は TextBlock Fact 1件のみを含む。
3. 列構成から、CSV は XBRL Fact の完全なフラット表現である。Presentation/Calculation/Definition リンクベースは含まれない。→ K-3.a.md の結論を裏付ける。
