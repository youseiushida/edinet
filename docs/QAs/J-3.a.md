### 推論（Reasoning）

#### 1. 注記テキストは textBlockItemType の Fact として XBRL に含まれる（J-3.1）

[F1] + [F4] + [F7] + [F8] + [F10] より、「重要な会計方針」「関連当事者取引」等の注記テキストは XBRL インスタンスの中に textBlockItemType の Fact として含まれる。具体的には:

- **格納形式**: HTML テキスト。`<table>`, `<p>`, `<br>` 等の HTML タグを含み、表形式のデータ（注記テーブル）や段落テキストがそのまま Fact の値として格納される。
- **periodType**: instant 352件、duration 337件（[F11]）。表紙情報・経営指標等は instant、会計期間全体に関する注記は duration で報告される。【検証時修正: 元の回答では「全て duration」としていたが誤り】
- **context**: 標準の `CurrentYearDuration` 等が使用される。連結/個別の区分は context の dimension で表現される。
- **1 Fact = 1 注記セクション**: 各 textBlock concept は1つの注記セクション全体（例: 「重要な会計方針」セクション全体）を1つの HTML テキストとして格納する。注記内の個別項目が別々の Fact に分かれるわけではない（包括タグ付け方式）。

[F9] より、附属明細表も同様に textBlockItemType で格納されるが、内部の個別数値は XBRL Fact として分離されていない。数値データの構造化抽出には HTML パースが必要である。

#### 2. 注記 concept の名前空間分布（J-3.2）

[F1] + [F2] + [F3] より、textBlockItemType の concept は **jpcrp_cor に集中** している。

**jpcrp_cor（689 件）**: 有報（開示府令）の注記事項の大部分を定義する。具体的には:
- 「重要な会計方針」に関する textBlock（`NotesSignificantAccountingPoliciesTextBlock` 等）
- 「関連当事者取引」に関する textBlock（`NotesRelatedPartyTransactionsTextBlock` 等）
- 「セグメント情報」に関する textBlock（`NotesSegmentInformationTextBlock` 等）
- 「事業等のリスク」（`BusinessRisksTextBlock`）
- 「経営者による財政状態、経営成績及びキャッシュ・フローの状況の分析」
- 附属明細表の各テーブル（[F9]）
- その他有報固有の記述セクション

**jppfs_cor（0件）**: 財務諸表タクソノミ（jppfs_cor）には textBlockItemType の concept は **存在しない**。jppfs_cor は財務諸表本表の勘定科目（monetaryItemType, stringItemType 等）のみを定義する。注記事項は全て jpcrp_cor 側で定義される。【検証時修正: 元の回答では「少数の textBlock concept が存在する」としていたが、スクリプト再実行で 0件と確認】

**提出者拡張**: 提出者は独自の textBlock concept を拡張タクソノミで定義しうる。提出者固有の注記事項や、標準タクソノミでカバーされない追加的な注記を記述するために使用される。

#### 3. 注記の Role URI（ELR 番号体系）

[F5] + [F6] より、注記事項は以下の ELR 番号体系に属する。

| ELR 番号帯 | 内容 | 備考 |
|-----------|------|------|
| `4____` | 日本基準の財務諸表注記事項 | jppfs/jpcrp の注記 role |
| `6____` | IFRS の財務諸表注記事項 | IFRS 注記 role |

jpcrp_rt の「その他」カテゴリ（309件）に注記関連のロールが多数含まれており、セグメント情報等の複雑な注記は連番付き分割ロール（`-01` 〜 `-09`）で複数のサブセクションに分かれている。

#### 4. 実用上の留意点

textBlockItemType の Fact はその名の通り「テキストブロック」であり、HTML 形式のナラティブ記述を丸ごと格納するものである。したがって:

- **構造化データとしての限界**: textBlock 内の個別数値（例: 減損損失の金額、退職給付債務の残高等）は XBRL の個別 Fact としてタグ付けされていない。これらを抽出するには HTML パース（テーブル構造の解析）が必要。
- **v0.1.0 での取扱い**: 財務4表（BS/PL/CI/SS/CF）の数値 Fact が主目的であり、textBlock の HTML パースは対象外。注記テキストの活用は将来の拡張課題とする。
- **サイズ**: textBlock Fact は1件あたり数千〜数万文字になることがあり、データ量の観点でも注意が必要。
