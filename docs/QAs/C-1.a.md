### 推論（Reasoning）

1. [F1] より、EDINET タクソノミは大きく4つのカテゴリに分かれる:（1）内閣府令タクソノミ（jpcrp, jpcrp-esr, jpcrp-sbr, jpctl, jpsps, jpsps-esr, jpsps-sbr, jplvh, jptoi, jptoo-*）、（2）財務諸表本表タクソノミ（jppfs）、（3）国際会計基準タクソノミ（jpigp）、（4）DEI タクソノミ（jpdei）。common は独立した基盤モジュール。

2. [F2] より、各内閣府令タクソノミの分割単位と対応する書類種別・様式番号が明確に定義されている。jpcrp は開示府令の大半（有報・届出書・半期報告書・発行登録書等）を網羅し、jpcrp-esr（臨時報告書）と jpcrp-sbr（自己株券買付状況報告書）のみ別分割。同様に jpsps（特定有価証券の大半）から jpsps-esr, jpsps-sbr が分離。他社株買付府令は様式ごとに5分割（jptoo-ton/pst/wto/tor/toa）。

3. [F2] + [F5] より、doc_type_code との対応を導出。ただし doc_type_code は書類種別（有報=120、半期=160 等）であり、タクソノミモジュールは府令+様式の粒度で分割されているため、1対1対応ではない。例えば jpcrp は doc_type_code 030, 080, 100, 120, 160 等の複数の書類種別に対応する。jptoi と jptoo-ton は同じ doc_type_code 240（公開買付届出書）だが、発行者/発行者以外で使用するモジュールが異なる。

4. [F3] + [F4] より、common モジュールは identificationAndOrdering スキーマのみを含み、要素定義（xsd:element）は0件。これは目次項目（identifierItem）の substitutionGroup を定義する基盤であり、他モジュールの目次項目要素がこの substitutionGroup を参照して宣言される。

5. [F4] より、jppfs は r/ ディレクトリに23業種のサブディレクトリを持ち、業種別にプレゼンテーション・定義・計算リンクが分かれている。jpigp は r/ に57ファイルを持つが業種サブディレクトリはない（IFRS は業種区分が異なるため）。

6. [F4] より、モジュールのバージョン日付は統一されていない。jpcrp, jpigp, jplvh, jppfs, jpsps, jptoi, jptoo-pst, jptoo-ton は 2025-11-01（最新）だが、jpcrp-esr は 2024-11-01、jpcrp-sbr/jpctl/jpsps-esr/jpsps-sbr は 2021-11-01、jpdei/common/jptoo-toa/jptoo-tor/jptoo-wto は 2013-08-31 と古い。これはタクソノミの更新が分割単位ごとに行われるため（[F2] L.821「タクソノミの更新は、タクソノミの分割単位ごとに行われ」）。

### 確信度

- 高（仕様書に明記 + スクリプト実行で実ファイルと突合済み）
- 補足: doc_type_code との対応は、仕様書の「分割単位表」と「書類種別コード表」の突合による推定。jptoi と jptoo-* が同じ doc_type_code を共有する箇所の区別は、府令コード（formCode）で行われると推測されるが、EDINET API 仕様書に府令コードとタクソノミモジュールの直接対応表は見つからなかった。
