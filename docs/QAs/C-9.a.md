### 推論（Reasoning）

#### C-9.1 periodType: instant vs duration

[F3] + [F11] より、periodType は以下のルールで設定される:

- **instant**: ある一時点において事実となり得る概念（ストック概念）。BS 科目（資産・負債・純資産の残高）が典型。
- **duration**: ある一時点において事実とならない概念（フロー概念）。PL/CF 科目が典型。また、Abstract/Table/Axis/Member/LineItems には一律 duration を設定。判断根拠がない場合のデフォルトも duration。

[F15] の実データ:
- instant=3,269 (33.7%)、duration=6,428 (66.3%)
- duration が多い理由は、PL/CF 科目に加えて Abstract 要素（2,739件）が全て duration であるため

「BS=instant, PL/CF=duration」は大筋正しいが、正確には periodType は「残高（ストック）か増減（フロー）か」で決まる。PL 科目のうち一部に instant のものが存在し得るが、実際のデータでは PL 科目は全て duration、BS 科目は全て instant。CF 科目も全て duration だが balance 属性は設定されない（[F12]）。

#### C-9.2 balance: debit/credit と符号

[F4] + [F12] より:
- balance は monetaryItemType の要素にのみ設定される
- 資産・費用 → debit（借方）、負債・純資産・収益 → credit（貸方）
- CF 科目は貸借が特定できないため設定なし

[F15] の実データ: 全 4,533 monetaryItemType のうち:
- debit=2,084、credit=1,433、設定なし=1,016（主に CF 科目）
- jppfs モジュール: 3,763 monetary のうち balance あり=3,035、balance なし=728（CF 科目）

**符号への影響**: balance 属性そのものは XBRL インスタンス内の Fact 値の符号を直接変更しない。Fact 値は常に正の数で報告される（debit 側の科目は正の値、credit 側も正の値）。ただし、表示時に Presentation リンクの `preferredLabel` が `negatedLabel` の場合は符号を反転して表示する必要がある。balance 属性は計算リンクにおける weight の妥当性検証や、異なる balance を持つ科目間の整合性チェックに使用される。

#### C-9.3 abstract: 見出し専用の識別

[F5] + [F17] より:
- abstract=true の要素はタイトル項目（名前末尾 `Abstract`）、目次項目（末尾 `Heading`）、Table、Axis、Member、LineItems
- abstract=true の要素は Fact を持てない（XBRL 仕様上、abstract 要素に対する Fact は不正）

[F15] の実データ: type x abstract のクロス集計で:
- stringItemType + abstract=true = 2,253（タイトル項目・目次項目・LineItems）
- domainItemType + abstract=true = 486（全ドメイン/メンバーが abstract）
- monetaryItemType は全て abstract=false（具体的な金額要素）
- textBlockItemType は全て abstract=false（テキストブロックは具体的な要素）

パーサー実装では、abstract=true の要素を Fact マッチングの対象から除外することで、不正な Fact の検出に利用できる。

#### C-9.4 nillable: 実質的に常に true

[F6] + [F9] + [F10] より:
- FRTA 2.1.6 に従い、原則として全要素の nillable は true
- 例外はパート要素（参照リンク用の特殊要素）のみ false だが、これは _cor_ ファイルではなく _pe_ ファイルに定義される

[F15] の実データ: 全 9,697 要素が nillable=true（例外なし）。廃止要素（_dep_）1,430 件も全て nillable=true。

パーサーは nillable を属性として保持すべきだが、実質的に常に true と仮定できるため、値が false の場合のみ警告を出す程度でよい。

#### C-9.5 substitutionGroup: tuple は存在しない

[F2] + [F8] より:
- EDINET は明示的に「タプルは使用しない」と宣言
- substitutionGroup の設定値は xbrli:item、xbrldt:hypercubeItem、xbrldt:dimensionItem、iod:identifierItem の4種のみ

[F15] + [F16] の実データ:

| substitutionGroup | 件数 | 用途 |
|---|---|---|
| xbrli:item | 8,000 | 通常の勘定科目、テキストブロック、メンバー等 |
| iod:identifierItem | 1,547 | 目次項目（EDINET 固有の identifierItem） |
| xbrldt:hypercubeItem | 136 | XBRL Dimensions の表（Table） |
| xbrldt:dimensionItem | 14 | XBRL Dimensions の軸（Axis） |

パーサーは tuple 処理を実装する必要がない。ただし、提出者拡張タクソノミでも tuple が使用されないことは仕様書の記述から確認できる（ガイドラインの substitutionGroup 選択肢に tuple がない）。

#### C-9.6 type: 13種のデータ型

[F7] の仕様書定義と [F15] の実データが完全に一致:

| type | 件数 | 概要 |
|---|---|---|
| xbrli:monetaryItemType | 4,533 | 金額（全体の46.7%、最多） |
| xbrli:stringItemType | 2,686 | 文字列（Abstract要素含む） |
| nonnum:textBlockItemType | 1,693 | テキストブロック（HTML含む記載事項） |
| nonnum:domainItemType | 486 | ドメイン/メンバー |
| xbrli:sharesItemType | 85 | 株数 |
| xbrli:nonNegativeIntegerItemType | 71 | 0以上の整数 |
| num:percentItemType | 56 | 割合(%) |
| xbrli:dateItemType | 43 | 日付 |
| num:perShareItemType | 22 | 一株当たり金額 |
| xbrli:decimalItemType | 11 | 小数 |
| xbrli:booleanItemType | 6 | true/false |
| dtr-types:ghgEmissionsItemType | 5 | 温室効果ガス排出量 |

パーサーの値変換では、type に応じて:
- monetaryItemType/sharesItemType/perShareItemType/percentItemType/decimalItemType/pureItemType/ghgEmissionsItemType → 数値としてパース（decimals/precision 属性で精度管理）
- stringItemType → プレーンテキスト
- textBlockItemType → HTML を含む可能性あり（iXBRL の場合はエスケープされた HTML）
- dateItemType → 日付文字列
- booleanItemType → true/false
- nonNegativeIntegerItemType → 非負整数
- domainItemType → ディメンションメンバーの識別子（通常 Fact の値としては出現しない）

#### C-9.7 パーサー実装への影響まとめ

| 属性 | パーサーへの影響 | 重要度 |
|---|---|---|
| periodType | Context の period 型と一致する必要がある。instant 要素に duration の Context を指定した Fact は不正。**致命的エラーとして扱うべき**。 | 高 |
| balance | 符号の解釈に影響。Presentation の preferredLabel が negatedLabel の場合に符号反転が必要。計算リンクの weight 検証にも使用。表示ロジックに影響するが、値のパース自体には影響しない。 | 中 |
| abstract | abstract=true の要素に対する Fact は不正。Fact 収集時のフィルタリングに使用。 | 高 |
| nillable | xsi:nil="true" の Fact を許容するか。実質全要素が true のため、常に許容と仮定してよい。 | 低 |
| substitutionGroup | tuple が存在しないため、item ベースのフラットな Fact モデルのみ実装すればよい。hypercubeItem/dimensionItem はディメンション処理で区別。identifierItem は目次項目として abstract=true と同様に Fact 対象外。 | 高 |
| type | 値の変換ロジックに直結。monetaryItemType は unit (通貨) と decimals を伴う。textBlockItemType は HTML コンテンツを含む可能性あり。 | 高 |

#### C-9.8 concept ローカル名のキャラクタセット

[F13] + [F14] より:
- 要素名は「{文字列(英数字)}」と規定され、LC3 (Label Camel Case Concatenation) 法で命名
- 英単語の先頭を大文字、2文字目以降を小文字として結合

[F15] の実データ:
- 使用文字: A-Y（大文字25文字、Z は未使用）、a-z（小文字26文字）、0-9（数字10文字）
- 日本語・全角文字・記号・アンダースコア・ハイフンは一切含まれない
- 名前長: 最短4文字、最長263文字、平均47.7文字

パーサー実装では:
- concept 名を辞書キーとして使用する場合、ASCII 英数字のみのため安全（エンコーディング問題なし）
- ファイル名に concept 名を使用する場合、263文字の名前が存在するため、ファイルシステムの制約（Windows の MAX_PATH=260）に注意が必要
