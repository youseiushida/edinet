### 推論（Reasoning）

1. **C-8.1 ハイパーキューブ構造**: [F1] + [F2] + [F8] より、定義リンクのハイパーキューブは以下の階層で構成される:
   - **ルート要素（Heading）** → **Table**（`all` arcrole, `closed="true"`, `contextElement="scenario"`）
   - **Table** → **Axis**（`hypercube-dimension` arcrole）
   - **Axis** → **Domain**（`dimension-domain` arcrole）
   - **Axis** → **Default Member**（`dimension-default` arcrole）
   - **Domain** → **Member**（`domain-member` arcrole）
   - 同時に **ルート要素** → **LineItems** → **科目**（`domain-member` arcrole）で科目の表示項目も接続される。

   [F5] の「ミラー」により、この構造は表示リンクの `parent-child` ツリーと対応する。定義リンクでは arcrole で関係の種類が区別されるが、表示リンクでは全て `parent-child` で統一されている。

2. **C-8.2 主要ディメンション**: [F7] より、EDINET で最も重要なディメンションは `ConsolidatedOrNonConsolidatedAxis`（連結/個別の区別、30ロール）であり、財務諸表本表と注記事項の大半のロールで使用される。デフォルトメンバーが `ConsolidatedMember` であるため、Context の segment にディメンション指定がない場合は「連結」として扱われる。`NonConsolidatedMember` が明示的に指定された場合のみ「個別」となる。2番目に多い `SequentialNumbersAxis`（11ロール）は表形式データの行番号に使用され、Row1Member〜Row63Member のドメインを持つ。その他の主要 Axis は `OperatingSegmentsAxis`（セグメント情報、7ロール）と `ComponentsOfEquityAxis`（SS の純資産内訳列、Domain=NetAssetsMember）である。

3. **C-8.3 v0.1.0 での必要性**: 連結/個別の判別は Context の `segment` 要素内の `xbrldi:explicitMember dimension="jppfs_cor:ConsolidatedOrNonConsolidatedAxis"` を読めば可能であり、定義リンクの解析は不要。ただし定義リンクには以下の有用な情報が含まれる: (a) 有効な Axis/Member の組み合わせ（`closed="true"` による制約）, (b) デフォルトメンバー（segment にディメンション指定がない場合の既定値）, (c) 科目一覧ツリー（拡張要素の位置付け）。v0.1.0 ではこれらを必須としないが、データ品質検証の際に参照する価値がある。

4. **C-8.4 arcrole 一覧**: [F1] + [F6] より、EDINET の定義リンクで使用される arcrole は 6 種類:

   | arcrole | 基底 URI | 件数 | 用途 |
   |---------|----------|------|------|
   | `general-special` | `http://www.xbrl.org/2003/arcrole/` | 7 | 科目一覧ツリー |
   | `all` | `http://xbrl.org/int/dim/arcrole/` | 50 | ルート要素 → Table |
   | `hypercube-dimension` | `http://xbrl.org/int/dim/arcrole/` | 58 | Table → Axis |
   | `dimension-domain` | `http://xbrl.org/int/dim/arcrole/` | 133 | Axis → Domain |
   | `dimension-default` | `http://xbrl.org/int/dim/arcrole/` | 38 | デフォルトメンバー |
   | `domain-member` | `http://xbrl.org/int/dim/arcrole/` | 654 | メンバー間・表示項目間 |

   `notAll` は [F6] の実ファイルで 0 件であり、EDINET では使用されていない。`notAll` はハイパーキューブの「除外」を定義するアークロールだが、EDINET のディメンション設計では除外ではなく包含（`all`）のみを使用する。
