### 提出者別名前空間の URI パターンと検出ルール

**URI パターン**:

```
http://disclosure.edinet-fsa.go.jp/{form_code}/{report_abbr}/{seq}/{edinet_code}-{branch}/{period_end}/{submission_num}/{filing_date}
```

**具体例**:
- J-GAAP サンプル: `http://disclosure.edinet-fsa.go.jp/jpcrp030000/asr/001/X99001-000/2026-03-31/01/2026-06-12`
- IFRS サンプル: `http://disclosure.edinet-fsa.go.jp/jpcrp030000/asr/001/X99002-000/2026-03-31/01/2026-06-12`
- トヨタ実データ: `http://disclosure.edinet-fsa.go.jp/jpcrp030000/asr/001/E02144-000/2025-03-31/01/2025-06-18`

**プレフィックス命名規約** (提出者別タクソノミ作成ガイドライン 図表 4-5-2):

```
jp{府令略号}{様式番号}-{報告書略号}_{EDINETコード}-{追番}
```

例: `jpcrp030000-asr_X99001-000`, `jpcrp030000-asr_E02144-000`

**提出者名前空間の判定ルール**:
1. 標準タクソノミの名前空間 URI: `http://disclosure.edinet-fsa.go.jp/taxonomy/` で始まる
2. 提出者別の名前空間 URI: `http://disclosure.edinet-fsa.go.jp/` で始まるが `/taxonomy/` を含まない
3. その他（XBRL 標準、XLink 等）: `http://www.xbrl.org/` 等で始まる

したがって `http://disclosure.edinet-fsa.go.jp/` で始まり `/taxonomy/` を含まないものが提出者別名前空間である。URI 内に EDINET コード（例: `E02144` や `X99001`）が含まれることも確認できる。

### 推論（Reasoning）

1. [F1]+[F4] より、J-GAAP の有報 `.xbrl` には 10 個の名前空間が宣言される: xbrli, xlink, link, xsi, iso4217, jpcrp_cor, jppfs_cor, jpdei_cor, xbrldi, および提出者別。仕様書の図表 5-2-1 に記載のある num, nonnum, xbrldt, dtr-types, ix, ixt は `.xbrl`（従来型 XBRL）では使用されないため宣言されていない（[F12] の「不要な名前空間は設定しません」に従う）。

2. [F5] より、IFRS の有報 `.xbrl` では上記に加えて `jpigp_cor` が追加される。[F9] より `jppfs_cor` は IFRS でも残存する（ディメンション要素の共用のため）。[F10] より IASB の `ifrs-full` は使われない。

3. [F2]+[F3]+[F7] より、提出者別名前空間の URI は `http://disclosure.edinet-fsa.go.jp/{form}/{report}/{seq}/{edinet_code}/{period_end}/{sub_num}/{filing_date}` パターンで決定論的に構築される。URI 内に EDINET コード（例: `E02144-000`）が含まれるため、URI 文字列から提出者を識別可能。

4. [F4] vs [F1] の差異分析: 仕様書 図表 5-2-1 は iXBRL 向けの名前空間一覧（ix, ixt を含む）であり、自動生成 `.xbrl` ではこれらが省略される。`.xbrl` パーサーとしては iXBRL 専用名前空間を無視してよい。

5. [F11] より、US-GAAP 適用企業の XBRL には専用の名前空間プレフィックスがない。包括タグ付けのみが行われ、財務諸表の詳細タグは設定されない。

6. **提出者名前空間の判定ルール**: 名前空間 URI が `http://disclosure.edinet-fsa.go.jp/` で始まり、パスに `/taxonomy/` を含まないものが提出者別名前空間である。
