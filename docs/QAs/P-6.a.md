#### 業種ディレクトリ内のファイル構造（例: `r/cna/`）

[F8] より、各業種ディレクトリ内のファイル命名パターン:

```
jppfs_{業種}_{連結区分}_{日付}_{リンク種別}_{財務諸表種別}.xml
```

- **業種**: `cna`, `bk1`, etc.
- **連結区分**: `ac`（連結）, `an`（個別）, `cm`（比較情報-連結）, `sc-t1`/`sc-t2`（株主資本等変動計算書-連結）, `sn-t1`/`sn-t2`（株主資本等変動計算書-個別）
- **リンク種別**: `cal`（計算）, `def`（定義）, `pre`（表示）
- **財務諸表種別の例**:
  - `bs` — 貸借対照表（基本）
  - `bs-1-BS-01-CA-Doubtful-1-ByAccount` — BS の貸倒引当金個別表示バリエーション
  - `bs-1-BS-03-PPE-Dep-1-ByAccount` — BS の有形固定資産減価償却個別表示
  - `bs-1-BS-04-PPE-Imp-1-ByAccount` — BS の有形固定資産減損個別表示
  - `bs-1-BS-05-PPE-DepImp-1-ByAccount` — BS の有形固定資産減価償却＋減損
  - `bs-1-BS-08-IOA-Doubtful-1-ByAccount` — BS の投資その他資産貸倒引当金
  - `pl` — 損益計算書（基本）
  - `pl-2-PL-09-CI-1-SingleStatementNetOfTax` — 包括利益一計算書方式（税効果後）
  - `pl-2-PL-09-CI-2-SingleStatementBeforeTax` — 包括利益一計算書方式（税効果前）
  - `cf-di-3-CF-01-Method-Direct` — CF 直接法
  - `cf-in-3-CF-03-Method-Indirect` — CF 間接法
  - `ss` — 株主資本等変動計算書

**→ C-3 への示唆**: 23業種コードが判明（bk1, bk2, cai, cmd, cna, cns, edu, elc, ele, fnd, gas, hwy, in1, in2, inv, ivt, lea, liq, med, rwy, sec, spf, wat）

**→ C-4 への示唆**: サフィックス体系が判明。連結区分（ac/an/cm/sc-t1/sc-t2/sn-t1/sn-t2）× リンク種別（cal/def/pre）× 財務諸表種別（bs/pl/cf/ss + バリエーション）

#### C-5 向け: ラベル構造のまとめ

[F3][F4] より:

- **構造**: `link:loc`（concept参照） → `link:labelArc`（arcrole=`concept-label`） → `link:label`（ラベル本体）
- **ラベルの属性**: `xlink:role`（ラベルロール）, `xml:lang`（言語）, `id`
- **主要ラベルロール**:
  - `http://www.xbrl.org/2003/role/label` — 標準ラベル
  - `http://www.xbrl.org/2003/role/verboseLabel` — 冗長ラベル（例: `資産の部 [タイトル項目]`）
  - `http://www.xbrl.org/2003/role/totalLabel` — 合計ラベル
  - `http://www.xbrl.org/2003/role/negatedLabel` — 符号反転ラベル
  - `http://www.xbrl.org/2003/role/periodStartLabel` / `periodEndLabel` — 期首/期末ラベル
  - EDINET 固有: 連結/個別/中間/業種ごとのラベルロール（上記 roleRef 表参照）
- **全ファイル行数**: 22,550行（単一ファイルに全 concept のラベルを格納）

#### C-10 向け: プレゼンテーションリンクベース構造のまとめ

[F5] より:

- **`link:presentationArc`**: 親子関係

  - `xlink:from` / `xlink:to`: `link:loc` の `xlink:label` を参照
  - `order`: 表示順（`1.0`, `2.0`, ... 小数可）
  - `preferredLabel`: 表示時に使用するラベルロール（省略時は標準ラベル）
- **`link:presentationLink`**: プレゼンテーションリンクの単位
  - `xlink:role`: 財務諸表の種類を示す role URI（例: `rol_std_ConsolidatedBalanceSheet`）

### 推論（Reasoning）

1. [F1] より、jppfs モジュールは語彙層（XSD 3 ファイル）、ラベル層（3 ファイル）、参照層（1 ファイル）、関係層（23業種 × 複数ファイル）で構成される。語彙・ラベル・参照は業種横断で共有され、関係層のみが業種別に分割されている。
2. [F2] + [F9] + [F10] より、concept 定義は `xsd:element` として一律に定義され、属性セットは限定的（name, id, type, substitutionGroup, abstract, nillable, balance, periodType の 8 属性）。`type` の値は `xbrli:monetaryItemType`(3,763), `xbrli:stringItemType`(283), `nonnum:domainItemType`(93) の 3 種、`substitutionGroup` は `xbrli:item`(4,086), `iod:identifierItem`(41), `xbrldt:hypercubeItem`(9), `xbrldt:dimensionItem`(3) の 4 種。abstract 要素および domainItemType 要素には balance 属性がないパターンが一貫している。よって C-9 の concept 属性は上記の 8 属性で網羅される。
3. [F3] + [F4] より、ラベルリンクベースは単一ファイル（22,550行）に全 concept のラベルを格納し、70行にわたる roleRef 宣言で EDINET 固有のラベルロール（連結/個別/中間/業種別）を参照している。C-5 のラベル構造は `loc → labelArc → label` の 3 要素パターンで統一。
4. [F1] + [F5] より、`jppfs_2025-11-01_pre.xml` は存在せず、プレゼンテーションリンクベースは `r/{業種}/{ファイル名}.xml` に分割されている。質問文の想定と異なるが、各ファイルの XML 構造は `loc → presentationArc` の標準パターンに従っている。C-10 の XML 構造は [F5] のサンプルで十分に判明。
5. [F6] より、ジェネリックラベル（gla.xml）は concept のラベルではなく roleType の英語ラベルを定義するものである。C-5 のラベル解決時に gla.xml は concept ラベルの情報源とはならない。
