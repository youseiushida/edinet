### 推論（Reasoning）

#### K-2.1 採用状況

1. [F1] より、EDINET は「インライン XBRL」形式での報告書作成を義務付けている。「作成します」という断定表現から、全面移行済みと判断できる。

#### K-2.2 file_type="1" の ZIP 内容

2. [F13] より、全5ターゲットの ZIP に `.xbrl` と `.htm` の両方が含まれることを実証。
3. [F2]+[F4] より、`.xbrl` は提出者が作成したものではなく、EDINET が iXBRL から自動生成したもの。

#### K-2.3 EDINET が変換済みを提供するか

4. [F2]+[F4]+[F13] より、EDINET は iXBRL → 従来型 XBRL の変換済みファイルを ZIP に同梱する。仕様書の記述と実ファイルの両方で確認済み。

#### K-2.4 iXBRL のパース方法

5. [F2]+[F13] より、v0.1.0 では自動生成 `.xbrl` をパースすれば、iXBRL パーサーの実装なしに全 Fact を取得可能。
6. [F7] より、iXBRL の要素構造は `ix:nonFraction`（数値）、`ix:nonNumeric`（文字列）、`ix:footnote`、`ix:header`、`ix:hidden` の5種類。
7. [F9]+[F15] より、scale 属性は `-2`, `0`, `3`, `6` の4値。XBRL 値 = 表示値 × 10^scale の理解で正しい。実ファイルで `0` と `6` を確認。
8. [F10]+[F15] より、format 属性は `ixt:numdotdecimal`（カンマ区切り数値）を確認。仕様書の記述と一致。

#### K-2.5 IXDS（Inline XBRL Document Set）

9. [F3] より、報告書インスタンスは「表紙」+「本文（複数可）」の IXDS 構成。
10. [F11]+[F14] より、Context / Unit の定義は表紙ファイルの ix:resources に集約。本文ファイルには Context/Unit 定義なし [F16]。
11. [F18] より、schemaRef も表紙の ix:references 内に定義。

#### K-2.6 パーサーの対応方針

12. [F2]+[F13] より、v0.1.0 では .xbrl のみパースすれば十分。iXBRL パーサーは v0.2.0+ で表示情報（HTML レイアウト、セクション構造）の取得が必要になった場合に実装を検討。

#### K-2.7 既存コードへの影響

13. [F13] より、全ターゲットの ZIP に `.xbrl` が存在するため、`find_primary_xbrl_path` は現状維持で動作する。修正不要。

#### K-2.8 構造的等価性

14. [F2]+[F4] より、自動生成 .xbrl は iXBRL の全 Fact を含む。iXBRL 固有の表示情報（HTML タグ、CSS スタイル）は .xbrl に含まれないが、XBRL データモデル上の情報（Fact、Context、Unit、DTS）は完全に等価。

#### K-2.9 変換ツール

15. [F2] より、EDINET が自動変換を提供するため、外部ツールは不要。

#### K-2.10 自動検出アルゴリズム

16. [F5]+[F17] より、iXBRL ファイルは `.htm` 拡張子 + `xmlns:ix` 名前空間宣言で識別可能。
17. [F8] より、DOCTYPE 宣言が禁止されているため、`<?xml version=...>` + `<html>` の組み合わせで判別可能。
18. 純粋な HTML レンダリング用 .htm ファイルの存在: P-1e の訂正報告書 (S100VUX5) で `PublicDoc/0000000_header.htm`（XBRL/PublicDoc/ 外）が確認されたが、これは XBRL 非対応の旧ファイルであり、XBRL/PublicDoc/ 配下の .htm は全て iXBRL。パスによる判別で十分。

#### K-2.11 XHTML パース耐性

19. [F5]+[F12]+[F17] より、iXBRL は XHTML1.1 準拠の well-formed XML。`<?xml version="1.0">` 宣言あり、DOCTYPE 禁止、タグは必ず閉じる制約。lxml の XML パーサーで処理可能。HTML パーサー（html5lib 等）は不要。

#### K-2.12 ix:tuple

20. [F19] より、仕様書全体に `ix:tuple` の記載なし。EDINET では ix:tuple は不使用と推定。

### 確信度

- 高（仕様書の明示的記述 + 全5ターゲットの実ファイル検証で裏付け済み）
- K-2.12 のみ確信度: 中（仕様書に記載なし = 不使用と推定。実ファイルでの網羅的検証はしていない）
- **決定ゲート結論**: ZIP に `.xbrl` が存在する → v0.1.0 は従来型 `.xbrl` パースで十分。`find_primary_xbrl_path` は現状維持。iXBRL パーサーは不要（Needs Review 対象外）。
