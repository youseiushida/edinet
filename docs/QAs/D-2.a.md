## D-2. US-GAAP 適用企業の XBRL

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。D-1 と同じ質問群を US-GAAP について回答する。

| サブ質問 | 回答 |
|----------|------|
| D-2.1 タクソノミ入手方法 | US-GAAP 専用のタクソノミモジュールは存在しない。ALL_20251101/taxonomy/ に `usgaap` や `us-gaap` ディレクトリはない。US-GAAP 関連の要素は `jpcrp_cor`（開示府令タクソノミ）内に定義されており、37要素が確認された。 |
| D-2.2 名前空間 | US-GAAP 専用の名前空間プレフィックスは存在しない。USGAAP 関連要素はすべて `jpcrp_cor` 名前空間（`http://disclosure.edinet-fsa.go.jp/taxonomy/jpcrp/.../jpcrp_cor`）に属する。 |
| D-2.3 勘定科目 | 2種類の要素が存在する。(a) TextBlock 型（14要素）: 連結BS/PL/CF等の包括タグ付け用。(b) SummaryOfBusinessResults 型等（20要素）: 主要な経営指標等セクション用19要素（売上高、営業利益、総資産等）+ 説明文要素1。残り3要素は Abstract。詳細は推論 3 参照。 |
| D-2.4 経常利益 | 該当なし。US-GAAP の財務諸表は包括タグ付けのみ（TextBlock）のため、個別の勘定科目による詳細タグ付けは行われない。SummaryOfBusinessResults 要素にも `Ordinary` を含む要素は存在しない（US-GAAP にも J-GAAP の「経常利益」に直接対応する概念はない）。 |
| D-2.5 Context | J-GAAP と同じ基本構造。包括タグ付けでも標準的な Context ID（`CurrentYearDuration` 等）が使用される。 |
| D-2.6 ラベル | `jpcrp_cor` のラベルファイル（`jpcrp_2025-11-01_lab.xml`）に含まれる。例: `RevenuesUSGAAPSummaryOfBusinessResults` → 「売上高」、`TotalAssetsUSGAAPSummaryOfBusinessResults` → 「総資産額」。 |
| D-2.7 US-GAAP 適用企業数 | 歴史的には10〜15社程度。近年は多くが IFRS に移行しており、さらに減少傾向。正確な現在数は仕様書からは確認不可（EDINET API で `AccountingStandardsDEI = "US GAAP"` をフィルタすることで把握可能）。 |

### 推論（Reasoning）

1. **D-2.1 タクソノミ入手方法**: [F1] より、ALL_20251101/taxonomy/ に US-GAAP 専用のディレクトリが存在しない。[F2] より、US-GAAP 関連の37要素はすべて `jpcrp_cor`（開示府令タクソノミ）の XSD 内に定義されている。IFRS が `jpigp_cor` という専用モジュールを持つのとは対照的に、US-GAAP は独立タクソノミを持たず、開示府令タクソノミに統合されている。これは [F6] の通り US-GAAP が包括タグ付けのみであるため、専用の勘定科目体系（詳細タクソノミ）を必要としないことと整合する。

2. **D-2.2 名前空間**: [F1] + [F2] より、US-GAAP 要素は `jpcrp_cor` 名前空間に属する。IFRS の `jpigp_cor` のような専用名前空間プレフィックスは存在しない。パーサー実装において US-GAAP を名前空間で判別することはできず、`AccountingStandardsDEI = "US GAAP"` による判別が必要（→ D-3 参照）。

3. **D-2.3 勘定科目**: [F3] + [F4] より、USGAAP 要素は以下の2カテゴリに分類される:
   - **(a) TextBlock 要素（14個）**: `ConsolidatedBalanceSheetUSGAAPTextBlock`、`ConsolidatedStatementOfIncomeUSGAAPTextBlock` 等。これらは財務諸表の各表を HTML ブロックとして丸ごと格納するための要素であり、包括タグ付け（block tagging）に使用される。
   - **(b) SummaryOfBusinessResults 要素等（20個）**: `RevenuesUSGAAPSummaryOfBusinessResults`（売上高）、`OperatingIncomeLossUSGAAPSummaryOfBusinessResults`（営業利益）、`TotalAssetsUSGAAPSummaryOfBusinessResults`（総資産額）等19要素は「主要な経営指標等の推移」セクションで使用される詳細タグ付け要素。残り1要素 `DescriptionOfFactThat...USGAAPFinancialInformation` は米国基準に基づく連結財務諸表作成の事実に関する説明文要素。

   IFRS の場合は `jpigp_cor` が PL/BS/CF の個別勘定科目を詳細に定義しているのに対し、US-GAAP では TextBlock による包括タグ付けのみであり、財務諸表内の個別科目は XBRL 化されない。

4. **D-2.4 経常利益**: [F6] より US-GAAP の財務諸表は包括タグ付けのみであるため、経常利益の個別タグ付けは行われない。SummaryOfBusinessResults 要素にも "Ordinary" を含む要素は存在しない。US-GAAP 上は J-GAAP の「経常利益」に直接対応する概念がなく、`ProfitLossBeforeTaxUSGAAPSummaryOfBusinessResults`（税引前利益）が最も近い概念として経営指標等に含まれる。

5. **D-2.5 Context**: [F6] より包括タグ付けであっても Context の基本構造は J-GAAP と同じ仕組みを使用する（`CurrentYearDuration`、`Prior1YearDuration` 等の標準 Context ID）。TextBlock は `duration` 型で記載される。

6. **D-2.6 ラベル**: [F9] より、US-GAAP 関連要素のラベルは `jpcrp_cor` のラベルファイルに含まれる。専用タクソノミモジュールがないため、ラベルの取得先も `jpcrp_cor` に一元化されている。

7. **D-2.7 企業数**: 仕様書やタクソノミからは US-GAAP 適用企業数の記載を確認できなかった。公開情報として、日本で US-GAAP を適用する上場企業は歴史的に10〜15社程度（例: IBM Japan、P&G Japan、コカ・コーラ ボトラーズ等の外資系企業子会社）だが、近年は IFRS への移行が進み減少傾向にある。正確な現在数は EDINET API の書類一覧から `AccountingStandardsDEI = "US GAAP"` でフィルタすることで取得可能。

8. **JMIS との比較**: [F5] + [F7] より、修正国際基準（JMIS）も US-GAAP と同様のパターン（TextBlock + SummaryOfBusinessResults）で、包括タグ付けのみ。JMIS は US-GAAP よりさらに適用企業数が少ない（JMIS は ASBJ が策定した日本独自基準で、2016年の適用開始以降ごく少数の企業のみが採用）。
