## H-4. 文字コード

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| H-4.1 `.xbrl` の文字エンコーディング | UTF-8 固定。XML 宣言で `encoding="UTF-8"` が明記される |
| H-4.2 ラベルリンクベースの文字エンコーディング | UTF-8 固定。XML 宣言で `encoding="UTF-8"` が明記される |

### 情報源（Fact）

- [F1] パス: `docs/仕様書/XBRL Instance Guideline 報告書インスタンス作成ガイドライン.md` L.761-763 — 「報告書インスタンスで使用する文字コード（エンコーディング形式）は、「UTF-8」です。報告書インスタンスでは、文字コードを UTF-8 に指定する際に BOM(Byte Order Mark) を付与することを原則とします。」
- [F2] パス: `docs/仕様書/EDINET提出書類ファイル仕様書.md` L.837-841 — 「EDINET で使用する文字コードは、「UTF-8」です。」
- [F3] サンプル確認: `docs/仕様書/2026/サンプルインスタンス/.../02_開示府令-有価証券報告書/S002XXXX/XBRL/PublicDoc/jpcrp030000-asr-001_X99001-000_2026-03-31_01_2026-06-12.xbrl` L.1 — `<?xml version="1.0" encoding="UTF-8"?>`（EDINET が自動生成する .xbrl ファイル。BOM なし）
- [F4] サンプル確認: 同ディレクトリの iXBRL ヘッダ `0000000_header_..._ixbrl.htm` — 先頭バイト `EF BB BF`（UTF-8 BOM あり）+ `<?xml version="1.0" encoding="utf-8"?>`
- [F5] サンプル確認: `ALL_20251101/taxonomy/jppfs/2025-11-01/label/jppfs_2025-11-01_lab.xml` — `<?xml version="1.0" encoding="UTF-8"?>`（BOM なし）

### 推論（Reasoning）

1. [F1] + [F2] より、EDINET の全ファイルは UTF-8 固定である。他のエンコーディングは仕様上許容されない。
2. [F3] より、EDINET が自動生成する `.xbrl` ファイルは XML 宣言に `encoding="UTF-8"` を明記し、BOM は付与しない。
3. [F4] より、提出者が作成する iXBRL `.htm` ファイルは UTF-8 BOM 付きである（仕様書 [F1] の「BOM 付与を原則」に準拠）。
4. [F5] より、標準タクソノミのラベルリンクベースは UTF-8・BOM なしである。
5. パーサー実装上の留意点: `.xbrl`（自動生成）とタクソノミ XML は BOM なし、iXBRL `.htm` は BOM ありという混在が発生する。lxml は BOM を正しく処理するため、`etree.parse()` / `etree.fromstring()` で問題ないが、バイト列を文字列にデコードしてから渡す場合は BOM 除去が必要。→ H-4b で詳述。

### 確信度

- 高（仕様書に明記 + サンプルファイルで実証）
