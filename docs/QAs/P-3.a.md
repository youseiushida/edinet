### 推論（Reasoning）

1. [F2] より、提出者別タクソノミの targetNamespace は提出ごとに一意の URL パターンで構成される。この URL はフォームコード・EDINET コード・決算期末日・提出番号・提出日を含み、同一企業でも提出ごとに異なる。
2. [F3] より、DTS 解決のエントリポイントは提出者別 .xsd の `import` 文。URL は `http://disclosure.edinet-fsa.go.jp/taxonomy/{module}/{version}/{file}` パターンで、ローカルタクソノミ `ALL_20251101` への変換が可能（URL パスの `/taxonomy/` 以下がローカルディレクトリに対応）。
3. [F3] より、トヨタは IFRS 適用企業のため `jpigp` (IFRS タクソノミ) を import している。J-GAAP 企業では `jpigp` の代わりに `jpgaap` が import されると推定。
4. [F4] より、linkbaseRef は EDINET 標準タクソノミのラベル・定義と、提出者別のプレゼンテーション・定義・計算・ラベルの2層構造。提出者別リンクベースは相対パスで同一 ZIP 内のファイルを参照。
5. [F5] より、拡張要素は TextBlock 型（テキスト注記）、数値型（独自勘定科目）、Member 型（ディメンション用）の3種類。92個の拡張要素はトヨタ固有の開示項目に対応。
