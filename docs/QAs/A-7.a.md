## A-7. 重複 Fact

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| A-7.1 重複 Fact の存在可能性 | YES。同一の concept + 同一の context + 同一の unit で、同一の値を持つ Fact が複数存在するケース（consistent duplicate）は、仕様上認められている |
| A-7.2 処理方針 | consistent duplicate（同一値）: 許可。inconsistent duplicate（異なる値）: **禁止**。EDINET 仕様書で明示的に禁止 |
| A-7.3 一意性の厳密な定義 | XBRL 2.1 仕様上、2つの Fact が「重複」と判定される条件は `concept` + `context` + `unit` + `xml:lang` の完全一致。ただし EDINET の従来型 .xbrl では Fact 要素に `xml:lang` が付与されないため（A-8 参照）、実質的に `concept` + `context` + `unit` の3項目で一意性が決まる |

### 情報源（Fact）

- [F1] パス: `docs/仕様書/XBRL Instance Guideline 報告書インスタンス作成ガイドライン.md` L.1609-1610 — §5-6-1-2 「同一の要素、コンテキスト及びユニットを用いるべき同一の値が詳細タグ付け対象範囲に複数回現れる場合、出現箇所の全てにおいてタグ付けします。」
- [F2] パス: 同上 L.1611-1628 — §5-6-1-2 「インライン XBRL において、同一コンテキスト及びユニットが設定された同一の要素による、異なる値のタグ付けは禁止します。例えば、次の図表のように同一の要素、コンテキスト及びユニットで、「303,080」及び「450,030」という二つの異なる値をタグ付けることはできません。」
- [F3] サンプルファイル: J-GAAP サンプル `.xbrl` — サンプルファイル内では重複 Fact は見つからなかった（サンプルは教科書的な作りのため）

### 推論（Reasoning）

1. [F1] より、consistent duplicate（同一値の重複タグ付け）は仕様上認められている。具体的には、同一の Fact（同じ concept + context + unit + 値）が書類内の複数箇所に出現する場合、全てにタグを付けることが求められている。これにより、例えば有報の「主要な経営指標等」と「連結損益計算書」の両方に同じ売上高が記載されている場合、両方にタグが付けられ、結果として `.xbrl` に同一の Fact が2つ現れる。

2. [F2] より、inconsistent duplicate（同一の concept + context + unit で異なる値）は**明示的に禁止**されている。仕様書に具体的な禁止例（`303,080` と `450,030` の2値）が示されている。

3. XBRL 2.1 仕様上の重複 Fact の一意性定義:
   - **数値 Fact**: `concept` (要素の QName) + `context` (contextRef) + `unit` (unitRef) の3項目
   - **テキスト Fact**: `concept` + `context` + `xml:lang` の3項目
   - ただし A-8 で確認した通り、EDINET の従来型 `.xbrl` では Fact 要素に `xml:lang` が付与されないため、テキスト Fact の一意性は実質 `concept` + `context` の2項目で決まる

4. **パーサーの処理方針**:
   - consistent duplicate（同一値）が検出された場合: 1つのみ保持する（重複を除去）。値は同一なのでどちらを採用しても結果は同じ
   - inconsistent duplicate（異なる値）が検出された場合: エラーまたは警告をログに記録する。仕様違反であるため、値の正確性を保証できない。保守的には最初に出現した値を採用し、ログで報告する

5. [F3] より、サンプルファイルでは重複 Fact は出現しなかった。ただし実際の提出データでは consistent duplicate が発生しうるため、パーサーはこのケースを処理できる必要がある。

### 確信度

- **consistent duplicate の許容**: 高（仕様書 L.1609-1610 で「全てにおいてタグ付けします」と明記）
- **inconsistent duplicate の禁止**: 高（仕様書 L.1611-1628 で「禁止します」と明記、具体例付き）
- **一意性の定義**: 高（XBRL 2.1 仕様の標準的な定義。EDINET での xml:lang 不使用は A-8 で確認済み）
