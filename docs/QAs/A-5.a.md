### フットノートリンクの構造

```
link:footnoteLink (xlink:role = 財務諸表の拡張リンクロール)
  ├── link:footnote (リソース: 注記番号テキスト "※1" 等)
  │     xlink:role = NotesNumber / NotesNumberPeriodStart / NotesNumberPeriodEnd
  │     xml:lang = "ja"
  ├── link:loc (ロケーター: Fact の ID を参照)
  │     xlink:href = "#IdFactXXXX"
  └── link:footnoteArc (アーク: Fact → 注記番号を結びつけ)
        xlink:arcrole = fact-footnote
        xlink:from = "fact"
        xlink:to = "footnote"
```

### footnoteRole の種類

| footnoteRole | 意味 | 用途 |
|---|---|---|
| `NotesNumber` | 注記番号 | 一般的な注記（※1, ※2 等） |
| `NotesNumberPeriodStart` | 期首注記番号 | 当期首の注記番号 |
| `NotesNumberPeriodEnd` | 期末注記番号 | 当期末の注記番号 |

### 推論（Reasoning）

1. [F1]+[F2] より、EDINET の報告書インスタンスにはフットノートリンクが含まれる。ただし日本基準の財務諸表本表に限定される。

2. [F8] より、実際のサンプル `.xbrl` で `link:footnoteLink` 要素を確認。連結貸借対照表（`rol_ConsolidatedBalanceSheet`）内の数値 Fact に対して ※1, ※8, ※2 等の注記番号が紐付けられている。

3. [F9] より、IFRS 有価証券報告書にも `link:footnoteLink` が存在するが、全て `jppfs` ロール URI（日本基準の個別財務諸表）のセクションに限られる。これは [F2] の「日本基準の財務諸表本表の値に対してのみ使用」と整合する。IFRS の連結財務諸表部分では footnoteLink は使われない。

4. [F3]+[F5] より、フットノートリンクの構造は仕様で厳密に規定されている。`arcrole` は `fact-footnote` のみ（`fact-explanatoryFact` は不使用）、一つのフットノートには一つの注記番号を対応付ける。

5. [F4] より、`order` 属性は使用しない。これは [F6] の GFM Rule 2.9.7（order 属性必須）を EDINET が対象外としていることと整合する。

6. [F10] より、iXBRL 形式の提出データにも `ix:footnote` として同等の情報が含まれる。EDINET が自動生成する `.xbrl` にもフットノートリンクが含まれるため、v0.1.0 の `.xbrl` パーサーでもフットノートリンクのパースは技術的に可能。

7. パースの必要性について: フットノートリンクは数値 Fact と注記番号の対応関係を提供する。注記番号は財務諸表の理解に有用だが、数値 Fact 自体の取得には不要。v0.1.0 MVP では数値の取得が主目的のため、フットノートリンクのパースは v0.2.0+ に後回しできる。
