### 推論（Reasoning）

1. [F3] より、file_type=1 の ZIP は `XBRL/PublicDoc/` と `XBRL/AuditDoc/` の2つのサブディレクトリを持つ。PublicDoc が提出本文書（財務データ）、AuditDoc が監査報告書。
2. [F3] より、file_type=1 には iXBRL HTML（`_ixbrl.htm`）、従来型 XBRL（`.xbrl`）、スキーマ（`.xsd`）、3種のリンクベース（`_pre.xml`, `_cal.xml`, `_def.xml`）、マニフェスト、画像が含まれる。XBRL パーサーに必要な情報はすべてこの ZIP に含まれる。
3. [F4] より、file_type=2 は ZIP ではなく PDF バイナリが直接返される。`DownloadFileType.PDF.is_zip` が `False` であることと一致する。
4. [F5] より、file_type=3 は添付文書（約款等）の PDF を ZIP で返す。XBRL とは無関係。
5. [F6] より、file_type=4 は今回のサンプルでは has_english=False のため検証できなかった。英文 XBRL の有無（A-8.2 の `xml:lang` と関連）は、英文ファイルを持つ書類を別途サンプリングする必要がある。
