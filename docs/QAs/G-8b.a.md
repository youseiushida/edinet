## §1. 外国企業の XBRL は国内企業と同じタクソノミ・名前空間構造か

**結論**: はい、日本に上場する外国企業（外国会社報告書等）も **同一の EDINET タクソノミ体系**（jpcrp + jppfs/jpigp + jpdei）を使用する。ただし、外国会社用の専用様式（第七号様式、第八号様式、第九号様式等）が割り当てられている。

### 情報源 (Fact)

- **F1**: 仕様書「EDINETタクソノミの概要説明」L.845-850: 図表 2-3-1:
  - No.15: 有価証券届出書 第七号様式 → `jpcrp`（外国会社・通常方式）
  - No.16: 有価証券届出書 第七号の四様式 → `jpcrp`（外国会社・組織再編成）
  - No.17: 有価証券報告書 第八号様式 → `jpcrp`（外国会社）
  - No.18: 有価証券報告書 第九号様式 → `jpcrp`（外国会社）
  - No.19: 半期報告書 第九号の三様式 → `jpcrp`（外国会社）
  - No.20: 半期報告書 第十号様式 → `jpcrp`（外国会社）
- **F2**: フレームワーク設計書 L.332-335: 外国会社の有価証券報告書（第七号～第九号様式）でも DEI ○ + 財務諸表本表タクソノミの列に ○。
- **F3**: P-4.a.md: Sony（IFRS 企業）の ZIP 構造分析で `jpigp_cor` の使用を確認。ディレクトリ構造は J-GAAP と同一。DEI 構造も同一。

### 推論 (Reasoning)

F1 により、外国会社の報告書は国内企業と同じ `jpcrp` タクソノミ分割単位に属し、様式番号のみが異なる（第七号～第十号様式 vs 国内の第二号～第五号様式）。F2 で財務諸表本表タクソノミの使用も確認されている。F3 の実データ（Sony IFRS）でも EDINET 標準の構造が使用されていることが実証されている。

## §2. 異なるタクソノミ（IFRS/US-GAAP ネイティブ）を使用するか

**結論**: いいえ。EDINET では IFRS ネイティブタクソノミ（ifrs-full）や US-GAAP タクソノミは **使用しない**。EDINET 独自の `jpigp`（国際会計基準タクソノミ）を使用する。US-GAAP の場合も EDINET タクソノミ体系内で対応する。

### 情報源 (Fact)

- **F4**: D-3.a.md: 会計基準の判別方法:
  - `AccountingStandardsDEI` = "Japan GAAP" → jppfs_cor
  - `AccountingStandardsDEI` = "IFRS" → jpigp_cor
  - `AccountingStandardsDEI` = "US GAAP" → jpigp_cor または jppfs_cor（EDINETの US-GAAP 対応は限定的）
  - `AccountingStandardsDEI` = "JMIS"（修正国際基準）→ jpigp_cor
- **F5**: 図表 2-3-7 (L.963-964): 財務諸表本表タクソノミは `jppfs`（日本基準）と `jpigp`（国際会計基準）の2つのみ。US-GAAP 専用タクソノミは存在しない。
- **F6**: P-4.a.md: Sony（IFRS企業）の分析で `jpigp_cor` 名前空間を確認。ifrs-full 名前空間は不使用。

### 推論 (Reasoning)

F5 により EDINET のタクソノミ体系は jppfs / jpigp / jpdei の3分割で完結しており、国際的なタクソノミ（ifrs-full, us-gaap）をそのまま使用することはない。F6 の実データでもこれが確認されている。外国企業であっても EDINET に提出する以上、EDINET タクソノミ体系に従う必要がある。

## §3. v0.1.0 でパースエラーが出るか

**結論**: 基本的にパースエラーは出ない。外国企業のXBRLも国内企業と同一のタクソノミ・名前空間・ファイル構造を使用するため、v0.1.0 の `.xbrl` パーサーでそのまま処理可能。

### 情報源 (Fact)

- **F7**: P-4.a.md: `find_primary_xbrl_path` が IFRS 企業（Sony）のZIPでも修正なしに動作することを確認。
- **F8**: K-2.a.md: v0.1.0 は `.xbrl` ファイルのみを解析。EDINET は iXBRL(.htm) から自動生成された `.xbrl` を ZIP に含めるため、外国企業でも `.xbrl` ファイルが存在する。
- **F9**: F-1.a.md: DEI 要素は全書類種別で共通。外国企業固有の DEI 要素は存在しない。

### 推論 (Reasoning)

F7-F9 により、外国企業の XBRL は以下の点で国内企業と同一:
1. ZIP 構造（PublicDoc/AuditDoc）
2. `.xbrl` ファイルの存在と命名規則
3. 名前空間（jpcrp_cor, jppfs_cor/jpigp_cor, jpdei_cor）
4. DEI 要素の構成

したがって、v0.1.0 のパーサーは外国企業のファイルをスキップする必要はなく、そのまま処理可能。会計基準の違い（Japan GAAP vs IFRS）は `AccountingStandardsDEI` で判別可能であり(F4)、使用される名前空間の差異（jppfs_cor vs jpigp_cor）に対応するだけでよい。
