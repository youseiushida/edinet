# H-1.a: schemaRef と linkbaseRef

## 質問

インスタンス文書からの DTS エントリポイントを確定する。（7問）

## 品質

確度: **A（実証済み）** — 実 Filing 3件（トヨタ、投信2件）+ 先行回答 P-2/P-3 で検証。

## 回答

### H-1.1: schemaRef の href は相対パスか絶対 URL か

**結論: 相対パス。**

- [F1] 全3件の Filing で schemaRef は **相対パス**（同一ディレクトリ内の提出者 XSD を指す）:
  - トヨタ: `jpcrp030000-asr-001_E02144-000_2025-03-31_01_2025-06-18.xsd`
  - 投信A: `jpsps070000-asr-001_G13272-000_2025-04-07_01_2025-07-03.xsd`
  - 投信B: `jpsps070000-asr-001_G13255-000_2024-03-10_01_2024-06-03.xsd`
- [F2] P-2.a.md [F5] でも確認済み
- **出典**: `H-1.schema_ref.py` 実行結果

### H-1.2: linkbaseRef はインスタンスに存在するか

**結論: インスタンス文書には存在しない。XSD の appinfo にある。**

- [F3] 全3件の Filing で、`.xbrl` インスタンス内の `linkbaseRef` は **0件**
- [F4] P-3.a.md [F4] より、linkbaseRef は全て **提出者 XSD の `xsd:appinfo`** 内に記載:
  - EDINET 標準ラベル（絶対 URL）: `jpcrp_2024-11-01_lab.xml`, `jppfs_2024-11-01_lab.xml` 等
  - 提出者固有リンクベース（相対パス）: `*_pre.xml`, `*_cal.xml`, `*_def.xml`, `*_lab.xml`, `*_lab-en.xml`
- **出典**: `H-1.schema_ref.py` + P-3.a.md [F4]

### H-1.3: DTS 連鎖の構造

**確認済み（H-1b 参照）:**

- [R1] DTS 連鎖: `schemaRef` → 提出者 XSD → `xs:import`（絶対 URL）→ EDINET 標準タクソノミ XSD → さらに `xs:import`（最大深さ2）
- 詳細は H-1b.a.md [F4] 参照

### H-1.4: xsi:schemaLocation 属性の有無

**結論: EDINET の .xbrl ファイルには存在しない。**

- [F5] 全3件の Filing で `xsi:schemaLocation` は **NOT FOUND**
- [R2] [F5] → DTS 解決に `xsi:schemaLocation` を参照する必要はない。`schemaRef` のみがエントリポイント。
- **出典**: `H-1.schema_ref.py` 実行結果

### H-1.5: Linkbase 発見経路

**結論: 経路 (b) のみ — XSD appinfo の linkbaseRef。**

- [F6] [F3]+[F4] → インスタンスには linkbaseRef なし（経路 (a) は不使用）
- [F7] 提出者 XSD の `xsd:appinfo` に全ての linkbaseRef が集約（経路 (b) のみ）
- [R3] パーサーは `schemaRef` → XSD → `xsd:appinfo` 内の `link:linkbaseRef` を辿ればよい。インスタンスの linkbaseRef 探索は不要。

### H-1.6: schemaRef の数（複数あるか）

**結論: 常に1件。**

- [F8] 全3件の Filing で schemaRef は **1件のみ**
- [R4] XBRL 2.1 は複数 schemaRef を許容するが、EDINET では提出者 XSD 1件のみが標準。複数 schemaRef への対応は不要。
- **出典**: `H-1.schema_ref.py` 実行結果

### H-1.7: roleRef / arcroleRef の有無

- [F9] roleRef:
  - トヨタ: **3件** — `rol_BalanceSheet`, `rol_StatementOfIncome`, `role/NotesNumber`
  - 投信A: **0件**
  - 投信B: **3件** — `rol_fnd_BalanceSheet-ListOfAccounts`, `role/NotesNumber`, `rol_fnd_StatementOfIncomeAndRetainedEarnings-ListOfAccounts`
- [F10] arcroleRef: 全3件で **0件**
- [R5] [F9] → roleRef はインスタンス内で使用されている拡張ロール（Extended Link Role）の宣言。財務諸表の構造（BS、PL 等）を特定するのに使われる。パーサーは roleRef を読み取り、Presentation Linkbase と突き合わせて表示構造を構築する必要がある。
- [R6] [F10] → arcroleRef は現時点では不使用。footnote link 等で将来必要になる可能性はあるが、v0.1.0 では無視可。
- **出典**: `H-1.schema_ref.py` 実行結果
