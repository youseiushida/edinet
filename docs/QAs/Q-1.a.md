# Q-1.a: DTS 不完全時の fallback

## 質問

パーサーが DTS 解決失敗時にどう振る舞うべきかの設計指針を出す。（3問）

## 品質

確度: **B（推論）** — H-1b の実証結果に基づく論理的推論。

## 回答

### Q-1.1: import 連鎖が途切れた場合、パース中断すべきか

**結論: 中断不要。部分的 DTS で続行可能。**

- [F1] H-1b.a.md [F9] より、ALL_20251101 に XBRL 国際標準スキーマ（`xbrl-instance-2003-12-31.xsd` 等 6件）は **一切含まれない**
- [F2] H-1b.a.md [F4] より、import 連鎖の末端は常にこれらの外部 URL に達する
- [R1] [F1]+[F2] → **全ての EDINET 書類において DTS は不完全になる**のが正常。外部スキーマの欠落でパースを中断すると、何もパースできない。
- [R2] XBRL 国際標準スキーマが定義する型（`xbrli:monetaryItemType`, `xbrli:stringItemType` 等）は **既知の有限集合**であり、ハードコードで対応可能:
  - `xbrli:monetaryItemType` → 金額（decimal）
  - `xbrli:stringItemType` → 文字列
  - `xbrli:sharesItemType` → 株数（decimal）
  - `xbrli:pureItemType` → 比率（decimal）
  - `nonnum:textBlockItemType` → テキストブロック（escaped HTML）
  - `nonnum:domainItemType` → ディメンションメンバー
  - `num:perShareItemType` → 1株当たり金額
- [R3] パーサーは外部 URL を検出したら **無視（skip）** し、既知の型定義はハードコードで補完する設計が最適。

### Q-1.2: ラベル取得のみなら完全 DTS は必要か

**結論: 不要。linkbaseRef → label XML の読取だけで十分。**

- [F3] H-1.a.md [F7] より、linkbaseRef は全て提出者 XSD の `xsd:appinfo` にある
- [F4] H-1b.a.md [F2] より、linkbaseRef URL は EDINET プレフィックス置換でローカル解決可能
- [R4] [F3]+[F4] → ラベル取得には以下の最小限ステップで十分:
  1. `schemaRef` → 提出者 XSD を読取
  2. XSD 内の `link:linkbaseRef` で `role="...labelLinkbaseRef"` を抽出
  3. 各 URL を EDINET プレフィックス置換でローカルパスに解決
  4. label XML を読取し、`{namespace_uri}:{local_name}` → ラベルのマッピングを構築
- [R5] この手順では `xs:import` 連鎖を辿る必要すらない。XSD の appinfo セクションのみで完結する。完全な DTS 解決（全 import を再帰的に辿る）は、型検証やディメンション処理が必要になるまで不要。

### Q-1.3: 不完全 DTS は正常系か

**結論: はい。EDINET の仕様上、不完全 DTS が正常。**

- [F5] H-1b.a.md [F11]+[F12]+[F13] より、ALL_20251101 は XBRL Taxonomy Package に非準拠（catalog.xml なし）
- [F6] H-1b.a.md [F9] より、XBRL 国際標準スキーマは含まれない
- [R6] [F5]+[F6] → EDINET は「オフラインで完全な DTS を構築する」ことを前提としていない。ALL_20251101 は EDINET 独自タクソノミのみを提供し、XBRL 標準スキーマの同梱は仕様外。
- [R7] パーサーの設計方針:
  - **DTS 解決は「最善努力」（best-effort）** とする
  - EDINET URL → ローカル解決: 必須（確実に成功する）
  - XBRL.org URL → ハードコードまたはスキップ: 許容
  - 解決失敗 → 警告ログのみ、パース続行

## 設計への影響（まとめ）

| 状況 | 対応 |
|---|---|
| EDINET URL 解決失敗 | **エラー**（タクソノミファイル欠落の可能性） |
| XBRL.org URL 解決失敗 | **スキップ**（正常、型情報はハードコード） |
| linkbaseRef 解決失敗 | **警告**（ラベルが欠落するが Fact 抽出は可能） |
