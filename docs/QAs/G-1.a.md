## §1. XBRL の構造的差異・名前空間

**結論**: 有報・四半期報告書・半期報告書は、すべて同一のタクソノミ分割単位 `jpcrp` を共有しており、名前空間は同じである。

### 情報源 (Fact)

- **F1**: 仕様書「EDINETタクソノミの概要説明」L.825-826: 「開示府令のうち、臨時報告書及び自己株券買付状況報告書を除いて一つのタクソノミとします。」
- **F2**: 同 L.831-850: 図表 2-3-1 において、有価証券届出書(No.1-7)、有価証券報告書(No.8-10)、半期報告書(No.11-13) がすべて `jpcrp` に分類。
- **F3**: 同 L.963: jppfs（日本基準財務諸表本表）、L.964: jpigp（IFRS）は報告書種別によらず共通のタクソノミ分割単位。
- **F4**: サンプルインスタンス 04_半期報告書(銀行業): ファイル名 `jpcrp040300-ssr-001_X99004-000_2026-09-30_01_2026-11-19.xbrl` → プレフィックス `jpcrp`。
- **F5**: サンプルインスタンス 02_有価証券報告書: ファイル名 `jpcrp030000-asr-001_...` → プレフィックス `jpcrp`。
- **F6**: P-1.a.md: 全5件の実ファイル検証でタクソノミ名前空間 `jpcrp_cor` を確認。

### 推論 (Reasoning)

F1 により、開示府令において臨時報告書(jpcrp-esr)と自己株券買付状況報告書(jpcrp-sbr)だけが別タクソノミとして分離されている。残りの有報・半期報告書・届出書等はすべて `jpcrp` という単一のタクソノミ分割単位に属する(F2)。サンプルファイル名(F4, F5)もこれを裏付け、名前空間 URI は `http://disclosure.edinet-fsa.go.jp/taxonomy/jpcrp/YYYY-MM-DD/jpcrp_cor` で統一されている。

財務諸表本表は `jppfs`(日本基準)または `jpigp`(IFRS) を別途使用するが、これも報告書種別によらず共通である(F3)。

## §2. 四半期用タクソノミモジュール

**結論**: 四半期決算専用の別タクソノミモジュールは存在しない。通期と同じ `jppfs`（日本基準の場合）を使用する。

### 情報源 (Fact)

- **F7**: 図表 2-3-7 (L.963): jppfs は「日本基準財務諸表の本表に係るタクソノミ要素を含む」とのみ記載。四半期用の jppfs-q 等は存在しない。
- **F8**: フレームワーク設計書 L.325-330: 有価証券報告書(第三号様式)と半期報告書(第四号の三様式)はいずれも DEI + 財務諸表本表タクソノミを使用する（同じ○マーク構成）。
- **F9**: F-1.a.md: `TypeOfCurrentPeriodDEI` の取り得る値は "FY"（通期）と "HY"（半期）の2つのみ。四半期を示す値は定義されていない。

### 推論 (Reasoning)

jppfs は期間の長さを区別しない汎用的な財務諸表本表タクソノミであり(F7)、期間の区別は Context の `startDate`/`endDate` および DEI の `TypeOfCurrentPeriodDEI` で表現される(F9)。四半期専用のタクソノミモジュールは存在しない。

## §3. 四半期の BS/PL/CF 省略

**結論**: 四半期報告書（従来制度下）では、キャッシュ・フロー計算書(CF)が第1・第3四半期で省略される場合があった。半期報告書では原則としてBS/PL/CFすべてを含む。

### 情報源 (Fact)

- **F10**: フレームワーク設計書 L.328-330: 半期報告書の各様式（第四号の三様式、第五号様式、第五号の二様式）はすべて DEI + 財務諸表本表タクソノミの○マークが付いている。
- **F11**: 四半期報告書制度（企業内容等の開示に関する内閣府令の旧規定）では、第1・第3四半期のCF計算書は免除規定があった。ただしXBRLタクソノミ上はCF要素が定義されており、任意で含めることが可能だった。

### 推論 (Reasoning)

XBRLタクソノミ（jppfs）自体はBS/PL/CFすべての要素を含んでおり、どの財務諸表を含めるかは提出者のタクソノミ拡張（提出者別タクソノミ）で決定される(F10)。制度上の省略はタクソノミ構造の差異ではなく、提出データの有無の差異として現れる。

## §4. 四半期報告書制度の廃止

**結論**: 2024年4月の金融商品取引法改正により四半期報告書制度は廃止。`doc_type=140` の新規提出は2024年4月以降ないと推定される。半期報告書(160)への移行と決算短信(TDnet)への二元化が進行。決算短信の XBRL は EDINET では取得不可。

### 情報源 (Fact)

- **F12**: F-1.a.md: `TypeOfCurrentPeriodDEI` の値に "Q1", "Q2", "Q3" 等の四半期値が存在せず "FY" と "HY" のみ。これは現行タクソノミが四半期報告書を想定していないことを示す。
- **F13**: 2024年4月施行の金融商品取引法改正（2023年11月公布）により、上場企業の四半期報告書（第1・第3四半期）は廃止。第2四半期は半期報告書として提出。
- **F14**: 仕様書「EDINETタクソノミの概要説明」L.831-850: 図表 2-3-1 に「四半期報告書」の記載がない。半期報告書(第四号の三様式、第五号様式等)のみ記載。
- **F15**: 決算短信は東京証券取引所（TDnet）の管轄であり、EDINET API の対象外。

- **F16a**: G-1.quarterly_check.py 実行結果（EDINET API 実データ検証、2026-02-23実行）:

| 期間 | doc_type=140 件数 | 備考 |
|------|------------------|------|
| 2024-02-12〜02-16 (廃止前) | **1,644件** | 制度存続中。全件 has_xbrl=True |
| 2024-05-13〜05-17 (廃止直後) | **703件** | 経過措置。2024年3月期Q2分の提出集中 |
| 2024-08-12〜08-16 | **1件** | 変則決算期（パルグループ、Q1=2024/03-05月） |
| 2024-11-11〜11-15 | **0件** | 完全廃止 |
| 2025-02-10〜02-14 | **0件** | 完全廃止 |
| 2025-05-12〜05-16 | **0件** | 完全廃止 |
| 2025-08-11〜08-15 | **0件** | 完全廃止 |

- **F16b**: 同スクリプト: 半期報告書(doc_type=160) は 2025-03-10〜14 に **102件** 存在。全件 has_xbrl=True。主に投資信託の半期報告書。

### 推論 (Reasoning)

F14 により、現行（2025年11月版）のEDINETタクソノミには四半期報告書の様式が含まれておらず、制度廃止が反映済みである。F12 の DEI 値からも四半期の概念がないことが確認できる。F13 の法改正により四半期報告書制度は2024年4月に廃止されたが、F16a の API 実データ検証で**経過措置の実態**が明らかになった:

1. **2024年5月**: 703件の doc_type=140 が提出された。これは2024年3月期の第2四半期（=半期）報告を旧制度の四半期報告書として提出したもの。
2. **2024年8月**: 変則決算期の企業1件のみ（パルグループ、3月〜5月決算のQ1）。
3. **2024年11月以降**: 完全にゼロ。四半期報告書の新規提出は事実上終了。

F16b により半期報告書(160)は2025年時点でも提出されており、現行制度として機能している。決算短信(F15)は TDnet 経由のみで、EDINET API では取得できない。

## §5. 半期報告書と四半期報告書の構造同一性

**結論**: 半期報告書(160)は四半期報告書(140)と同一のタクソノミ分割単位 `jpcrp` を使用しており、名前空間は同じ。Context の期間パターンは6ヶ月間（半期）となる点が異なる。

### 情報源 (Fact)

- **F16**: 図表 2-3-1 (L.841-843): 半期報告書の様式は第四号の三様式（銀行業等）、第五号様式（一般）、第五号の二様式（少額）の3つ。すべて `jpcrp`。
- **F17**: サンプルインスタンス 04_半期報告書(銀行業): ファイル名 `jpcrp040300-ssr-001_...` → `jpcrp` + 様式番号 `040300`(第四号の三様式)。
- **F18**: サンプルインスタンス 05_半期報告書(建設業): ファイル名 `jpcrp040300-ssr-001_...` → 同じく `jpcrp` + `040300`。
- **F19**: F-1.a.md: `TypeOfCurrentPeriodDEI` = "HY" が半期報告書を示す。

### 推論 (Reasoning)

半期報告書は `jpcrp` タクソノミを使用し(F16, F17, F18)、有報と同じ名前空間構造を持つ。四半期報告書も同じ `jpcrp` を使用していた（制度廃止前）。両者の構造的差異は、Context の期間（半期=6ヶ月、四半期=3ヶ月）と DEI の `TypeOfCurrentPeriodDEI` 値のみである。タクソノミモジュール、名前空間、要素定義に差異はない。
