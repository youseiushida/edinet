### 推論（Reasoning）

#### 1. ラベルロール URI 完全リスト（C-5.1）

[F7] + [F1] + [F2] より、ラベルロールは以下の 3 層に分類される。

**A. XBRL 標準ラベルロール（10 種）**

| No | 名称 | URI | jppfs での出現数 |
|----|------|-----|-----------------|
| 1 | 標準ラベル | `http://www.xbrl.org/2003/role/label` | 4,139 |
| 2 | 冗長ラベル | `http://www.xbrl.org/2003/role/verboseLabel` | 4,139 |
| 3 | ドキュメンテーション | `http://www.xbrl.org/2003/role/documentation` | 0（jppfs）/ 214（全体） |
| 4 | 合計ラベル | `http://www.xbrl.org/2003/role/totalLabel` | 184 |
| 5 | 正値ラベル | `http://www.xbrl.org/2003/role/positiveLabel` | 52 |
| 6 | 負値ラベル | `http://www.xbrl.org/2003/role/negativeLabel` | 52 |
| 7 | 期首ラベル | `http://www.xbrl.org/2003/role/periodStartLabel` | 3 |
| 8 | 期末ラベル | `http://www.xbrl.org/2003/role/periodEndLabel` | 3 |
| 9 | 廃止日付ラベル | `http://www.xbrl.org/2009/role/deprecatedDateLabel` | deprecated/ のみ |
| 10 | 廃止理由ラベル | `http://www.xbrl.org/2009/role/deprecatedLabel` | deprecated/ のみ |

**B. EDINET 固有ラベルロール（財務諸表区分 x ラベル種別）**

URI パターン: `http://disclosure.edinet-fsa.go.jp/{パス}`

| カテゴリ | パスパターン | 実在する末尾ロール種別 |
|----------|------------|----------------------|
| 連結 | `jppfs/Consolidated/role/` | label, negativeLabel, positiveLabel, totalLabel |
| 中間連結 | `jppfs/ConsolidatedInterim/role/` | label, negativeLabel, periodEndLabel, periodStartLabel, positiveLabel, totalLabel |
| 個別 | `jppfs/NonConsolidated/role/` | label |
| 中間個別 | `jppfs/NonConsolidatedInterim/role/` | label, negativeLabel, periodEndLabel, periodStartLabel, positiveLabel, totalLabel |
| 業種固有 | `jppfs/{業種略号}/role/` | label, negativeLabel, positiveLabel, totalLabel（業種により異なる） |
| 業種+区分 | `jppfs/{業種}/ConsolidatedInterim/role/` 等 | label（bnk, inv, liq のみ） |
| セグメント | `jpcrp/Segment/role/` | label |
| 代替ラベル | `jpcrp/std/alt(連番)/role/` | label, negativeLabel, positiveLabel, totalLabel |
| 代替+区分 | `jpcrp/std/alt/Consolidated/role/` | label |
| 代替3 | `jpcrp/std/alt3/role/` | label |

実ファイルから確認された全モジュール横断の distinct EDINET 固有 roleRef URI は 72 種。

**C. 仕様上は存在し得るが実データに未出現のロール**

[F7] の注記「label は必要に応じて verboseLabel, documentation 等に読み替え」より、仕様上は EDINET 固有ロールにも verboseLabel, documentation 等の末尾が存在し得る。しかし [F2] の全モジュール横断 roleRef では verboseLabel/documentation 末尾の EDINET 固有ロールは 0 件。これらは理論上可能だが現行タクソノミでは使用されていない。

#### 2. 同一 concept の複数 role ラベルの選択（C-5.2）

[F7] + [F9] + ガイドライン L.2676 より、各 role のラベルは異なる表示用途に対応しており、「優先順位」ではなく「用途に応じた選択」が正しい。選択メカニズムは `presentationArc` の `preferredLabel` 属性:

- preferredLabel 指定あり: 指定された role のラベルを使用
- preferredLabel 指定なし（省略時）: 標準ラベル（`http://www.xbrl.org/2003/role/label`）を使用

[F4] より、実際の preferredLabel の使用分布は totalLabel が 970 件（最多）、次いで業種/区分固有 label が続く。

#### 3. gla.xml の役割（C-5.3）

[F11] + [F12] より、gla.xml（ジェネリックラベルリンク）は拡張リンクロール（roleType）に対する英語ラベルを定義するもの。concept のラベルではない。

- 名前空間: `http://xbrl.org/2008/generic`（gen）, `http://xbrl.org/2008/label`（label）
- アークロール: `http://xbrl.org/arcrole/2008/element-label`
- 参照先: `jppfs_rt_*.xsd` の roleType 定義

パーサーが concept のラベルを解決する際に gla.xml を参照する必要はない。ただし、roleType に表示名を付けたい場合（例: 「連結貸借対照表」を英語表示する等）には gla.xml が有用。

#### 4. 提出者別ラベル vs 標準ラベル（C-5.4）

[F8] + [F9] + [F10] より:

- **日本語ラベル**: 上書き**不可**。EDINET タクソノミの名称リンクを必ずリユーズする。提出者は用途別ラベル（totalLabel, periodStartLabel 等）や代替ラベル（alt role）の**追加**のみ可能
- **英語ラベル**: 上書き**可**。参考情報の位置付け
- **priority 属性**: 名称リンクを上書きする場合（英語ラベル等）、提出者別タクソノミ内で priority 属性の値が重複しないように設定。XBRL の arc override メカニズムにより、高い priority の arc が低い priority を上書きする
- [F14] より、EDINET は use="prohibited" による arc の無効化を許容しているため、提出者が既存の labelArc を prohibited で無効化し、自分の labelArc で置き換えることが可能（ただし日本語ラベルでは禁止されている）

#### 5. ラベル解決の完全なアルゴリズム（C-5.5）

[F8] + [F9] + [F4] + XBRL 仕様を統合したアルゴリズム:

```
入力: concept (QName), requested_role (URI), lang ("ja" | "en")
出力: ラベル文字列

1. requested_role の決定
   - presentationArc に preferredLabel 属性があればその URI
   - なければ http://www.xbrl.org/2003/role/label（標準ラベル）

2. ラベル検索（DTS 内の全 labelLink を統合）
   a. 提出者別タクソノミの _lab.xml から検索
      - concept に対する labelArc (arcrole=concept-label) を収集
      - requested_role かつ lang が一致する label を取得
   b. EDINET 標準タクソノミの _lab.xml から検索
      - 同上のロジック

3. arc override の解決（同一 concept + 同一 role + 同一 lang の重複時）
   - priority 属性が高い方が有効
   - 同一 priority で use="prohibited" の arc があれば対象の arc を無効化
   - use="prohibited" でない最高 priority の label を採用

4. フォールバック（ラベルが見つからない場合）
   a. requested_role のラベルが存在しない場合
      → 標準ラベル (http://www.xbrl.org/2003/role/label) にフォールバック
   b. 標準ラベルも存在しない場合
      → concept の local name をフォールバック表示名とする
      （例: jppfs_cor:CashAndDeposits → "CashAndDeposits"）
```

**重要な注意**: gla.xml は concept ラベルの情報源ではないため、上記アルゴリズムに含めない。

#### 6. ラベルが存在しない concept の扱い（C-5.6）

[F1] + [F15] より:
- EDINET タクソノミの全 concept（4,139 個 in jppfs）には標準ラベル（label）と冗長ラベル（verboseLabel）が必須（◎）で設定されている。[F1] で label 4,139 件 = loc 4,139 件であることから確認済み
- 提出者が新規拡張した要素にも標準ラベル・冗長ラベルの設定は必須
- 日本語・英語の両方を設定する規定（[F7] L.1545）
- [F1] より日本語ラベルファイルと英語ラベルファイルのラベル要素数が 9,169 で完全一致 → 日本語のみ/英語のみの concept は存在しない（jppfs においては）
- [F13] より documentation ロールは jppfs には 0 件だが jpcrp 等には存在。documentation は任意設定のため欠落は正常

フォールバックとして local name を使用することは安全だが、正しく実装されたパーサーでは到達しないはず。

#### 7. extended link role（C-5.7）

[F3] より、jppfs の labelLink の xlink:role は `http://www.xbrl.org/2003/role/link` が 1 件のみ（ファイル全体で 1 つの labelLink 要素に全ラベルが格納される構造）。EDINET 独自の extended link role は使用されていない。

これは XBRL 仕様の一般的なパターンと一致する。ラベルリンクベースでは通常デフォルト role のみを使用し、role による分割はプレゼンテーション/定義リンクベースで行われる。

#### 8. negatedLabel / negativeLabel の使用状況（C-5.8）

[F4] + [F5] + [F6] を統合:

**8a. ラベル定義の存在（_lab.xml 内）**

negativeLabel ロールのラベルは 3 モジュールで計 122 件定義されている:
- jppfs: 75 回（52 件の正値/負値ペア + α）
- jpigp: 41 回
- jpcrp: 6 回（例: 「経常損失」「当期純損失」「セグメント損失」）

**8b. プレゼンテーションでの preferredLabel としての使用**

全 26,578 presentationArc 中、negativeLabel が preferredLabel に設定されているのは **2 件のみ**。いずれも IFRS の CF 計算書（jpigp）:
- `ShareOfLossProfitOfInvestmentsAccountedForUsingEquityMethodOpeCFIFRS`
- `LossGainOnSaleOfPropertyPlantAndEquipmentOpeCFIFRS`

日本基準（jppfs）の presentationArc では negativeLabel は **0 件**。

**8c. EDINET 固有の区分別 negativeLabel ロール**

Consolidated/role/negativeLabel 等の EDINET 固有 negativeLabel ロールは _lab.xml でラベルが定義されている（[F1]: Consolidated 2件、ConsolidatedInterim 5件、NonConsolidatedInterim 5件 等）が、presentationArc の preferredLabel として使用されている件数は **0 件**。

**8d. パーサーへの影響**

negativeLabel / negatedLabel は XBRL 仕様上「表示時に符号を反転する」ことを意味する。EDINET では IFRS のごく一部でのみ使用されるため、実装優先度は低い。ただし正確な表示のためには対応が必要。具体的には:
- preferredLabel が negativeLabel ロールの場合、Fact の数値を表示する際に符号を反転（正→負、負→正）
- ラベル文字列は既に負値用の表記（例:「持分法による投資損失」）が設定されているため、ラベル表示自体は通常のラベル解決で対応可能
