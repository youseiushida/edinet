### 訂正報告書の XBRL 提出条件（130, 150, 170）

仕様書より:
- 法定縦覧期間内の書類を訂正する場合 → XBRL 提出**必須**
- 法定縦覧期間満了後に訂正する場合 → XBRL 提出**不要**（has_xbrl=False となりうる）
- 訂正報告書は完全再提出形式: 元の書類と同一構造の ZIP が提出される

法定縦覧期間:
- 有報: 5年 + 延長5年 = 最大10年
- 四半期報告書: 3年 + 延長7年 = 最大10年
- 半期報告書: 5年 + 延長5年 = 最大10年

### doc_type=170 の特殊性

実データで doc_type=170（訂正半期報告書）のファイル名が `jpcrp040300-q2r` であった。これは元の半期報告書が四半期報告書形式（第四号の三様式）で提出されたことを示す。半期報告書は `jpcrp050000-ssr`（第五号様式）と `jpcrp040300`（第四号の三様式）の両方がありうる。

### パーサー設計への影響

1. **v0.1.0 で対応すべき doc_type**: 120, 130, 140, 150, 160, 170（財務諸表を含む主要書類）
2. **ZIP 構造は全 doc_type で共通**: `XBRL/PublicDoc/` + `XBRL/AuditDoc/`（カテゴリ B は AuditDoc なし）
3. **AuditDoc の有無で 2 グループ**: 財務諸表あり（AuditDoc あり） vs 財務諸表なし（AuditDoc なし）
4. **府令略号は多様**: jpcrp, jpsps, jpctl, jptoo, jplvh の 5 種。パーサーは府令略号に依存しない設計が望ましい
5. **カテゴリ B は最小構成**: 6〜9 ファイルで、PublicDoc に .xbrl 1件 + .xsd 1件 + .htm 2件 + manifest
6. **has_xbrl=False のケース**: 訂正報告書で法定縦覧期間満了後に提出された場合。パーサーは `has_xbrl` フラグで事前にフィルタすればよい
7. **特定有価証券（投資信託）**: `jpsps` 府令で別タクソノミ。同一 doc_type=120 に混在するため、府令略号またはタクソノミ名前空間でフィルタが必要

### 推論（Reasoning）

1. [F1] より、全 12 doc_type で has_xbrl=True の実 ZIP を取得でき、全てに .xbrl ファイルが含まれる。EDINET では XBRL 対象書類は必ず .xbrl を含む ZIP を提供する。
2. [F1] より、AuditDoc の有無で明確に 2 グループに分かれる: 財務諸表あり（doc_type=120/130/140/150/160/170/030）→ AuditDoc あり、財務諸表なし（180/235/240/290/350）→ AuditDoc なし。
3. [F1] より、カテゴリ B（財務諸表なし）は最小構成で 6 ファイル（.xbrl 1 + .xsd 1 + .htm 2 + manifest 1 + linkbase）。カテゴリ A は 21〜46 ファイル。
4. [F1] の府令略号から、EDINET では 5 種の府令タクソノミが使用される: jpcrp（開示府令）、jpsps（特定有価証券府令）、jpctl（内部統制府令）、jptoo（公開買付府令）、jplvh（大量保有府令）。
5. [F1] の doc_type=120 サンプルが投信（jpsps070000）であったことから、同一 doc_type に一般事業会社と投資信託が混在する。パーサーは府令略号で対象をフィルタする必要がある。
6. [F1]+[F2]+[F3] を総合すると、一般事業会社の有報（jpcrp030000）は PublicDoc .xbrl 1件 + AuditDoc .xbrl 2件が標準。投信は AuditDoc 3件になることもある。
