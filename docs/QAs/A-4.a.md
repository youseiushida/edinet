## A-4. Unit 要素の構造

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| A-4.1 通貨単位の表現 | `<xbrli:unit id="JPY"><xbrli:measure>iso4217:JPY</xbrli:measure></xbrli:unit>` が標準形式。仕様書 図表 5-5-1 で規定 |
| A-4.2 株式数の単位 | `<xbrli:unit id="shares"><xbrli:measure>xbrli:shares</xbrli:measure></xbrli:unit>`。仕様書 図表 5-5-4 で規定 |
| A-4.3 純粋数値の単位 | `<xbrli:unit id="pure"><xbrli:measure>xbrli:pure</xbrli:measure></xbrli:unit>`。割合(%)、整数、小数、人数に使用。仕様書 図表 5-5-3 で規定 |
| A-4.4 複合単位（EPS/BPS 等） | `xbrli:divide` 要素で表現。実例は下記参照 |
| A-4.5 unit が存在しない Fact | テキスト系 Fact には `unitRef` 属性なし。`ix:nonNumeric` 要素（文字列/文章）に unitRef は不要 |
| A-4.6 UTR 準拠 | EDINET は XBRL UTR に準拠。UTR に登録されているものを利用可能（仕様書 L.1454）。パーサーでの UTR バリデーションは不要 |
| A-4.7 複数通貨の共存 | 可能。仕様書 図表 5-5-2 で「その他の通貨」として `iso4217:USD` 等を定義可能。海外子会社のセグメント情報等で共存しうる |

### 複合単位（divide 構造）の実例

J-GAAP サンプル `.xbrl` L.3645-3654 からの **実物コピー**:

```xml
<xbrli:unit id="JPYPerShares">
<xbrli:divide>
<xbrli:unitNumerator>
<xbrli:measure>iso4217:JPY</xbrli:measure>
</xbrli:unitNumerator>
<xbrli:unitDenominator>
<xbrli:measure>xbrli:shares</xbrli:measure>
</xbrli:unitDenominator>
</xbrli:divide>
</xbrli:unit>
```

### EDINET で使用される全 Unit 一覧

| ユニット ID | measure / 構造 | 対応データ型 | 用途 |
|------------|---------------|-------------|------|
| `JPY` | `iso4217:JPY` | `monetaryItemType` | 日本円の金額 |
| `{通貨3文字}` | `iso4217:{通貨3文字}` (例: `iso4217:USD`) | `monetaryItemType` | 日本円以外の金額 |
| `pure` | `xbrli:pure` | `pureItemType`, `percentItemType`, `decimalItemType`, `nonNegativeIntegerItemType` | 割合(%)、整数、小数、人数 |
| `shares` | `xbrli:shares` | `sharesItemType` | 株式数 |
| `JPYPerShares` | `iso4217:JPY / xbrli:shares` (divide) | `perShareItemType` | 1株当たり金額(日本円) |
| `tCO2e` | `utr:tCO2e` | `dtr-types:ghgEmissionsItemType` | CO2換算温室効果ガス排出量 |

### 推論（Reasoning）

1. [F1]-[F8] より、EDINET 仕様書は 6 種類のユニットを定義している: JPY, その他通貨, pure, shares, JPYPerShares, tCO2e。このうち tCO2e は ESG（温室効果ガス排出量）対応として 2025 年版で追加された新しいユニットである。

2. [F10] より、J-GAAP サンプルの `.xbrl` には 4 ユニット（JPY, pure, shares, JPYPerShares）が定義されている。tCO2e と他通貨は使用されていないため未定義（[F12] の「参照されないユニットは定義しません」に従う）。

3. [F10] の divide 構造は仕様書 [F8] の記載と完全に一致する。パーサーでは `xbrli:divide` → `xbrli:unitNumerator` / `xbrli:unitDenominator` のネスト構造をパースする必要がある。

4. [F3] より、複数通貨の共存は仕様上可能である。海外子会社がある企業のセグメント情報等で `iso4217:USD` 等が使われるケースがありうる。ただし、一般的には全額が日本円換算されるため、`JPY` のみで足りるケースが多い。

5. [F4] より、`pure` ユニットは多用途で、割合（%）だけでなく整数・小数・人数にも使われる。パーサーでは `pure` を「無次元数値」として扱えばよい。

6. テキスト系 Fact（`ix:nonNumeric` / 従来型 XBRL のテキスト要素）には `unitRef` 属性がない。サンプル `.xbrl` の DEI 要素（`EDINETCodeDEI`, `FilerNameInJapaneseDEI` 等）で確認済み。

### 確信度

- 高（仕様書 6 種 + サンプル 4 種の完全一致 + P-2 の実データで裏付け済み）
- tCO2e は仕様書で確認済みだがサンプルデータでは未出現（使用するインスタンスがまだ少ないため）
