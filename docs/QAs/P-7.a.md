## P-7. テスト用サンプルデータの入手方法

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| P-7.1 EDINET が公式に提供するサンプル XBRL は存在するか | **存在する**。ガイドラインに「金融庁が提供するサンプルインスタンス」と明記されている。手元の 2026 サンプルインスタンス配下にも `.xbrl` が 56 件ある（PublicDoc 27 件 + AuditDoc 29 件）。 |
| P-7.2 `ALL_20251101/samples/` がそれに該当するか。pytest フィクスチャに使えるか | **インスタンスファイルには該当しない**。`samples/` は `entryPoint_*.xsd`（91件）のみで `.xbrl` は 0 件。したがって、インスタンスパーサーの pytest フィクスチャとしては直接使えない（タクソノミ解決/DTS の fixture には有用）。 |
| P-7.3 軽量で要素タイプが多いサンプルの推奨入手方法 | 公式 2026 サンプルインスタンスから選定するのが最短。最小サイズ群（47〜77 facts）は型が偏るため、**軽量かつ多様性重視なら `jplvh010000-lvh ... .xbrl`（115 facts）** を第一候補にする。`JPYPerShares` まで必要なら次点で `jpcrp040300-ssr ... .xbrl`（442 facts）を併用する。 |

- [F8] コマンド結果: `docs/QAs/scripts/results/P-7.sample_instances.2026-02-24.txt` L.1-2 — 2026 サンプルインスタンス配下の `.xbrl` は合計 56 件

### 推論（Reasoning）

1. [F1] + [F2] + [F8] より、EDINET/FSA の公式サンプル XBRL は存在し、手元データでも実体（`.xbrl`）を確認できる。
2. [F3] + [F4] + [F5] より、`ALL_20251101/samples/` はエントリーポイント XSD 集であり、インスタンス `.xbrl` の置き場ではない。したがって pytest の**インスタンス入力 fixture**としては不適、ただし **DTS/タクソノミ読込 fixture**には適している。
3. [F6] より、facts 数が最小のファイル群は `units=['pure']` かつ `dim_facts=0` が多く、型の網羅性が不足する。
4. [F7] より、`jplvh010000-lvh...`（115 facts）は軽量帯でありながら数値/非数値、dimension、複数 unit、boolean/date/decimal/text を同時に含む。よって「軽量かつ要素タイプ多様」の第一候補に合理性がある。
5. [F7] より、`JPYPerShares` のような追加ユニットまで含めるには `jpcrp040300-ssr...`（442 facts 以上）級が必要になる。単一ファイルで完全網羅を狙うより、用途別に 2 ファイル併用する方がテスト効率がよい。

### 確信度

- P-7.1: 高（仕様書明記 + 実ファイル確認）
- P-7.2: 高（`samples` の実体をファイル単位で確認）
- P-7.3: 中-高（実測統計に基づく推奨。ここでの「要素タイプ」は実運用上の入力タイプ多様性を指し、XSD の全データ型を1ファイルで完全網羅する保証ではない）
