## A-9. テキスト Fact の HTML コンテンツ

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| A-9.1 テキスト Fact は HTML フラグメントとして格納されるか | YES。`textBlockItemType` の Fact は XHTML フラグメントを含む。`stringItemType` は含まない |
| A-9.2 エスケープ HTML か混合コンテンツか | `.xbrl`（自動生成）: エスケープされた HTML 文字列（`&lt;p&gt;...`）。`.htm`（iXBRL）: `escape="true"` 属性により混合コンテンツ（生 XHTML）をインスタンス値に含める |
| A-9.3 `xml:lang` 属性の有無 | TextBlock Fact 自体には `xml:lang` なし。`xml:lang` は `link:footnote` / `ix:footnote` 要素にのみ付与される |

### 情報源（Fact）

- [F1] パス: `docs/仕様書/XBRL Instance Guideline 報告書インスタンス作成ガイドライン.md` L.1800-1813 — `ix:nonNumeric` 要素の仕様。escape 属性の役割:
  - `escape="true"`: 値にレイアウトの意味があり、XHTML タグをインスタンス値に含める（`textBlockItemType`）
  - `escape="false"`: 値にレイアウトの意味がなく、タグはインスタンス値に含めない（`stringItemType`、省略可）
- [F2] パス: 同上 L.1823-1829 — `stringItemType` では `escape="true"` を禁止。`textBlockItemType` は `escape="true"` を指定してテーブル・リスト等の XHTML タグを含む
- [F3] パス: 同上 L.1831 — 「`stringItemType` で `br` / `p` タグを使用しても、XBRL インスタンス値ではこれらのタグは削除される。ブラウザ表示のみに影響」
- [F4] パス: `docs/仕様書/EDINETタクソノミの設定規約書.md` L.2164-2169 — escape 属性の規約:

- [F5] パス: `docs/仕様書/EDINET提出書類ファイル仕様書.md` L.889-899 — XHTML 1.1 では `lang` 属性は廃止され `xml:lang` のみ使用可能

### .xbrl と iXBRL (.htm) での表現の違い

| 観点 | .xbrl（自動生成） | .htm（iXBRL） |
|---|---|---|
| textBlockItemType の HTML | エスケープ済み（`&lt;table&gt;`） | 生 XHTML（`<table>`）+ `escape="true"` |
| stringItemType の HTML タグ | 含まれない（EDINET が除去済み） | ブラウザ表示用のみ（インスタンス値には含まれない） |
| xml:lang | footnote 要素のみ | ix:footnote 要素のみ |

### 推論（Reasoning）

1. [F1]+[F2]+[F4] より、EDINET は `textBlockItemType` と `stringItemType` で明確に HTML の扱いを分けている。`textBlockItemType` は `escape="true"` により XHTML タグをインスタンス値に含め、`stringItemType` は含めない。

2. [F6]+[F7] より、自動生成 `.xbrl` 内の TextBlock Fact では、HTML タグが XML エンティティとしてエスケープされている（`&lt;table&gt;` 等）。これは XML ドキュメント内にエスケープされた文字列として格納される形式であり、混合コンテンツ（XML 要素として直接埋め込まれる形式）ではない。

3. [F8] より、iXBRL `.htm` ファイルでは `escape="true"` 属性が指定され、`ix:nonNumeric` 要素内に生の XHTML タグ（`<table>`, `<div>` 等）が直接記述されている。iXBRL → XBRL 変換時に、これらのタグがエスケープされて `.xbrl` に格納される。

4. [F3] より、`stringItemType` では `br` / `p` タグを HTML 表示用に記述できるが、XBRL インスタンス値には含まれない。v0.1.0 で `.xbrl` をパースする場合、stringItemType の値はプレーンテキストとして扱えばよい。

5. [F9]+[F10] より、`xml:lang` 属性は TextBlock Fact 自体には付与されず、footnote 要素にのみ付与される。テキスト Fact の言語は Context の `xml:lang` や Document 全体の言語設定で暗黙的に決まると推定される。

6. v0.1.0 パーサーへの影響: `.xbrl` をパースする場合、TextBlock の値は XML エンティティがデコードされた HTML 文字列として取得される（XML パーサーが自動的にエンティティを解決するため）。追加のエスケープ処理は不要。

### 確信度

- 高（仕様書の明示的記述 + サンプル `.xbrl` と iXBRL `.htm` の両方で実例確認済み）
- `xml:lang` がテキスト Fact に付与されない点: 高（全サンプルで確認済み。仕様書でも TextBlock Fact への `xml:lang` 付与は規定されていない）
