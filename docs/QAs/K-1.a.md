## K-1. has_xbrl フラグの信頼性

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| K-1.1 has_xbrl=True なのに XBRL なし | サンプル 5/5 件で一致（0/5 ミスマッチ）。フラグは信頼できる |
| K-1.2 has_xbrl=False なのに XBRL あり | iXBRL タグ内容検証済みで 0/5 ミスマッチ。フラグは信頼できる。詳細は推論 2 |
| K-1.3 撤回書類のダウンロード可否 | ダウンロード不可（404）。詳細は推論 3 |
| K-1.4 空提出（DEI のみ）ケース | サンプルでは未検出。詳細は推論 4 |

### 情報源（Fact）

- [F1] スクリプト: `docs/QAs/scripts/K-1.xbrl_flag.py` 実行結果 — 直近7日間で 1220 件取得、has_xbrl=True: 1155 件、has_xbrl=False: 65 件
- [F2] スクリプト実行結果（Step 2） — has_xbrl=True の5件サンプル: 全件で `XBRL/PublicDoc/` 配下に `_ixbrl.htm` および `.xbrl` ファイルが存在。スクリプト出力: `has_xbrl=True のうち実際に XBRL なし: 0/5`
- [F3] スクリプト実行結果（Step 3） — has_xbrl=False の5件サンプル: 全件で `PublicDoc/` 配下に `.htm` ファイルが存在（`XBRL/` プレフィックスなし、`_ixbrl` サフィックスなし）。スクリプトは各 .htm ファイルの内容をパースし、`xmlns:ix`, `<ix:`, `xmlns:xbrli` の有無を検証。**全件で iXBRL タグが検出されず plain HTML と判定**。スクリプト出力: `has_xbrl=False のうち実際に XBRL あり: 0/5`
  - 例: `PublicDoc/0000000_header.htm`, `PublicDoc/0000001_honbun.htm`（plain HTML 確認済み）
  - 書類種別: 発行登録追補書類、訂正確認書、訂正発行登録書
- [F4] スクリプト実行結果（Step 4） — 撤回書類（withdrawal_status="1"）: 1件検出。`download_document()` は `EDINET API error 404: Not Found` を返す
- [F5] スクリプト実行結果（Step 5） — 半期報告書（doc_type_code="160"）3件を DEI のみチェック: 全件で財務NS（`jppfs_cor` 等）が検出され、Fact 指標数も 9 程度。DEI のみのケースは未検出

### 推論（Reasoning）

1. [F2] より、has_xbrl=True のフラグは信頼できる。サンプル全件で `XBRL/PublicDoc/` 配下に XBRL 関連ファイルが存在した。
2. [F3] より、has_xbrl=False の書類にも `.htm` ファイルは存在するが、スクリプトによる iXBRL タグ内容検証（`xmlns:ix`, `<ix:`, `xmlns:xbrli` の正規表現検索）の結果、全件で iXBRL タグが検出されず plain HTML と判定された。加えて、パス構造にも差異がある:
   - XBRL あり: パス `XBRL/PublicDoc/*.htm`, ファイル名に `_ixbrl` サフィックス
   - XBRL なし: パス `PublicDoc/*.htm`（`XBRL/` プレフィックスなし）、`_ixbrl` サフィックスなし
   - パスパターン判定と内容検証の両方で一致しており、has_xbrl フラグは **正確** である。
3. [F4] より、撤回された書類は API から 404 が返り、ダウンロードできない。`withdrawal_status != "0"` の書類はスキップすべき。`Filing` モデルの `withdrawal_status` フィールドで事前フィルタリング可能。
4. [F5] より、今回のサンプルでは DEI のみの空提出は検出されなかった。ただし、調査対象が半期報告書（投資信託）のみであり、臨時報告書（doc_type_code="140"）はサンプル期間に存在しなかった。臨時報告書等での空提出の可能性は排除できない。→ 確信度: 中

### has_xbrl フラグ信頼性まとめ

| ケース | スクリプト生出力 | 結果 | 備考 |
|--------|------------------|------|------|
| has_xbrl=True → XBRL あり | `0/5` ミスマッチ | **信頼できる** | strict XBRL パス判定で確認 |
| has_xbrl=False → XBRL なし | `0/5` ミスマッチ | **信頼できる** | .htm は iXBRL タグ内容検証で plain HTML と確認 |
| 撤回書類 | 404 エラー | ダウンロード不可 | withdrawal_status でフィルタ |
| DEI のみ空提出 | 未検出 | 不明 | 臨時報告書等で可能性あり |

### 実装への示唆

- `has_xbrl` フラグを信頼して事前フィルタリングに使用可能
- `withdrawal_status != "0"` の書類はダウンロード前にスキップすべき
- `XBRL/PublicDoc/` パスプレフィックスと `_ixbrl` サフィックスが XBRL の判別基準（既存の `find_primary_xbrl_path()` もこの構造を前提としていると推測）

### 確信度

- **高**（has_xbrl=True の信頼性 — パス判定で確認）
- **高**（has_xbrl=False の信頼性 — パス判定 + iXBRL タグ内容検証の両方で裏付け）
- **高**（撤回書類のダウンロード不可）
- **中**（DEI のみ空提出 — サンプル範囲が限定的。臨時報告書のサンプルが不足）
