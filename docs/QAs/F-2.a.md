## F-2. DEI の利用シーン

### 質問への対応

q.md のサブ質問ごとに回答箇所を対応付ける。本回答は D-3.a.md および F-1.a.md の既存回答を統合したものであり、新規スクリプトは不要。

| サブ質問 | 回答 |
|----------|------|
| 会計基準の判別 | **可能**。`AccountingStandardsDEI` の値で判別できる。取りうる値は `"Japan GAAP"`、`"IFRS"`、`"US GAAP"`、`"JMIS"` の4種、または `xsi:nil="true"`（対象外書類）。これが最も正式かつ確実な判別方法。→ D-3 参照 |
| 業種カテゴリの判別 | **部分的（別記事業のみ）**。`IndustryCodeWhenConsolidatedFinancialStatementsArePreparedInAccordanceWithIndustrySpecificRegulationsDEI`（連結用）および個別用の2要素が存在するが、これは財務諸表等規則の「別記事業」（銀行、保険、電力等）に該当する場合のみ設定される。EDINET API の業種カテゴリ（C-3）とは異なる概念であり、一般企業の業種判別には使用できない。 |
| 連結/個別の有無の判別 | **可能**。`WhetherConsolidatedFinancialStatementsArePreparedDEI`（`booleanItemType`）で判別できる。`true` = 連結財務諸表あり、`false` = 連結財務諸表なし、`xsi:nil="true"` = 対象外書類。 |
| 報告期間の種別の判別 | **可能（通期/半期のみ）**。`TypeOfCurrentPeriodDEI` の値で判別できる。`"FY"` = 通期（年度）、`"HY"` = 半期（中間期）。四半期報告書制度は廃止されたため、四半期に対応する値は存在しない。なお、報告期間の種別は EDINET API の `doc_type_code` からも判別可能。 |

### 情報源（Fact）

観察事実のみを記載する。推論や解釈は含めない。本回答の Fact は既存回答からの引用である。

- [F1] D-3.a.md 推論1 + [F14] — `AccountingStandardsDEI` の取りうる値は `"Japan GAAP"` / `"IFRS"` / `"US GAAP"` / `"JMIS"` / nil の5パターン。添付5（提出者別タクソノミ作成ガイドライン添付5）で4値+nil が明記
- [F2] F-1.a.md 推論6 — `IndustryCodeWhen...DEI` は連結用・個別用の2要素が存在。日本語ラベルは「別記事業（連結）」「別記事業（個別）」。財務諸表等規則の「別記事業」に該当する場合の業種コードであり、EDINET API の業種カテゴリ（C-3）とは別概念
- [F3] F-1.a.md 推論3 + [F10] — `WhetherConsolidatedFinancialStatementsArePreparedDEI` は `xbrli:booleanItemType`。ドキュメンテーションラベルに `true`（連結あり）/ `false`（連結なし）/ nil（対象外）の3値が明記
- [F4] F-1.a.md 推論5 + [F11] — `TypeOfCurrentPeriodDEI` は `xbrli:stringItemType`。ドキュメンテーションラベルに `"FY"`（年度）/ `"HY"`（中間期）の2値が明記
- [F5] F-1.a.md [F19] — 添付5 Row 27: 「継続開示書類における当会計期間の「通期(FY)、中間期(HY)」の別を記載します。」（FY / HY の2値のみ、四半期に対応する値なし）

### 推論（Reasoning）

1. **会計基準の判別**: [F1] より、`AccountingStandardsDEI` は4つの会計基準を文字列値で直接示すため、最も確実な判別手段である。名前空間による推定（D-3.a.md 推論2）も補助的に可能だが、US-GAAP には専用名前空間がなく（D-2.a.md 参照）、`AccountingStandardsDEI` のほうが網羅的かつ正確。パーサー実装では DEI をまず読み取り、会計基準に応じた処理パスに分岐させるのが推奨される。

2. **業種カテゴリの判別**: [F2] より、DEI の `IndustryCode...DEI` は「別記事業」（銀行業、保険業等の特殊業種）に限定された概念である。一般的な業種分類（C-3 で扱う33業種分類等）は DEI には含まれない。業種判別が必要な場合は EDINET API の書類一覧レスポンスに含まれる業種情報を使用するか、証券コードから外部データベースで引く必要がある。

3. **連結/個別の有無**: [F3] より、`WhetherConsolidatedFinancialStatementsArePreparedDEI` で連結財務諸表の有無を判別できる。`true` の場合は連結財務諸表セクションが存在し、`false` の場合は個別財務諸表のみ。この情報は財務諸表パースの前処理（連結/個別のどちらを探索すべきか）で有用。

4. **報告期間の種別**: [F4] + [F5] より、`TypeOfCurrentPeriodDEI` は `"FY"` と `"HY"` の2値のみ。四半期報告書制度の廃止に伴い、四半期に対応する値は定義されていない。なお、EDINET API の `doc_type_code`（例: `120` = 有価証券報告書、`160` = 半期報告書）からも同等の判別が可能だが、DEI のほうが XBRL インスタンス内で自己完結的に判別できる利点がある。

- 業種カテゴリの判別: **高** — F-1.a.md で XSD 定義とラベルから「別記事業」限定であることを確認済み。C-3 の業種カテゴリとの非対応は概念の違いから明確
