## C-10. リンクベースの XML 構造（loc / arc / arcrole）

### 質問への対応

| サブ質問 | 回答 |
|----------|------|
| C-10.1 `link:loc` の構造 | `xlink:href` は相対 URI + フラグメント識別子（例: `../../jppfs_cor_2025-11-01.xsd#jppfs_cor_CashAndDeposits`）。`xlink:label` は concept 名そのもの（例: `CashAndDeposits`）で、`{prefix}_{name}` 形式のフラグメントから prefix を除いた部分と 100% 一致。スコープは同一 `extendedLink` 要素内でユニーク（同一 concept が同一リンク内で2回参照される場合は `_2` サフィックスを付与）。 |
| C-10.2 `link:presentationArc` の構造 | `xlink:from`/`xlink:to` は `link:loc` の `xlink:label` 値を参照。`xlink:arcrole` は `http://www.xbrl.org/2003/arcrole/parent-child`（固定）。`order` は表示順（小数可）。`preferredLabel` はラベルロール URI（省略時は標準ラベル）。詳細は後述の XML 例参照。→ C-6.a.md 参照 |
| C-10.3 `link:calculationArc` の構造 | `xlink:arcrole` は `http://www.xbrl.org/2003/arcrole/summation-item`（固定）。`weight` は arc 要素の属性で `"1"`（加算）または `"-1"`（減算）。cna 業種 BS の 57 アーク全てが `weight="1"`（BS は全て加算関係）。→ C-7.a.md 参照 |
| C-10.4 `link:definitionArc` の構造 | arcrole は 5 種: `all`(1), `hypercube-dimension`(1), `dimension-domain`(1), `dimension-default`(1), `domain-member`(75)。`all` アークのみ `xbrldt:closed="true"` と `xbrldt:contextElement="scenario"` を持つ。→ C-8.a.md 参照 |
| C-10.5 `link:labelArc` の構造 | `xlink:arcrole` は `http://www.xbrl.org/2003/arcrole/concept-label`（固定）。`xlink:from` は `link:loc` の `xlink:label`、`xlink:to` は `link:label` リソースの `xlink:label` を参照。1 つの loc に対して複数の label リソース（標準ラベル、冗長ラベル、業種別ラベル等）が接続される。 |
| C-10.6 具体的な XML 例 | 4 種全てについて実タクソノミファイルから抽出した完全な XML 例を後述のセクションに記載。 |

### 推論（Reasoning）

1. **C-10.1 loc 構造**: [F1][F5] より、`link:loc` の `xlink:href` は常に相対 URI + `#` + フラグメント識別子の形式。Presentation/Calculation/Definition の全 loc において `xlink:href` のパス部分は `../../jppfs_cor_2025-11-01.xsd` の 1 種のみ（79/79 = 100%）。`xlink:label` は XSD フラグメントの concept 名部分と 100% 一致しており、これは EDINET タクソノミの一貫した命名規則である。同一 extendedLink 内でユニーク性を保証するために `_2` サフィックスが使われるケースは Definition リンクで確認された（`ConsolidatedMember` が `dimension-default` と `dimension-domain` の両方のターゲットとなるため）。

2. **C-10.2 presentationArc**: [F1] の実ファイルで確認。`xlink:arcrole` は全 78 arc で `parent-child` のみ。`xlink:from`/`xlink:to` は `link:loc` の `xlink:label` 値を参照する。`order` は同一親の子要素間の表示順序を決定し、`1.0`, `2.0`, ..., `10.0` 等の値が使用される。`preferredLabel` は 19/78 arc で設定されており、合計行（`totalLabel`）と連結区分（`Consolidated/role/label`）の 2 用途が確認された。→ C-6.a.md に詳細な仕様書根拠あり。

3. **C-10.3 calculationArc**: [F2] の実ファイルで確認。`xlink:arcrole` は全 57 arc で `summation-item` のみ。`weight` は arc 要素の属性として設定される。BS の cna ファイルでは全て `weight="1"` だが、C-7.a.md で報告された PL のサンプルでは `weight="-1"` も使用される（費用科目の減算）。Presentation と異なり `Abstract` 要素は計算ツリーに含まれない（値を持つ科目のみ）。

4. **C-10.4 definitionArc**: [F3] の実ファイルで確認。5 種の arcrole が使い分けられる。BS の cna ファイルでは `domain-member` が 75/79 arc と大多数を占め、ハイパーキューブ定義（`all`, `hypercube-dimension`, `dimension-domain`, `dimension-default`）は各 1 arc ずつ。`all` arc のみ `xbrldt:closed="true"` と `xbrldt:contextElement="scenario"` の属性を持つ。科目ツリー部分は Presentation の `parent-child` 構造のミラーであり、全て `domain-member` arcrole で接続される。→ C-8.a.md に arcrole 一覧の詳細あり。

5. **C-10.5 labelArc**: [F4] の実ファイルで確認。`xlink:arcrole` は全 9,169 arc で `concept-label` のみ。`xlink:from` は `link:loc` の `xlink:label`（= concept 名）、`xlink:to` は `link:label` リソースの `xlink:label`（= `label_{ConceptName}` 等）を参照する。1 つの concept に対して平均 2.2 個のラベルリソースが接続されている（9,169 arc / 4,139 loc）。最小は標準ラベル + 冗長ラベルの 2 個、業種別ラベルを持つ concept では 6 個以上になる（例: `CashAndDeposits` は標準 + 冗長 + cns + ivt + sec = 5 個）。

6. **4 種の比較**: [F1]-[F4] + [F5] より、4 種のリンクベースは共通の `link:linkbase` > `link:roleRef` > `link:{type}Link` > `link:loc` + `link:{type}Arc` の XML 構造を持つ。主な差異は (a) arc 要素の arcrole 種類数、(b) 固有属性の有無、(c) Label のみリソース要素（`link:label`）を持つこと、(d) Definition のみ `link:arcroleRef` を持つこと、の 4 点。
