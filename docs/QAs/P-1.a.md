### 推論（Reasoning）

1. [F1]+[F2]+[F3]+[F4]+[F5] より、全5ターゲット（有報・四半期・訂正・銀行業含む）で `.xbrl` ファイルが ZIP 内に存在する。これは仕様書の「EDINET に登録すると XBRL インスタンスファイルが自動作成される」という記述（Instance Guideline L.393）と完全に一致する。
2. [F6] より、ZIP 構造は書類種別・企業規模・業種を問わず一貫している。`find_primary_xbrl_path` は `.xbrl` を PublicDoc/ から検索する設計であり、全ターゲットで正常に動作する。
3. [F3] より、四半期報告書は通期と同一構造だがファイル数が大幅に少ない（19 vs 45-71）。AuditDoc の監査報告書が1種類のみ（連結/個別ではなく四半期レビュー報告書のみ）。
4. [F4] より、銀行業（三菱UFJ）は一般事業会社と同じファイル名パターン `jpcrp030000-asr-001` を使用。業種固有のタクソノミ（jpbk 等）はファイル名には現れないが、XSD の import で参照される可能性がある。
5. [F5] より、訂正報告書は完全再提出形式。差分のみではなく、原本と同一構造の完全な ZIP が提供される。
